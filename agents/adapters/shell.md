# Shell Adapter

## Purpose

Use local shell commands as the deterministic baseline runner.

## Detection

- Bash/macOS/Linux: `command -v sh`
- Windows: `Get-Command powershell`

## Expected Inputs

- Role: planner, todo, verifier, wiki-maintainer, or watcher
- Board root
- Project root
- Runner id
- Command to execute

## Completion Contract

- Exit code `0`: completed or idle, depending on output keys.
- Non-zero exit code: failed.
- Output should prefer `key=value` lines when used by Autoflow.

## Approval Boundary

Shell commands must be configured by the user or generated by Autoflow runtime
scripts. The renderer should never execute arbitrary shell text directly.
