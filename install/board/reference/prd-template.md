# PRD Template

Use this for `tickets/prd/prd_NNN.md` files, whether created from an approved
`$autoflow` handoff or generated by the planner runner from an order.

The PRD is the first authoritative place for scope, assumptions, acceptance
criteria, and verification intent.

```md
# PRD prd_NNN: <title>

## Project

- ID: prd_NNN
- Title: <title>
- AI: <authoring agent>
- Status: draft | generated | approved
- Priority: normal
- Change Type: code | docs | cleanup | infra
- Requires Secrets: []

## Source

- Origin: tickets/order/order_NNN.md | conversation | manual
- User Request: "<short preserved request or summary>"
- Related Work: <wiki/ticket/order references when useful>

## Problem

<what is wrong, missing, confusing, or requested>

## Goal

<the outcome this PRD must make possible>

## Scope

- In Scope: <included behavior/modules/docs>
- Out of Scope: <excluded behavior/modules/docs>
- Assumptions: <planner/spec assumptions that should be verified or preserved>
- Remaining Unknowns: <unknowns that do not block ticket creation>

## Main Screens / Modules

- Module: <module, screen, command, or document area>
- Path: `path/to/file-or-folder`

## Allowed Paths

- path/to/file-or-folder

## Global Acceptance Criteria

- [ ] <observable criterion grounded in user-visible behavior, command output, or file evidence>
- [ ] <observable criterion with concrete paths, commands, states, or examples>
- [ ] <observable criterion covering reset/error/regression behavior when relevant>

## Verification

- Command: npm run test
- Command: none-shell
- Notes: <manual checks, browser checks, evidence expectations, or why no shell command applies>

## Conversation Handoff

- Source: <conversation/order path>
- Summary: <short Korean handoff for the next runner>

## Notes

- <constraints, related wiki/ticket references, risks, or split/follow-up notes>
```

## Notes

- Keep human-facing prose Korean by default. Preserve parser-sensitive headings,
  ids, paths, commands, code, and runtime formats.
- PRD acceptance criteria may later become Todo `Done When`; keep them concrete
  enough for `autoflow tool lint-ticket`.
