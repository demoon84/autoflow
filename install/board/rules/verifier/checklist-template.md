# Verifier Checklist Template

Use this checklist while reviewing one verifier ticket.

- [ ] Ticket Title and Goal are understood.
- [ ] Done When items are checked against the actual diff, not only worker notes.
- [ ] Acceptance Probe output is present or the absence is explained.
- [ ] Changed files are relevant to the ticket scope.
- [ ] Worker verification evidence is observable and reproducible enough for the risk level.
- [ ] Decision is one of `pass`, `revise`, or `replan`.
- [ ] Decision reason names the concrete evidence used.
