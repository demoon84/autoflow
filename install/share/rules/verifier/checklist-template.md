# Verifier Checklist Template

Verifier ticket 하나를 review할 때 이 checklist를 쓴다.

- [ ] Ticket Title과 Goal을 이해했다.
- [ ] Done When 항목을 worker note만이 아니라 실제 diff 기준으로 확인했다.
- [ ] Acceptance Probe output이 있거나, 없으면 이유가 설명되어 있다.
- [ ] 변경 파일이 ticket scope와 관련 있다.
- [ ] Worker verification evidence가 risk level에 충분히 관찰 가능하고 재현 가능하다.
- [ ] Decision이 `pass`, `revise`, `replan` 중 하나다.
- [ ] Decision reason이 사용한 구체적 evidence를 언급한다.
