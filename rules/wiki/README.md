# Wiki Rules

The wiki is derived project knowledge maintained from board files, logs, and
approved conversation summaries.

The board is the ledger. The wiki is the map.

## Source Priority

1. Tickets and verification records
2. Verifier logs
3. Project specs and plans
4. Approved conversation summaries
5. Existing wiki pages

## Maintenance Rules

- Link wiki claims back to tickets, verification records, or logs.
- Do not use wiki text as proof that work is done.
- Update `wiki/index.md` when pages are created or renamed.
- Append chronological changes to `wiki/log.md`.
- Flag contradictions instead of silently overwriting them.
- Keep summaries observable and specific.

## Page Types

- Project overview
- Feature page
- Decision page
- Architecture page
- Learning page
- Metrics summary
