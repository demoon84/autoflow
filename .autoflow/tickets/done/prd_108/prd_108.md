## Core Scope

- Goal: memo_062의 광범위한 "AI가 주도하고 shell은 도구" 전환을 worker가 한 턴에 끝낼 수 있는 첫 단계로 축소해, 기존 흐름을 깨지 않으면서 tool entrypoint와 계약을 먼저 명시한다.
- In Scope: `bin/autoflow` 또는 동등한 CLI entrypoint에 tool catalog/listing을 추가하거나 정리하고, planner/worker가 직접 호출하는 board/worktree/guard/wiki helper의 thin contract를 문서와 스크립트에 맞춰 노출하며, agent/protocol 문서가 shell을 workflow brain으로 설명하지 않도록 정렬한다.
- Out of Scope: 모든 shell 정책 분기 제거, retry/park/needs_user 런타임 의미 재설계, product UI 변경, 데스크톱 진행 시각화, destructive git 동작 추가, `.autoflow/wiki/` 직접 수정.

## Global Acceptance Criteria

- [ ] planner/worker/wiki가 직접 호출할 수 있는 thin tool 목록이 `bin/autoflow tool list` 또는 동등한 안정된 CLI/문서 entrypoint에서 확인된다.
- [ ] board 조회, worktree 상태 조회, diff 백업, guard 실행, wiki query, ticket 이동/상태 출력 등 핵심 helper는 "정책 결정 없이 동작과 결과만 반환"하는 contract로 정리된다.
- [ ] `start-plan.sh`, `run-role.sh`, 관련 공용 helper는 기존 parser-sensitive key (`status`, `source`, `replan_skipped.*`, `failure_class`, `recovery_state`, `board_root`, `project_root`) 호환을 유지한다.
- [ ] `.autoflow/agents/*` 와 `.autoflow/protocols/*` 는 shell/runtime 을 safety-kernel tool 로 설명하고, planning/recovery/merge/pass-fail 판단은 AI 역할의 책임으로 계속 명시한다.
- [ ] 이번 1차 슬라이스는 기존 retry limit, `needs_user`, dirty-root safety boundary를 제거하거나 우회하지 않고, 후속 단계에서 정책 분기 제거를 진행할 수 있는 도구 카탈로그 기반만 마련한다.
- [ ] 관련 scaffold mirror 와 smoke/contract 검증이 active board와 동기화된다.
