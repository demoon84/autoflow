# PRD 217 — SKILL 제거 Phase 6: Wiki content + AGENTS.md 정리

## Goal

`.autoflow/wiki/skills/` 의 사람-작성 컨텐츠를 정리(승급 또는 archive 이동)하고, `.autoflow/wiki/skills-local/` 디렉토리 전체를 비운다. AGENTS.md 의 rule 18a, 18b, 18c, 18d, 18e 단락을 제거하고 rule 18 의 `skill_review` evidence 등 잔여 문구를 정리한다. `.gitignore` 의 `!.autoflow/wiki/skills/` 예외 처리도 함께 제거한다.

## Background

- order_190 phase 6 (마지막). 코드/CLI/UI 가 모두 SKILL 을 더 이상 참조하지 않게 된 뒤 wiki 컨텐츠와 정책 문서를 정리한다.
- `.autoflow/wiki/skills/` (committed, 사람 작성): 의미 있는 항목은 `.autoflow/wiki/learnings/` 로 합치거나 AGENTS.md rule 로 승급, 나머지는 archive 디렉토리(또는 삭제). 사용자 가이드라인은 "원하는 항목은 learnings 로, 그 외는 정리".
- `.autoflow/wiki/skills-local/` (gitignored, 자동 생성): 통째 삭제 OK.
- AGENTS.md rule 18a~18e 제거. rule 18 의 wiki autocommit 정책에서 `skill_review.*` evidence 언급도 정리.
- `.gitignore` 의 `!.autoflow/wiki/skills/` (또는 동등 패턴) 예외를 제거해 wiki 전체가 일관되게 ignore 처리되도록.

## Scope

- `.autoflow/wiki/skills/`
  - 의미 있는 항목 추출 → `.autoflow/wiki/learnings/` 로 합치거나 AGENTS.md rule 로 승급.
  - 나머지 → `.autoflow/wiki/skills/.archive/` 로 이동 또는 직접 삭제.
  - 최종적으로 `.autoflow/wiki/skills/` 가 비어 있거나 `.archive/` 디렉토리만 존재해야 한다.
- `.autoflow/wiki/skills-local/`: 디렉토리 통째 삭제.
- `AGENTS.md`:
  - rule 18a, 18b, 18c, 18d, 18e 단락 제거.
  - rule 18 본문에서 `skill_review.*` evidence / `autoflow skill ...` 관련 문장 정리.
  - rule 번호 재할당 금지 — 이후 번호(19~27) 는 그대로 둔다 (다른 인용이 깨지지 않도록 18a~18e 만 제거하고 18 주변 정리).
- `.gitignore`: `!.autoflow/wiki/skills/` (또는 그에 상응하는 not-ignore 예외) 라인 제거. `.autoflow/wiki/` ignore 본 라인은 유지.

## Out of Scope

- 코드 / CLI / UI (phase 1~5 에서 처리됨).
- AGENTS.md 의 다른 rule.
- learnings 의 새 문서 작성 형식 표준화 (필요 시 별도 PRD).

## Allowed Paths

- `AGENTS.md`
- `.gitignore`
- `.autoflow/wiki/skills/**`
- `.autoflow/wiki/skills-local/**`
- `.autoflow/wiki/learnings/**`
