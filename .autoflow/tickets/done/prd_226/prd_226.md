## Core Scope

- Goal: 러너 config 변경 흐름을 "config.toml write → next-tick pickup + UI optimistic" 으로 정리하여 5–15s latency 를 < 1s 로 줄인다. 자동 재시작 트리거가 있으면 제거하고, 사용자 명시 "재시작" 액션은 별도로 보존한다.
- In Scope:
  - `apps/desktop/src/renderer/main.tsx` 의 `saveRunnerConfig` (line ≈ 2156):
    - dropdown 변경 → 저장 시 IPC 응답 이전에 카드 state 의 model/agent/reasoning 값을 optimistic 반영. IPC 실패 시 이전 값으로 롤백.
    - 저장 후 무거운 `loadBoard()` 전체 snapshot fetch 대신 해당 runner row 만 partial update 또는 `loadBoard()` 호출 자체를 제거 (다른 board 변화는 다음 polling cycle 에서 흡수).
    - `setRunnerAction(runner.id, "config_applying")` 의 lock 시간을 IPC 왕복으로 한정해 UI 가 1s 이상 잠기지 않게 한다.
  - `apps/desktop/src/main.js` 의 `configureRunner` (line ≈ 3571):
    - `runners set` 호출 후 자동 runner kill/respawn 트리거가 있으면 제거. 사용자가 "재시작" 별도 액션을 누른 경우만 재시작.
  - `packages/cli/runners-project.sh` 와 `runtime/board-scripts/runners-project.sh` 의 `runners set` 명령:
    - config.toml 갱신 후 자동 재시작 신호를 emit 하지 않음 (현재 emit 하면 제거).
  - `runtime/board-scripts/run-role.sh` (또는 `.autoflow/scripts/run-role.sh`) 의 tick 진입점:
    - 매 tick 시작 시 config.toml 의 model/agent/reasoning 을 다시 read 해 새 값으로 adapter 를 spawn. 이미 그렇게 동작하고 있으면 회귀 없는지 확인만.
- Out of Scope:
  - claude/codex/gemini CLI cold start 자체를 줄이는 warm-up cache (제어 어려움, 권장 안 함).
  - runner-common.sh 의 polling/realtime 구조 변경.
  - 사용자가 명시 "재시작" 액션을 눌렀을 때의 즉시 재시작 흐름 (보존).
  - 모델 swap 도중 진행 중 tick 의 강제 중단 (불가).
  - 새 IPC 채널 추가.
