## Core Scope

- Goal: Add YAML frontmatter to the 28 existing wiki pages currently flagged by `lint_missing_frontmatter.*` using a **deterministic-first** approach so the retrofit costs zero adapter tokens in the common case. Each retrofitted page gains `kind`, `slug`, `title`, `created`, `updated`, and `tags` derived purely from the file path, the first H1 line, and `git log`. The adapter is invoked only as a tie-breaker when a deterministic field cannot be filled, and even then it runs over a per-page byte-budget capped prompt. After the retrofit, `wiki lint` reports zero `lint_missing_frontmatter.*` entries and zero new `lint_orphan.*` / `lint_broken_link.*` regressions.
- In Scope:
  - New action `wiki-project.sh retrofit-frontmatter [project-root] [board-dir-name] [--dry-run] [--page <board-relative-path>] [--runner RUNNER_ID]` that walks `.autoflow/wiki/{decisions,features,learnings,architecture}/*.md` (excluding `README.md`, `index.md`, `log.md`, `project-overview.md`, and any page that already has a leading `---`).
  - Deterministic field extraction:
    - `kind` ← `decision` / `feature` / `learning` / `architecture` based on the page's directory under `wiki/`.
    - `slug` ← `basename "$file" .md`.
    - `title` ← first non-empty `# ` H1 line, with the leading `# ` stripped. If no H1 exists, fall back to a Title-Cased version of the slug.
    - `created` ← UTC ISO timestamp of the page's first commit (`git log --diff-filter=A --follow --reverse --format=%cI -- <file>`); fall back to today's date when git history has nothing.
    - `updated` ← UTC ISO timestamp of the page's most recent commit (`git log -1 --format=%cI -- <file>`); fall back to `created` when there is no later commit.
    - `tags` ← deterministic union of `[ kind, slug, ...directory ancestors below wiki/ ]`, deduplicated, lowercased, hyphenated. e.g. `wiki/features/finish-pass-inline-merge-summary.md` → `tags: [feature, finish-pass-inline-merge-summary, features]`.
  - The frontmatter is prepended to the file (followed by one blank line) only if no `---` block is already present. Existing body bytes are preserved verbatim.
  - `--page` mode retrofits a single page; default mode walks all eligible pages.
  - `--dry-run` prints the proposed frontmatter for each page on stdout in `key=value` form without modifying any file.
  - Output emits, per page, `retrofit.<n>.page=<path>` plus `retrofit.<n>.status={written|skipped_already_has_frontmatter|skipped_excluded|dry_run}`. Top-line summary keys: `retrofit_total`, `retrofit_written`, `retrofit_skipped`, `retrofit_dry_run`.
  - Optional adapter assist: when `--allow-adapter` is passed AND the deterministic title/tag extraction yielded a placeholder fallback for a given page, the script may call the wiki adapter on a per-page prompt capped at `AUTOFLOW_WIKI_RETROFIT_PROMPT_BYTES` bytes (default `4096`) to suggest a better `title` and at most three additional `tags`. Without `--allow-adapter`, the script never calls any adapter.
  - Update `.autoflow/agents/wiki-maintainer-agent.md` and `.autoflow/reference/wiki.md` to describe the action, the env var, and the deterministic-first invariant. `grep -n AUTOFLOW_WIKI_RETROFIT_PROMPT_BYTES .autoflow/agents/wiki-maintainer-agent.md` returns at least one hit.
  - Lint cross-check: after a successful retrofit run, `bash packages/cli/wiki-project.sh lint . .autoflow` emits `missing_frontmatter_count=0` and `lint_orphan.*` / `lint_broken_link.*` counts unchanged versus pre-retrofit (i.e. no regression).
- Out of Scope:
  - Frontmatter for `wiki/answers/*.md` and `wiki/sources/*.md` — those layers already get frontmatter at creation.
  - Frontmatter for `index.md`, `log.md`, `project-overview.md` — these are managed by the deterministic baseline and the retrofit must not touch their `AUTOFLOW:BEGIN`/`AUTOFLOW:END` regions.
  - A Dataview-style query layer over the new frontmatter — that's a separate follow-up.
  - Backfill of `source_count` or analytics fields. v1 of frontmatter only carries the six deterministic fields above; richer metadata is a follow-up.
  - Any change to the desktop UI or `update-wiki.sh` baseline.

## Notes

- Token cost rationale: this PRD is intentionally adapter-free in the common path. With `git log` and path heuristics handling all six fields, the entire retrofit costs zero adapter tokens. The `--allow-adapter` escape hatch exists so the operator can opt into adapter polish when title/tag extraction lands on a fallback, but each opt-in call is hard-capped by `AUTOFLOW_WIKI_RETROFIT_PROMPT_BYTES`.
- Risk: a page whose first non-empty line is not an H1 (e.g. starts with a paragraph) will get a Title-Cased slug as title. Acceptable for v1; the operator can re-run with `--allow-adapter` per page later.
- Risk: `git log --follow` can fail on pages that were created via copy rather than `git mv`. The script falls back to today's date for `created` in that case and logs a `retrofit_warning.<n>=missing_git_history_for=<page>` line so the operator can spot it.
- Risk: re-running on a page where the user manually wrote a partial `---` block (e.g. only `---\nkind: foo\n---`) is handled by skip-if-leading-dashes; we never merge into an existing frontmatter block in v1.
- Heartbeat: planner-1 will pick this PRD up on its next 1-minute tick. owner-1 needs only a shell adapter for the deterministic path (no LLM required), so this PRD is verifiable even when codex/claude are not on PATH.
