# Project PRD

## Meta

- Project Key: prd_095
- Title: Codex runner 기본 reasoning fast(=low) 일치
- Status: populated

## Goal

Autoflow에서 Codex agent를 사용하는 runner의 기본 reasoning을 "fast"로 정렬한다. Codex CLI의 `model_reasoning_effort`에는 `fast` 리터럴이 없으므로, repo의 기존 codex reasoning 옵션 체계(`low/medium/high/xhigh`)에 맞춰 가장 가벼운 단계인 `low`를 "fast"의 의미값으로 채택한다. 보드의 runner config, scaffold, CLI command preview, 데스크톱 runner 설정 UI가 같은 값을 일관되게 보여주도록 맞춘다.

## Core Scope

### In Scope

- `.autoflow/runners/config.toml`의 `owner-1`(agent=codex) `reasoning`을 현재 `medium`에서 `low`로 낮춰 현 보드의 Codex 실행이 fast 프리셋으로 동작하게 한다.
- `scaffold/board/runners/config.toml`의 codex 기본 runner(`planner-1`, `owner-1`, `wiki-1`)에서 빈 값으로 두던 `reasoning`을 `low`로 명시해 새 설치/스캐폴드도 fast 모드로 출하한다.
- `apps/desktop/src/renderer/main.tsx`의 `runnerOptionLabels`에 `codex` 키를 추가해 `low/medium/high/xhigh` reasoning 값을 한국어 라벨로 노출하고, `low` 라벨에 fast 의미를 함께 표기해 UI에서 fast=low 매핑이 명확하도록 한다(예: `"low": "낮음 (fast)"`).
- `apps/desktop/src/renderer/main.tsx`의 reasoning select가 codex 행에서도 라벨을 사용하도록 기존 `displayRunnerOption` 호출 흐름 안에서만 바뀐다(추가 컴포넌트/구조 변경 없음).
- `packages/cli/runners-project.sh`의 codex command preview가 `-c model_reasoning_effort="low"`를 그대로 출력하는지(이미 그렇게 동작) 확인하고 회귀가 없는지 검증한다. 코드 수정이 필요하지 않으면 변경하지 않는다.

### Out of Scope

- Codex CLI에 새로운 reasoning 리터럴(`fast`, `minimal` 등)을 추가하는 작업.
- `claude`/`gemini`/`opencode` agent의 reasoning 기본값/UI 변경.
- runner 추가/삭제, role 토폴로지 변경, runner config schema 변경.
- 데스크톱 다른 화면(작업 보드, 칸반, 위키, 스냅샷, 로그) 레이아웃/스타일 변경.
- 사용자가 이미 보드에서 직접 다른 reasoning 값을 저장한 경우의 마이그레이션/덮어쓰기 자동화.
- 새로운 테스트 추가나 smoke 신설(기존 verification command만 통과시키면 된다).
