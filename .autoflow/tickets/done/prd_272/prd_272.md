## Goal

LLM Wiki(`.autoflow/wiki/`)의 핵심 노이즈 원인인 자동 생성 `sources/` 파일 100개를 archive 하고, `index.md` 를 카테고리 기반으로 재작성하며, 모든 신규 wiki 페이지가 따를 4단 구조(Symptom / Cause / Fix / Verification)를 `page-template.md` 에 확정한다. `wiki-maintainer-agent.md` 도 이 구조를 의무화하도록 재작성하고, 언어 정책(한국어/영어 영역 분리)과 citations 강화 규칙을 템플릿에 박는다.

## Background

현재 wiki 는 233 파일 / 8.7K 줄이나 실질 가치 있는 내용은 11–16개에 불과하다. sources/ 100개가 PRD/ticket 본문의 한 단락 요약이라 RAG 검색 노이즈 점거, index.md 는 ticket 제목 ls 덤프, 페이지 평균 30–50줄에 행동 가능한 패턴 부재. 이번 PRD 는 기계적 정리(sources 이동) + 정책/구조 문서 재작성을 묶어 처리한다.

## Scope

- In Scope: `sources/` archive 이동, `index.md` 재작성, `page-template.md` 재정의, `wiki-maintainer-agent.md` 재작성, 언어 정책 및 citations 규칙, runtime 미러
