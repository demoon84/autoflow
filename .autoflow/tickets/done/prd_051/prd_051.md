# Project PRD

## Project

- ID: prd_051
- Name: Skip unchanged planner and ticket idle turns
- Title: Add unchanged-input adapter skips for idle planner and ticket runner ticks
- Goal: Reduce recurring Plan AI and Impl AI adapter token usage by skipping unchanged idle loop ticks after deterministic runtime preflight has already reported no actionable work.
- AI: planner-1
- Status: generated

## Core Scope

- Goal: Extend the existing unchanged-input skip pattern in `run-role.sh` beyond Wiki AI so planner and ticket loop runners can avoid adapter prompts when their watched board inputs are unchanged and the latest deterministic preflight is still idle.
- In Scope:
  - Add planner idle skip logic for loop-mode planner runs after `start-plan.sh` reports `status=idle` and `reason=no_actionable_plan_input`.
  - Add ticket idle skip logic for loop-mode ticket runs after `start-ticket-owner.sh` reports `status=idle` and `reason=no_actionable_ticket`.
  - Fingerprint only the board inputs that can change planner/ticket actionability, such as `tickets/inbox/`, `tickets/backlog/`, `tickets/reject/`, `tickets/todo/`, and `tickets/inprogress/`.
  - Persist the latest idle fingerprint in runner state files under `.autoflow/runners/state/` or the existing runner state directory.
  - Emit parseable skip output and runner log/state updates when an unchanged idle planner or ticket tick is skipped before `write_agent_prompt`.
  - Mirror the source/runtime implementation where `run-role.sh` exists in both CLI source and runtime board scripts.
  - Add smoke coverage that proves planner and ticket loop runners invoke the adapter on the first idle tick, skip the second unchanged idle tick, and invoke again when a relevant board input changes.
- Out of Scope:
  - Do not skip non-idle planner or ticket preflight results.
  - Do not skip when the runner is not in loop mode, `--dry-run` is used, or the role is not planner/ticket.
  - Do not change Wiki AI skip behavior except for shared helper extraction if needed.
  - Do not change `start-plan.sh`, `start-ticket-owner.sh`, ticket lifecycle, claim semantics, verification semantics, or implementation worktrees.
  - Do not suppress adapter calls when there is actionable work, active resume work, or a changed board fingerprint.
