## Core Scope

- Goal: runner adapter preflight 가 같은 차단 결과를 연속으로 반환할 때 단순 재시도 루프를 멈추고, active ticket 이 `healthy`처럼 보이지 않게 recovery evidence 를 남겨 planner/worker 흐름이 다음 safe action 으로 넘어가게 한다.
- In Scope:
  - `packages/cli/run-role.sh`와 `runtime/board-scripts/run-role.sh`의 budget/rate/prompt preflight skip 경로가 `consecutive_preflight_skip_count`, `consecutive_preflight_skip_result`, `last_preflight_skip_at` 또는 동등한 runner state 필드를 기록한다.
  - 같은 preflight skip result 가 threshold(기본 3회)에 도달하면 `last_result=circuit_breaker_tripped` 또는 동등한 machine-readable result 와 함께 원인 result, count, threshold, cooldown 종료 시각을 runner state/log 에 남긴다.
  - 반복 preflight circuit breaker 는 adapter prompt 를 또 실행하지 않으며, active ticket 이 있으면 `active_recovery_status=blocked`, `active_recovery_failure_class=tooling_failure`, `active_recovery_reason=repeated_preflight_skip` 또는 동등한 planner-visible recovery signal 을 노출한다.
  - runner/list 출력과 desktop bridge 가 반복 preflight breaker evidence 를 표시할 수 있도록 필요한 state/list 필드를 보존한다.
  - start-plan / planner recovery 경로가 반복 preflight recovery signal 을 board recovery 입력으로 인식할 수 있게 하되, planner 가 runner process 를 stop/restart 하거나 OS process 를 kill 하지는 않는다.
  - smoke test 를 추가해 임시 board 에서 worker preflight 가 같은 `token_budget_exceeded`를 3회 연속 만났을 때 circuit breaker state 와 recovery signal 이 기록되고, 같은 prompt retry 루프가 계속 돌지 않음을 검증한다.
- Out of Scope:
  - `prd_179` / `tickets_178`의 stale token-cache source/freshness guard 구현.
  - token budget 사용량 계산식, quota 기본값, 비용 환산 UI 재설계.
  - runner 또는 OS process 직접 stop/restart/kill, worktree 삭제, active ticket 수동 이동.
  - timeout 전용 `consecutive_timeout_count` 회로 재작성. 이번 작업은 preflight skip 반복 결과를 다룬다.
  - `tickets/inprogress/tickets_177.md`의 adapter-running heartbeat 구현 자체 변경. 필요한 경우 recovery signal 표시에만 접점을 둔다.
  - `git push`.
