# Project

- ID: prd_157
- Title: Reasoning level 동적 dispatch — tick 작업 복잡도 기반 reasoning 자동 선택
- AI: planner/worker
- Status: draft

## Core Scope

- Goal: 단순 tick (idle 확인, 단순 ticket 처리) 은 reasoning=low, 복잡한 tick (PRD 분할, 여러 reject 재계획) 은 high 로 동적 dispatch 해 thinking tokens 15-25% 절감.
- In Scope:
  - **복잡도 추정 휴리스틱**:
    - 입력 사이즈 (tickets count, PRD body 크기)
    - 작업 종류 (idle / single ticket / multi-PRD / reject 처리)
    - 직전 tick 결과 (success → low, failed/blocked → high)
  - **dispatch 매핑**:
    - simple (idle, 단일 ticket) → low
    - normal (단일 PRD 처리) → medium
    - complex (multi-PRD, reject 재계획, blocked recovery) → high
  - `packages/cli/run-role.sh` 의 어댑터 호출 직전 reasoning 동적 결정:
    - config.toml 의 reasoning 은 default
    - 동적 결정 결과로 override (단, config 가 명시적 fixed 면 무시)
  - 환경변수: `AUTOFLOW_REASONING_DYNAMIC_ENABLED` (기본 0 = off, 1 로 opt-in).
- Out of Scope:
  - ML 기반 복잡도 학습 — 휴리스틱만.
  - 모델 선택 동적 (별도 PRD).
  - reasoning level 이 없는 어댑터 (gemini 일부) 는 적용 안 함.
