## Project

- Status: draft

## Core Scope

- Goal: 데스크톱 앱의 "작업" 페이지 위키봇(`wiki-1`) 카드에서 사용자가 모델만이 아니라 agent 자체를 codex / claude / gemini 셋 중 어느 것이든 직접 골라 저장할 수 있게 한다. 현재는 작업 페이지 카드의 agent 셀렉터가 숨겨져 있어 모델 dropdown 이 현재 agent (예: gemini) 의 모델 목록으로 잠겨 보이고, 다른 agent 로 바꾸려면 Settings/Runners 패널까지 들어가야 한다.
- In Scope:
  - `apps/desktop/src/renderer/main.tsx` 의 작업 페이지 runner 카드(ai-progress-row, 약 5468행의 `RunnerConfigControls` 호출) 에서 위키봇(role=`wiki-maintainer`) 카드에 한해 `showAgent={true}` 로 agent 셀렉터를 노출한다.
  - agent 셀렉터의 옵션은 이미 정의된 `runnerAgentOptions = ["codex", "claude", "gemini"]` 그대로 — 세 개 다 선택 가능해야 한다.
  - agent 변경 시 옆의 모델 dropdown 과 추론 강도 dropdown 이 새 agent 의 옵션 (codex → `gpt-5.x` 시리즈 / claude → `opus`, `sonnet`, `haiku` 등 / gemini → `gemini-3-flash-preview` 등) 으로 자동 재정렬된다 (이미 `normalizeRunnerSelections` 가 처리하는 동작 유지).
  - 저장(✓) 버튼이 agent 변경분도 `.autoflow/runners/config.toml` 의 `wiki-1` 에 반영하고 다음 tick 부터 새 어댑터로 호출되게 한다 (이미 `hasDraftChanges` 가 agent 비교를 포함하므로 동작 확인만 필요).
  - 위키봇 카드 layout / CSS (`runner-config-no-agent` 클래스, grid template) 가 agent 셀렉터 추가로 깨지지 않도록 조정.
- Out of Scope:
  - 새로운 LLM 어댑터 추가 (codex / claude / gemini 어댑터는 이미 `run-role.sh` 에 존재).
  - 두 wiki 런너 동시 실행 (이전 초안의 멀티 런너 안 — 사용자가 단일 런너 운영을 원함).
  - Settings/Runners 패널 변경 (이미 agent 셀렉터가 디폴트 노출).
  - Plan AI / Impl AI progress 카드의 agent 셀렉터 노출 정책 (필요하면 후속 PRD 에서 다룸).

## Global Acceptance Criteria

- [ ] 작업 페이지 위키봇 카드(role=`wiki-maintainer`) 에 agent 셀렉터가 노출되고, 옵션이 `codex` / `claude` / `gemini` 셋 다 보인다.
- [ ] 셀렉터에서 `codex` 를 고르면 옆 모델 dropdown 이 codex 모델 목록 (`gpt-5.5`, `gpt-5.4`, `gpt-5.4-mini`, `gpt-5.3-codex`, `gpt-5.3-codex-spark`, `gpt-5.2`) 로, `claude` 면 (`opus`, `opus-1m`, `sonnet`, `haiku`) 로, `gemini` 면 (`gemini-3-flash-preview`, `gemini-2.5-pro`, `gemini-2.5-flash`, `gemini-2.5-flash-lite`) 로 자동 갱신된다.
- [ ] 셀렉터에서 `codex` 또는 `claude` 를 고르면 추론 강도 dropdown 이 활성화되고 해당 agent 의 옵션이 보인다. `gemini` 를 고르면 추론 강도 dropdown 은 "지원 안 함" 상태로 비활성화된다 (현재 동작 유지).
- [ ] agent 를 변경하고 저장(✓) 을 누르면 `.autoflow/runners/config.toml` 의 `[[runners]] id = "wiki-1"` 항목 `agent` / `model` / `reasoning` 필드가 새 값으로 기록되고, 다음 tick 의 `.autoflow/runners/logs/wiki-1.log` `adapter_start` 라인에 새 agent 가 찍힌다.
- [ ] agent 셀렉터 추가로 위키봇 카드 layout 이 깨지거나 가로 스크롤이 생기지 않는다 — Plan AI / Impl AI 카드 layout 은 영향 받지 않는다 (위키봇 카드만 노출 변경).
