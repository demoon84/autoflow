## Goal

현재 PTY mode 워커는 `ensureBoardWatcher()` 의 fs.watch 이벤트에만 의존해 wake 받는다. Allowed Paths 가 `tickets/` 밖(`.autoflow/wiki/`, `.autoflow/agents/` 등)일 때 자기강화 wake 트리거가 0건이 되어 최대 수 시간 정체가 실측됐다(Todo-279, 2026-05-10 01:01~08:13 KST, 약 7시간). `queueHasPendingWork` 게이트 + idle 감지 + fingerprint 캐시 기반의 안전망 폴링을 도입해, **빈 큐 헛 wake 0회 + 작업 중 방해 0회 + 동일 fingerprint 반복 wake 0회** 를 보장한다. fs.watch 는 빠른 신호용으로 유지하되 같은 게이트로 통과시킨다.

## Background

실측 정체 케이스: Todo-279(Wiki 구조 정리) — Allowed Paths 가 `.autoflow/wiki/`, `.autoflow/agents/` 라 워커 편집이 worker wake 라우팅 경로(`tickets/todo/`, `tickets/inprogress/`) 밖이어서 self-sustaining wake 0건. `apps/desktop/src/main.js:1050` 의 백업 폴링은 코드 주석에만 존재하고 미구현. fs.watch recursive:false / handle 손실 / OS sleep-wake 시 silent event drop 으로 더욱 취약.
