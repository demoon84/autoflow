# Project

- ID: prd_152
- Title: Verifier 모델/reasoning 다운그레이드 + ABA 검증으로 토큰 30% 절감
- AI: planner/worker
- Status: draft

## Core Scope

- Goal: Verifier runner 의 모델/reasoning 을 한 단계 낮춰 누계 토큰 ≈30% 절감하면서 검증 결과 품질(통과율 / fail 정확도) 은 동등 수준 유지.
- In Scope:
  - `.autoflow/runners/config.toml` 의 `[runners]` `verifier` 항목:
    - `model = "opus-1m"` → `model = "sonnet-4.6"` (또는 동등 mid-tier)
    - `reasoning = "high"` → `reasoning = "medium"`
  - **ABA 검증 프로토콜** (필수):
    - 변경 전 baseline 캡처: `.autoflow/metrics/token-cache.tsv` 의 verifier 누계, verifier_pass_count, verifier_fail_count, 평균 호출당 토큰 (직전 7일).
    - 변경 후 7일 운영 → 같은 지표 측정.
    - 비교 기준:
      - 토큰: -20% 이상이면 통과
      - pass_rate: ±3%p 이내 동일이면 통과
      - fail_count: 비정상적 증가 (baseline × 1.5 초과) 없으면 통과
    - 한 항목이라도 미통과 시 자동 롤백 (config 원복) + 결과 기록.
- Out of Scope:
  - planner / worker / wiki 모델 변경 (별도 PRD).
  - claude CLI 자체의 capability 변경.
  - 검증 로직 자체 (verifier-agent.md 흐름) 변경.
