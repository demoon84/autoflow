## Core Scope

- Goal: wiki + `tickets/done/` 본문을 sqlite FTS5 virtual table 로 색인하고, `--rag` 쿼리를 FTS5 MATCH + bm25() 로 전환한다. 인덱스는 gitignored cache 에 두고, FTS5 / sqlite / 인덱스 미존재 시 기존 chunk grep fallback 으로 안전하게 떨어진다.
- In Scope:
  - 인덱스 스키마: `wiki_search` FTS5 virtual table (`path`, `chunk_idx`, `title`, `body`, `tokenize="unicode61 remove_diacritics 2"`).
  - 인덱서 스크립트 `.autoflow/scripts/wiki-search-index.sh` (idempotent — 같은 path+chunk_idx+content_hash 면 skip).
  - 인덱스 위치: `.autoflow/runners/state/wiki-search.db` (gitignored).
  - chunk 단위: 1024 char (overlap 128 char).
  - `packages/cli/wiki-project.sh` 의 `--rag` mode 가 인덱스가 있고 FTS5 사용 가능할 때 MATCH + bm25() 로 top-k 반환. 결과 포맷은 기존 chunk 결과와 호환 (`path`, `chunk_start_line`, `chunk_end_line`, `score`, snippet).
  - opt-in env: `AUTOFLOW_WIKI_FTS_INDEX=on` 일 때만 인덱서가 동작. 기본 off (fail-safe). query 측은 인덱스 파일이 있으면 자동 사용.
  - fallback: 인덱스 없음 / FTS5 미빌드 / sqlite3 미설치 → 기존 chunk grep 으로 떨어지고 stderr 에 한 줄 경고.
  - `.gitignore` 에 `.autoflow/runners/state/wiki-search.db` 추가 (이미 `state/` 자체가 gitignored 면 노옵).
  - `AGENTS.md` rule 18 (wiki 자동화 규칙) 에 "RAG 인덱스는 phase 1 에서 sqlite FTS5 + BM25 이며 인덱스 파일은 gitignored, opt-in" 한 문단 추가.
- Out of Scope:
  - Phase 2: Wiki AI runner 의 incremental re-index 자동화 (별도 후속 order/PRD).
  - Phase 3: vector embedding / sentence-transformers / hybrid re-rank.
  - Anthropic / OpenAI embedding API 호출.
  - skill 시스템의 RAG hook 유지/제거 결정 (prd_215 와의 충돌은 본 PRD 가 건드리지 않음 — 사용자 별도 결정 사항).
  - `autoflow wiki search-rebuild` 같은 사용자 강제 재구축 CLI (phase 2 후보).
