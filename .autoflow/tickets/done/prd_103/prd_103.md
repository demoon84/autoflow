## Core Scope

- Goal: 기존 `AiProgressRow` 2줄 구조는 유지하면서 `.ai-progress-track`의 track line, dot, 단계 label 배치가 카드 내부 가용 폭을 더 넓게 쓰도록 CSS 중심으로 조정한다.
- In Scope:
  - `apps/desktop/src/renderer/styles.css`의 `.ai-progress-track`, `--progress-track-inset`, `--progress-track-width`, `.ai-progress-step`, `.ai-progress-dot` 관련 계산을 점검하고 조정한다.
  - AI 대시보드의 planner / worker / wiki runner 카드 모두에서 단계 트랙이 중앙 일부 폭에 몰리지 않고 좌우로 넓게 퍼져 보이도록 한다.
  - 첫 단계와 마지막 단계의 dot/label이 카드 border, 내부 padding, 상태 badge, 시작/정지 controls와 겹치지 않도록 여백을 유지한다.
  - 좁은 desktop 폭에서도 단계 label이 겹치거나 이전/다음 영역을 침범하지 않게 유지한다.
  - 필요할 때만 `apps/desktop/src/renderer/main.tsx`의 track wrapper/className 구조를 좁게 보조 수정한다.
- Out of Scope:
  - runner 상태 계산, 단계 이름, 단계 수, 시작/정지 동작 변경.
  - 기존 `AiProgressRow` 헤더 재배치 구조를 다시 설계하는 변경.
  - 토큰 사용량, runner label, status badge, active ticket button, config controls의 기능 변경.
  - desktop 앱의 다른 카드/보드 레이아웃 변경.
  - retry-limit reject(`reject_003`, `reject_071`, `reject_074`) 재시도 또는 dirty-root finalization blocker 처리.
