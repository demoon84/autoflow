- Status: draft

## Core Scope

- Goal: 데스크톱 작업 흐름 페이지의 UI 를 한 번에 다듬는다. 사이드 메뉴 라벨 단순화, AI 카드 헤더에서 어댑터 / runner-id / 모델 정보를 제거해 역할 이름만 노출, AI 관리 페이지에 흩어져 있던 시작/정지·모델/추론 저장 컨트롤을 각 AI 카드 안으로 흡수해 한 화면에서 전부 조작 가능하도록 만든다. 결과적으로 사용자는 작업 흐름 페이지만 열어 두면 모니터링 + 제어 + 설정을 모두 처리한다.
- In Scope:
  - **사이드 메뉴**:
    - `apps/desktop/src/renderer/main.tsx` 의 `settingsNavigation` 배열에서 `progress` 항목 라벨을 `작업 흐름` → `작업` 으로 변경.
  - **AI 카드 헤더 단순화**:
    - 카드 좌측 상단의 어댑터/runner 라벨을 다음으로 교체:
      - `Claude(planner-1)` → `Planner`
      - `Codex(AI-1)` 또는 `Codex(owner-1)` → `Worker`
      - `Codex(위키봇)` 또는 `Codex(wiki-1)` → `위키봇`
    - 모델·추론 보조 텍스트(`Opus 4.7 · 높음`, `gpt-5.5 · medium` 등) 를 카드 헤더에서 **삭제**. 모델/추론 정보는 카드 내부 컨트롤로 이동(아래 항목).
    - 라벨 매핑은 `runner.role` 기준으로 결정(`planner|plan` → `Planner`, `ticket-owner|owner` → `Worker`, `wiki-maintainer|wiki` → `위키봇`). runner.id 가 정의된 슬러그 외인 경우 fallback 으로 기존 표기 유지.
  - **프로그래스바 줄바꿈**:
    - 카드 헤더의 단계 표시기(progress dots + label) 가 좁은 폭에서 한 줄에 들어가지 않을 때 **여러 줄로 wrap** 되도록 CSS `flex-wrap: wrap`(또는 grid auto-flow) 적용. 단계 라벨이 잘리거나 ... 처리되지 않게.
    - 유저 액션 영역(시작/정지) 과 충돌하지 않도록 별도 row 로 강제 가능.
  - **시작/정지 버튼을 카드 우측 상단으로**:
    - AI 관리 페이지에 있는 runner 의 start/stop/restart 컨트롤을 각 AI 카드의 우측 상단에 inline 으로 배치(아이콘 버튼). 상태(`running`/`stopped`/`failed`) 에 따라 토글 표시.
    - shadcn `Button` (variant="ghost" 또는 "outline") + lucide 아이콘(`Play`, `Square`, `RotateCw`) 사용.
    - 같은 핸들러(`onControl(action, runnerId)`) 재사용해 두 군데에서 같은 동작.
  - **모델/추론/저장 컨트롤을 터미널 위로**:
    - AI 카드 안의 콘텐츠 박스(현재 stdout 미리보기/diff 가 보이는 영역) **위쪽에** 한 줄 컨트롤 row 추가:
      - 모델 셀렉터 (예: `Opus`, `Opus 4.7`, `gpt-5.5`, ...)
      - 추론 셀렉터 (예: `medium`, `high`)
      - "저장" 버튼 (현재 dropdown 값으로 runner config 갱신)
    - 변경 후 저장하지 않은 상태에서는 버튼이 활성화되고 "저장" 라벨 옆에 dot indicator 로 dirty 표시.
    - 어댑터별 옵션은 기존 `runnerAgentModelOptions` / `runnerAgentReasoningOptions` 재사용. prd_017 의 claude=medium/high 정책도 이 위치에서 그대로 적용.
  - **AI 관리 페이지의 중복 컨트롤 처리**:
    - 같은 컨트롤이 양쪽에 보이면 혼란이라, AI 관리 페이지의 runner 콘솔에서는 컨트롤을 **유지하되 시각적으로 동등하게** 작동(둘 중 어디서 저장해도 동기화). 즉 두 곳이 같은 상태를 공유하는 controlled 컴포넌트.
    - 또는 AI 관리 페이지를 향후 단순 목록(테이블/Add/Remove 만)으로 정리하는 후속 PRD 의 사전 단계로 두기 — 본 PRD 에서는 "양쪽 동기화" 만 보장.
- Out of Scope:
  - AI 관리 페이지 자체 레이아웃 재설계 (컨트롤 hoist 후 빈 자리 정리는 다른 PRD).
  - AI 카드 내부 콘텐츠 박스(터미널 스타일 stdout) 의 디자인 변경.
  - 메뉴 아이콘(`Workflow`) 변경.
  - 데이터 모델(`AutoflowRunner` 타입) 변경.

## Global Acceptance Criteria

- [ ] 사이드 메뉴의 `작업 흐름` 항목이 `작업` 으로 표시된다.
- [ ] 작업 흐름 페이지의 각 AI 카드 헤더 좌측이 `Planner` / `Worker` / `위키봇` 중 하나로만 표시된다(어댑터·runner-id 없음).
- [ ] AI 카드 헤더에서 모델·추론 보조 텍스트(`Opus 4.7 · 높음` 등) 가 사라진다.
- [ ] 카드 폭이 좁아져 단계 dots 가 한 줄에 못 들어갈 때 두 줄 이상으로 wrap 되며, 라벨이 잘리거나 dots 가 겹치지 않는다.
- [ ] 각 AI 카드 우측 상단에 시작/정지/재시작 아이콘 버튼이 있고, 누르면 AI 관리 페이지의 같은 동작이 호출된다(같은 `onControl` 핸들러).
- [ ] 각 AI 카드의 터미널 박스 위쪽 한 줄에 모델 셀렉터, 추론 셀렉터, 저장 버튼이 있고, 변경 후 저장 버튼이 활성화 + dirty dot 표기, 클릭 시 runner config.toml 이 갱신되어 다음 tick 에 반영된다.
- [ ] AI 관리 페이지에서 같은 runner 의 모델/추론을 바꿔 저장하면 작업 페이지의 카드 컨트롤도 동기화된다(반대 방향도 동일).
- [ ] `cd apps/desktop && npx tsc --noEmit` 가 0 errors.
- [ ] `cd apps/desktop && npm run check` 가 통과한다.
- [ ] 시각 회귀: 통계, Wiki, 티켓 정보 페이지 영향 없음.
