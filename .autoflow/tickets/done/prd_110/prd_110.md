## Core Scope

- Goal: 기존 진행 카드의 2행 구조와 progress track/controls 흐름은 유지하면서, `AiProgressRow` 헤더의 토큰 사용량 표시 정책과 아이콘/텍스트 정렬을 runner 공통 기준으로 정리한다.
- In Scope:
  - `apps/desktop/src/renderer/styles.css`의 `.ai-progress-agent`, `.ai-progress-agent-title`, `.ai-progress-token-usage`와 관련 광역 ellipsis 규칙을 점검해 토큰 사용량이 잘리지 않도록 조정한다.
  - planner / worker / wiki 진행 카드 모두에서 토큰 사용량 라벨이 말줄임표 없이 자연 폭 또는 허용된 줄바꿈으로 전부 보이게 한다.
  - 같은 줄의 아이콘, runner 이름, 토큰 사용량 텍스트가 시각적으로 세로 가운데에 맞도록 line-height, flex alignment, wrapper 구조를 정리한다.
  - `apps/desktop/src/renderer/main.tsx`는 CSS만으로 정렬이 안전하지 않을 때에만 좁게 보조 수정한다.
  - `hideProgressTrack` 분기와 worker/non-worker 헤더 변형 모두에서 같은 정렬 정책이 유지되게 한다.
- Out of Scope:
  - progress track 단계 폭/간격 자체 재조정.
  - runner 카드 1행 3열 보드 배치 재설계.
  - 메인 윈도우 최소 폭 정책 변경.
  - start/stop 버튼, adapter/model/reasoning selector, runner lifecycle 동작 변경.
  - retry-limit reject(`reject_003`, `reject_071`, `reject_074`) 재시도 또는 broad dirty-root finalization blocker 처리.
