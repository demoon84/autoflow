## Core Scope

- Goal: Phase 1+2 위에 skill 활용 mechanism 추가. planner/worker 가 새 작업 받을 때 매칭 skill 자동 주입 + 사람/스크립트가 명시 호출 + 활용 결과 stats 자동 기록.
- In Scope:
  - **Skill 매칭 (RAG-style)**:
    - 입력 (PRD title / order body / ticket scope) 에서 keywords 추출
    - 매칭 점수: keyword 점수 (50%) + path overlap (30%) + pattern_type 가중치 (20%)
    - top-N (기본 N=2) skill 본문을 prompt 의 system 영역 또는 별도 `<learned_skill>` 태그로 주입
    - 환경변수:
      - `AUTOFLOW_SKILL_INJECTION_ENABLED` (기본 1)
      - `AUTOFLOW_SKILL_INJECTION_TOP_N` (기본 2)
      - `AUTOFLOW_SKILL_INJECTION_MIN_SCORE` (기본 30, 0~100)
  - **Explicit invocation**:
    - `autoflow skill apply <project-root> <board-dir-name> <skill_name> --to-ticket <ticket_NNN>` — 명시 호출. ticket prompt 에 강제 주입.
    - 데스크톱 UI 의 ticket 카드에 "skill 적용" 버튼 (Phase 4 와 연동).
  - **활용 stats 자동 갱신**:
    - skill prompt 주입 시 sidecar `view_count`, `last_viewed_at` 자동 bump.
    - LLM 응답에서 `skill_used: <skill_name>` marker 검출 → `last_used_at` 갱신.
    - 후속 verify 결과:
      - pass → `success_count++`
      - fail/reject → `failure_count++`
      - threshold (예: failure_count > 3 && success_rate < 50%) 도달 시 sidecar `state: low_confidence` 마킹 → 매칭 가중치 자동 하향.
  - **Template 변수 + Preprocessing**:
    - SKILL.md 본문에 `${AUTOFLOW_BOARD_ROOT}`, `${AUTOFLOW_TICKET_ID}`, `${AUTOFLOW_PRD_KEY}` 변수 (preprocessing 시 치환).
    - 인라인 shell ``!`cmd` `` 는 1차 제외 (보안 위험).
  - **Skill nudge (활용 측)**:
    - LLM 응답에 "이 작업 패턴이 자주 나옴" 감지 시 → 다음 tick 에 `autoflow skill auto-extract` 자동 호출 (Phase 2 와 정합).
- Out of Scope:
  - 데스크톱 UI (Phase 4)
  - Security scan (Phase 5)
  - agentskills.io 호환 (Phase 5)
  - skill clustering (Phase 5+)
  - Deterministic mode (skill 신뢰도 100% 시 LLM 호출 없이 직접 실행) — 후속 PRD
