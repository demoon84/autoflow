## Goal

PRD_253/Todo-257이 false-pass로 처리된 것을 교정한다. `.autoflow/scripts/runner-stage.js`를 실제로 구현하고, `start-ticket-owner.sh`와 `finish-ticket-owner.sh`에 자동 호출을 통합한다. `apps/desktop/src/main.js` buildInitialPrompt에 stage 호출 지시를 명시하고, agent.md 3개에 워크플로를 기술한다. LLM 능동 보고가 아니라 shell 스크립트가 stage를 강제 업데이트해서 `active_ticket_id` stale 문제를 해결한다.

## Background

- PRD_253/Todo-257이 runner-stage.js 도입(LLM 능동 stage 보고)을 false-pass로 완료 처리함
- 실제 파일 없음: `.autoflow/scripts/runner-stage.js` 미존재, `start-ticket-owner.sh`에 호출 없음
- 결과: worker state의 `active_ticket_id`가 절대 업데이트되지 않아 UI가 stale한 ticket id를 영원히 표시
- 이 PRD는 PRD_253의 미완성 구현을 완전히 대체·교정하는 재발행임

## Execution Candidates

### Candidate 1: runner-stage.js 실구현 + lifecycle 스크립트 통합

**Scope:**
- `.autoflow/scripts/runner-stage.js` 신규 구현 (Node.js, executable)
  - `node runner-stage.js <stage> [--runner <id>] [--ticket <ticket-id>] [--note <text>]` 인터페이스
  - runner state 파일의 `active_stage`, `active_ticket_id`, `active_ticket_title` 갱신
  - `--help` 플래그 지원
- `start-ticket-owner.sh`: ticket claim 직후 `runner-stage.js inprogress` 자동 호출 추가 (실패해도 main 흐름 차단 안 함)
- `finish-ticket-owner.sh`: pass 분기에서 done 이동 직후 `runner-stage.js idle` 호출 추가
- `apps/desktop/src/main.js`: buildInitialPrompt worker 케이스에 "after each phase, also call `node .autoflow/scripts/runner-stage.js <stage>`" 추가
- `.autoflow/agents/ticket-owner-agent.md`, `plan-to-ticket-agent.md`, `wiki-maintainer-agent.md`: 워크플로 1줄 추가
- `runtime/board-scripts/start-ticket-owner.sh`, `runtime/board-scripts/finish-ticket-owner.sh`: 동기화
