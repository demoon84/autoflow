## Core Scope

- Goal: `tickets/inbox/order_146.md`의 Hermes self-learning loop 제안을 Phase 1 기본 인프라로 좁혀, skill 파일 저장 규약, CLI create/match/update-stats, 완료 후 best-effort 추출 trigger를 구현한다.
- In Scope:
  - `.autoflow/wiki/skills/` 카테고리를 신설하고 `README.md`, `skill-template.md`, `.gitkeep`를 둔다. 현재 보드와 `scaffold/board/wiki/skills/` 템플릿을 함께 맞춘다.
  - `packages/cli/skill-project.sh`를 신설해 다음 명령을 제공한다:
    - `autoflow skill create <project-root> <board-dir-name> --from-ticket <tickets_NNN|path>`: 완료 ticket/verify/PRD 근거에서 `skill_NNN.md` 초안을 생성한다.
    - `autoflow skill match <project-root> <board-dir-name> --keywords "<text>" [--limit N]`: frontmatter keywords, applies_to module, title/body keyword overlap 기준으로 후보 skill을 점수와 함께 출력한다.
    - `autoflow skill update-stats <project-root> <board-dir-name> <skill_NNN> --result pass|fail`: `success_count`, `failure_count`, `last_used_at`를 안전하게 갱신한다.
  - `bin/autoflow`에 `skill` subcommand dispatch를 추가하고 CLI README에 사용법을 적는다.
  - `.autoflow/scripts/finish-ticket-owner.sh`와 `runtime/board-scripts/finish-ticket-owner.sh`의 pass 완료 경로에서 `autoflow skill create ... --from-ticket <ticket>`을 best-effort로 호출한다.
  - 자동 추출 실패는 ticket 완료를 막지 않고 warning/log만 남긴다. `AUTOFLOW_SKILL_AUTO_EXTRACT=off`이면 호출하지 않는다.
  - ticket-owner agent와 루트/스캐폴드 AGENTS 문서에 skill 추출은 wiki baseline update가 아니라 별도 learned-skill artifact이며, 실패해도 자율 흐름을 멈추지 않는다는 정책을 추가한다.
- Out of Scope:
  - planner/worker prompt에 skill 본문을 자동 주입하는 RAG injection.
  - reject auto-replan, blocked recovery, orchestration cleanup에서 skill을 자동 추출하는 planner-side trigger.
  - Desktop 사이드바 "스킬" 화면, 사람 편집 UI, 비활성화 토글.
  - semantic embedding 기반 매칭, cross-project skill 공유, LLM 호출 없는 deterministic execution.
  - 기존 `.autoflow/wiki/index.md`나 managed wiki baseline 페이지 재작성.
