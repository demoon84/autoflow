- Status: draft

## Core Scope

- Goal: 데스크톱 작업 흐름 페이지의 Plan AI 카드 단계 표시기가 "loop 가 살아있다 = 계획 진행 중" 으로 잘못 매핑되는 버그를 고친다. start-plan.sh 가 `status=idle` / `reason=no_actionable_plan_input` 을 반환했고 active ticket 도 없는데 단계가 "계획" 으로 켜지면 사용자가 실제 진행 상황을 오인한다.
- In Scope:
  - `apps/desktop/src/renderer/main.tsx` 의 `runnerStageKey()` 안 planner 분기 (`if (role === "planner" || role === "plan")`) 의 조건 정밀화.
  - 새 매핑 규칙:
    - `isFailLike` → `blocked`
    - 최근 출력에 `source=backlog-to-todo` / `source=reject-replan` / `todo_ticket=` 흔적 → `done` (방금 todo 한 건 만들고 끝)
    - `status === "running"` AND (`hasActiveTicket` OR `event=adapter_start` 가 `stateText` 에 보임) → `planning`
    - 그 외 → `idle`
  - 기존 wiki 분기와 동일한 패턴(`status === "running" && (hasActiveTicket || event=adapter_start)`)을 따라 일관성 유지.
  - planner 카드의 첫 단계 라벨 "대기" 가 idle 상태에서 활성으로 보이도록 (현재 정의된 plannerFlowStages 중 `key:"idle"` 가 first stage 라 자연 매칭).
- Out of Scope:
  - ticket-owner / wiki-maintainer 등 다른 role 의 stage 로직 변경.
  - `plannerFlowStages` 자체 라벨/메타 변경.
  - start-plan.sh 의 출력 키 변경.
  - 시각 디자인(색상, 아이콘) 변경.
