## Core Scope

- Goal: 데스크톱 앱의 토큰 사용량 표시가 정확하도록 두 경로(통계 화면 총량 + AI 진행 현황 카드별) 의 집계 source 를 점검하고, 누락된 경로를 채워 (1) 총량 badge 가 실제 사용량을 반영하고 (2) 각 AI runner 카드의 사용량이 일관된 source 에서 표시되도록 한다.
- In Scope:
  - **경로 분석 + 정합화**:
    - 경로 A (per-runner 카드 표시): `apps/desktop/src/main.js` 의 `readRunnerTokenUsage` (line 1564) → `aggregateLiveTokenUsage` (1484) + `.autoflow/metrics/token-cache.tsv` 읽기. 결과는 `runner.tokenUsage` 로 들어가 `apps/desktop/src/renderer/main.tsx:5709-5803` 의 `tokenUsageLabel` 로 카드에 표시.
    - 경로 B (총량 badge): `apps/desktop/src/renderer/main.tsx:483` 의 `tokenUsageCount: statusNumber(metrics, "autoflow_token_usage_count")`. 이 값은 `packages/cli/metrics-project.sh:337-369` 의 `count_autoflow_token_metrics` 가 `.autoflow/metrics/telemetry-runs.jsonl` 에서 `token_input + token_output` 합산해 채움. 표시 위치: `main.tsx:3560,3690,3743` 등 (통계 화면 / 카드 / 미터).
  - **누락 보강 1 — telemetry-runs.jsonl 에 token 정보 주입**:
    - `packages/cli/run-role.sh:499-563` `run_role_record_worker_tick_telemetry` 가 `telemetry record` 를 호출할 때 `--token-input` / `--token-output` 인자를 누락하고 있어 모든 runner 의 telemetry-runs.jsonl 행이 `token_input=0, token_output=0` 로 기록됨 → `autoflow_token_usage_count` 항상 0 ≈ **통계 화면 총량이 0 또는 비현실적으로 낮은 근본 원인**.
    - 어댑터 stdout 에서 token 정보를 추출하는 로직(현재 desktop main.js 만 보유) 을 CLI 측으로 이식하거나 공통 유틸로 분리해 `telemetry record` 호출 직전에 token_input/output 계산 후 주입.
    - 적용 대상: 현재 worker(ticket) 만 telemetry record 호출(line 507 `[ "$public_role" = "ticket" ] || return 0`). **planner / verifier / wiki 도 동일하게 telemetry record 호출하도록 확장**해야 총량이 의미 있어짐.
  - **누락 보강 2 — gemini 어댑터 stdout 토큰 노출 (order_087 와 연동)**:
    - 별도 order_087 에서 다루는 gemini token 출력 추가가 본 PRD 의 (1) per-runner 카드 source 와 (2) telemetry-runs.jsonl source 양쪽 모두에 영향. order_087 가 선행되어야 wiki runner 가 두 경로 모두에 정확한 토큰을 기여.
    - 본 PRD 는 order_087 가 만들어내는 stdout 표준 marker (`total_tokens=N` 등) 를 양 경로가 인식하도록 보장.
  - **UI 표현 점검 + 수정**:
    - 총량 badge (`main.tsx:3560` `<Badge>토큰 사용량</Badge> <strong>{formatCount(tokenUsageCount)}</strong>`) 의 source 와 카드별 합계가 일치하는지 sanity check.
    - `hasTokenData = tokenUsageCount > 0 || tokenReportCount > 0` 분기(line 3546) 가 실제로 token 데이터 유무를 반영하는지 확인. 누락 보강 후 0 이 아닌 값이 정상 표시되어야 함.
    - 각 AI 카드의 `tokenUsageLabel` (line 5711) 이 enabled 모든 runner 에 대해 호출 발생 후 표시되는지 확인. 0 일 때 빈 문자열로 숨기는 현재 동작은 유지.
    - 통계 화면 "사용 토큰" 카드 (line 3634 `{ label: "사용 토큰", value: tokenUsageCount, displayValue: formatCount(tokenUsageCount) }`) 와 다른 표시 위치들의 일관성 확인.
- Out of Scope:

  - 과거 telemetry-runs.jsonl 의 token=0 행 backfill (historical data 보정 안 함).
  - 토큰 비용 ($) 환산 표시 추가 (별도 후속 PRD 에서 다룰 수 있음).
  - 모델별 token 가격 차이 분석 / 시각화.

## Global Acceptance Criteria

- [ ] 통계 화면의 "토큰 사용량" badge 가 enabled runner 들이 LLM 호출 누적 발생한 뒤 0 이 아닌 값으로 표시된다 (현재는 항상 0 또는 매우 낮음).
- [ ] AI 진행 현황 화면의 각 enabled runner 카드 (planner, worker, verifier, wiki) 가 호출이 발생한 뒤 카드 하단의 "N 토큰 사용" 라벨에 실제 값이 표시된다 (단, gemini 사용 runner 는 order_087 완료 후에 정확).
- [ ] 통계 화면 총량 ≈ 각 AI 카드 토큰 사용량의 합 (±10% 오차 허용 — source 가 다르므로 완전 일치는 아닐 수 있음. sanity check 수준).
- [ ] `.autoflow/metrics/telemetry-runs.jsonl` 의 신규 행이 `token_input` / `token_output` 가 0 이 아닌 값으로 기록된다 (claude/codex 어댑터를 사용하는 worker / planner / verifier 부터 우선). gemini 는 order_087 완료 후.
- [ ] `packages/cli/run-role.sh` 의 telemetry record 호출이 worker(ticket) 뿐 아니라 planner / verifier / wiki 도 포함하도록 확장된다 (role gating 제거 또는 명시적 허용 리스트).
- [ ] `npm run desktop:check` 가 통과한다 (TypeScript 미사용 식별자 / 빌드 에러 없음).
- [ ] 토큰 데이터가 전혀 없는 신규 보드(첫 실행 직후) 에서 "토큰 사용량" badge 와 카드 라벨이 적절한 fallback (예: 0 또는 "—") 로 표시되고 ReferenceError / NaN 없음.
- [ ] 기존에 동작하던 (claude / codex) 어댑터 토큰 추적이 회귀 없이 유지된다.
