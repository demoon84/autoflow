# PRD: 작업 흐름 핀 레이어 목록 정렬 깨짐 수정

- ID: prd_114
- Title: 작업 흐름 핀 레이어 목록 정렬 깨짐 수정
- Status: backlog
- Created: 2026-05-02
- Source: memo_074

## Goal

작업 흐름 핀 레이어(Workflow Pin Layer)의 목록 UI에서 발생하는 그리드 정렬 깨짐 현상을 수정하고, 동일한 ID의 메모가 중복 노출되지 않도록 개선한다.

## Context

핀 레이어 목록에서 상태 배지가 있는 행과 없는 행 간에 그리드 컬럼 위치가 어긋나 날짜 컬럼이 정렬되지 않는 문제가 있다. 또한, 동일한 번호의 메모가 inbox와 done 폴더 양쪽에 존재할 경우 목록에 중복으로 표시되어 사용자 혼란을 야기한다.

## Done When

- [ ] `.workflow-pin-item` 그리드 레이아웃이 배지 유무와 관계없이 고정된 4개 컬럼 슬롯(`ID | 제목 | 배지 | 날짜`)을 유지함.
- [ ] `apps/desktop/src/renderer/main.tsx`의 `memoFiles` 리스트 생성 시, 동일한 파일명(ID)을 가진 메모가 중복 노출되지 않도록 중복 제거(dedupe) 처리됨 (inbox 우선).
- [ ] 핀 레이어 목록에서 날짜 컬럼이 모든 행에서 세로로 일직선 정렬됨.
- [ ] `npm run desktop:check` 통과.

## Allowed Paths

- `apps/desktop/src/renderer/main.tsx`
- `apps/desktop/src/renderer/styles.css`

## Verification

- ORDER 핀 레이어를 열어 배지가 있는 항목과 없는 항목의 날짜 컬럼이 정렬되는지 확인.
- 동일한 `memo_NNN`이 inbox와 done에 모두 있을 때 한 번만 표시되는지 확인.
- 좁은 폭에서도 레이아웃이 깨지지 않는지 확인.
