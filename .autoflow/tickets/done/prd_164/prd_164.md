## Core Scope

- Goal: Phase 1~3 의 skill 인프라/curator/활용을 데스크톱 앱에서 사람이 직접 보고 편집/관리할 수 있는 UI 추가.
- In Scope:
  - **사이드바 메뉴 추가**:
    - `apps/desktop/src/renderer/main.tsx` 의 `settingsNavigation` 에 `{ key: "skills", label: "스킬", icon: ... }` 추가 (위치: "LLM 위키" 메뉴 바로 아래 또는 "통계" 위 — Plan AI 결정).
    - `visibleSettingsSection === "skills"` 분기 신설.
  - **Skill 목록 + 상세 페이지**:
    - 좌측: skill 카드 목록 (in-repo / agent-created / archived 카테고리 분리, 탭 또는 필터)
      - 각 카드: name + description + pattern_type 뱃지 + 활용 통계 (success_count, last_used_at, success_rate)
      - 정렬: success_rate ↓ / last_used_at ↓ / 알파벳
      - 검색: keyword / pattern_type 필터
    - 우측: 선택된 skill 본문 미리보기 (markdown render)
      - frontmatter 메타 (name, description, applies_to 등) 가독 표시
      - 본문 (`Trigger`, `Recommended Procedure`, `Pitfalls`, `Verification Pattern`, `Source Evidence`)
      - 활용 통계 그래프 (mini sparkline — 최근 30일 활용 빈도)
  - **사람 액션 토글**:
    - **Pin** 토글 — frontmatter `pinned: true` 변경. lifecycle 자동 transition 우회.
    - **Archive** 버튼 — `.archive/` 이동 (수동 archive). 복구 가능.
    - **Restore** 버튼 — `.archive/` 에서 복구 (archived skill 만 표시).
    - **Edit** 버튼 — 본문 편집 (markdown editor 또는 외부 editor 호출).
    - **Disable** 토글 — 일시 비활성 (매칭에서 제외, 본문 보존). frontmatter `disabled: true`.
  - **활용 통계 페이지** (선택):
    - 전체 활용 횟수, top-3 skill, low_confidence 개수 카운터.
    - 시간별 활용 추세 (mini chart).
    - 통계 페이지 (PRD-129 / order_129) 와 연계해 통계 화면에도 카드 1개 추가.
  - **빈 상태 안내**:
    - skill 0건일 때 "PRD 완료 / reject 회복 / blocked 회복 후 자동 생성됩니다" 가이드.
    - skill 활용 0건일 때 "다음 PRD 처리 시 매칭되면 자동 표시됩니다" 가이드.
- Out of Scope:
  - Skill 자체 자동 생성 / 매칭 / curator (Phase 1~3 책임)
  - Security scan UI (Phase 5)
  - agentskills.io 호환 UI (Phase 5)
  - 인라인 shell editor (보안 위험, 보류)
