# Project

- ID: prd_153
- Title: PROMPT_BYTES cap 확장 — planner/worker/verifier 도 wiki 와 동일 정책 적용
- AI: planner/worker
- Status: draft

## Core Scope

- Goal: 현재 wiki 만 적용된 prompt input byte cap 정책을 planner/worker/verifier 로 확장해 비정상 큰 prompt 의 토큰 소비를 안전 범위로 제한 (10-15% 절감).
- In Scope:
  - 환경변수 신설:
    - `AUTOFLOW_PLANNER_PROMPT_BYTES` (기본 64KB) — planner 보드 컨텍스트 cap
    - `AUTOFLOW_WORKER_PROMPT_BYTES` (기본 96KB) — worker PRD/ticket 본문 cap
    - `AUTOFLOW_VERIFIER_PROMPT_BYTES` (기본 32KB) — verifier 검증 컨텍스트 cap
  - `packages/cli/run-role.sh` 의 prompt 생성 단계에서 cap 적용:
    - cap 초과 시 head/tail 스플릿 (앞 60% + 뒤 40% — 핵심 instruction 과 최신 변경 보존), 중간 elide marker `[... N bytes elided to save tokens ...]` 삽입.
    - wiki 의 기존 cap 헬퍼 (`packages/cli/wiki-project.sh:1330` 부근) 패턴 재사용.
  - 데스크톱 표시 (선택): cap 적용된 호출에 미세 표시 (디버그 only, 사용자 미노출).
- Out of Scope:
  - cap 자체 동적 결정 (학습 기반) — 후속 PRD.
  - output cap (별도 PRD-154).
  - prompt 내용 차분/요약 (별도 PRD-159).
