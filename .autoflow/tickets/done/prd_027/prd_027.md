# Project PRD

## Project

- ID: prd_027
- Title: Replace the desktop design kit from shadcn/Radix/Tailwind to MUI
- AI: Codex
- Status: draft

## Core Scope

- Goal: Replace the Autoflow desktop app design-kit foundation from shadcn-compatible Radix/Tailwind wrappers to MUI while preserving the current desktop app behavior, Korean user-facing copy, and board-reading workflows.
- In Scope:
  - Update project guidance so desktop UI work prefers MUI instead of shadcn/ui.
  - Add MUI dependencies and Emotion styling dependencies to the desktop app.
  - Introduce a desktop MUI theme with existing Autoflow colors, typography, radius, and density.
  - Wrap the React renderer with MUI `ThemeProvider` and `CssBaseline`.
  - Replace local shadcn-style UI wrappers used by the renderer with MUI equivalents.
  - Remove shadcn-specific configuration and unused Radix/Tailwind/CVA utility dependencies after migration.
  - Keep the desktop app as a local Autoflow board viewer/control surface; do not change ticket lifecycle semantics.
- Out of Scope:
  - Rewriting CLI, runner, ticket owner, wiki, or verifier runtime behavior.
  - Adding new product features unrelated to the UI design-kit migration.
  - Changing the user-facing Korean language policy.
  - Repositioning the desktop app or redesigning the whole IA beyond what is required for component parity.
  - `git push`.
