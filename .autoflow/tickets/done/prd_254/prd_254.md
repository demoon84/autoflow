## Goal

자주 호출되는 소형 board 무결성 스크립트 5개를 Node.js로 변환한다. 기존 `.sh`는 thin wrapper로 유지해 bash CLI 호환을 보장하고, 데스크톱 main process에서 `require`로 직접 호출할 수 있는 구조를 만든다.

## Background

Autoflow는 이미 Node.js(Electron + autoflow CLI) 환경에서 동작하며, 새로 만드는 자동화(`runner-stage.sh` → `runner-stage.js` 등)도 `.js`로 방향을 잡았다. 기존 `.sh`를 점진 통일하면 공통 유틸(state file parser, ticket markdown parser) 재사용, subprocess fork 회피, cross-platform 호환 이점을 얻는다.

## Scope

다음 5개 스크립트의 Node.js 버전 작성:
- `lint-ticket.js` (`lint-ticket.sh` 대응, ~189라인)
- `board-guard.js` (`board-guard.sh` 대응, ~474라인)
- `integrate-worktree.js` (`integrate-worktree.sh` 대응, ~107라인)
- `state-db.js` (`state-db.sh` 대응, ~219라인)
- `path-conflict-check.js` (`path-conflict-check.sh` 대응)

공통 유틸 모듈도 함께 추출:
- `boardState.js` (state file read/write)
- `ticketMd.js` (ticket markdown parser)
- `runnerLog.js` (event log writer)

## Acceptance Criteria (Global)

- 5개 `.js` 파일이 `runtime/board-scripts/` 및 `.autoflow/scripts/`에 존재
- 각 `.js`는 `node <script>.js` 및 `require('<script>')` 두 방식으로 동작
- 기존 `.sh`는 `node <script>.js "$@"` thin wrapper로 유지 (deprecation warning 포함)
- `node --check` 구문 검사 통과
- `autoflow ...` 사용자 CLI 인터페이스 변경 없음

## Execution Candidates
