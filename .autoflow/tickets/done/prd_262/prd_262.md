## Problem

현재 fs.watch 기반 wake 시스템은 main process가 tickets 폴더 변경을 감지하면 `[wake] <path>` 텍스트를 PTY stdin에 주입한다. LLM이 paste 처리 중이거나 mid-thinking 중이면 wake 메시지가 흡수/오인식되고, 정상 도달 여부 검증도 불가하다.

## Goal

push 방식(텍스트 주입)과 polling 방식(JSONL 큐 파일)을 hybrid로 운영한다. main process는 fs.watch 결과를 큐 파일에도 기록하고, LLM은 매 turn 끝/startup scan 시 `runner-wake.js poll`로 누락 없이 wake 이벤트를 수집한다. poll이 truth source, 텍스트 주입은 보강 신호.

## Scope

### runner-wake.js 3개 subcommand

- **emit**: 큐 파일에 이벤트 1줄 append (main process / board 내부 호출)
- **poll**: 마지막 poll 이후 이벤트 반환 + `last_polled_at` 갱신
- **notify**: 다른 runner 큐에 wake event 푸시 (runner간 즉시 깨우기)

큐 파일 위치: `runners/state/<runner-id>-wake.queue.jsonl` (gitignored)
큐 size: 기본 200 events, FIFO trim

### main.js 통합

`ensureBoardWatcher.broadcast` 가 `[wake]` 텍스트 주입 외에 `runner-wake.js emit`도 호출 (hybrid).

### agent.md / buildInitialPrompt 명시

startup scan 직전 + 매 turn 끝에 poll 호출하도록 agent.md 3개와 worker prompt에 1줄 추가.

### finish-ticket-owner 연동

worker가 ticket pass 직후 `runner-wake.js notify --target wiki`로 wiki 즉시 알림 (옵션).
