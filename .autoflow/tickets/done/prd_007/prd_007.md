# Project Spec

## Meta

- PRD Key: prd_007
- Title: AI workflow card — single-line agent meta + worker id + 공정률 percent
- Status: populated

## Goal

데스크톱 "작업 흐름" 의 AI 카드 좌측 메타 영역을 현재의 3-라인 구조 (1) `codex` (2) `gpt-5.4 · medium` (3) `worker-3 · AI` 에서 더 직관적인 형태로 바꾼다.

새 형식 (위→아래 3라인):

1. `Codex gpt-5.4 medium` — agent 이름(첫 글자 대문자) · model · reasoning 을 공백으로 결합. 라벨·구분자 없음.
2. `worker-3` — worker id 만 (역할 라벨 제거).
3. `공정률: N%` — 현재 stage 기반 진행 백분율.

추가 라벨이나 구분자(`·`, `/`, `:`) 없이 시각 잡음을 줄이고, 사용자 시선이 단계 트랙·진행 백분율로 자연스럽게 흐르도록 한다.

## Core Scope

### In Scope

- `AiProgressRow` (`apps/desktop/src/renderer/main.tsx`) 의 `<div className="ai-progress-agent">` 블록 재구성:
  - Line 1: 공백 join `<Agent> <model> <reasoning>` 형태. agent 이름은 첫 글자 대문자 (예: codex → "Codex", claude → "Claude", gemini → "Gemini", opencode → "Opencode", shell → "Shell"). 빈 항목은 자동 생략 — reasoning 없음이면 `Codex gpt-5.4`, model 도 없으면 `Codex`.
  - Line 2: `displayWorkflowRunnerId(runner.id)` 결과만 (예: `worker-3`).
  - Line 3: `공정률: ${percent}%` — `progressRatio * 100` 을 정수 반올림.
  - 라인 사이 `· / -` 같은 구분자 없음, 단순 줄바꿈 + 작은 간격.
- 진행률 계산은 기존 `ownerFlowStages` 인덱스 기반 (`Math.round(progressRatio * 100)`).
  - reject 단계: `공정률: 거절` 로 표기. percent 가 헷갈리지 않게 한 가지 표현으로 통일.
  - idle / 단계 0: `공정률: 0%`.
- agent 이름 capitalize 헬퍼: 첫 글자만 대문자로 만드는 작은 유틸 함수 (예: `capitalizeAgentName`) 도입 또는 inline. 빈 문자열은 그대로 빈 값.
- CSS 클래스 정리:
  - 기존 `.ai-progress-agent strong / p / span` 스타일을 새 라인 구조에 맞게 보정 (글꼴/간격/정렬). 카드 전체 가로폭과 단계 트랙 정렬은 회귀 없게.
  - 신규 클래스 `.ai-progress-agent-meta` (Line 1), `.ai-progress-agent-id` (Line 2), `.ai-progress-agent-rate` (Line 3) 도입 또는 기존 요소 재라벨.
- accessibility: aria-label 은 단계 트랙 쪽이 그대로 유지, agent 영역은 의미 변하지 않으므로 별도 추가 불필요.

### Out of Scope

- 보드 전체 완료율 (`completion_rate_percent` 메트릭) 표기 — 이번엔 per-runner 단계 기반 percent 만.
- 단계 트랙 / 진행 표시 / 카드 본체 우측 stage 라벨 등 다른 카드 요소 변경.
- AI 관리 메뉴의 runner 표시 변경.
- 진행률 percent 의 단위·언어 다국어 (한국어 "공정률" 고정).
- 진행률 계산 알고리즘 변경 (예: 시간 기반·티켓 가중치 등). 기존 stage 인덱스 그대로.
- worker id 의 prefix 변환 규칙 변경 (`displayWorkflowRunnerId` 그대로 사용).
