## Core Scope

- Goal: `apps/desktop/src/renderer/styles.css` 안에서 `.ai-progress-board` 그리드를 항상 `repeat(3, minmax(0, 1fr))` 로 유지하고, AI Autoflow 탭에 영향을 주는 미디어쿼리 분기(2열/1열로 떨어뜨리는 4행 분기)를 제거한다.
- In Scope:
  - `@media (max-width: 1279px)` 분기(line ~4009) 안의 `.ai-progress-board { grid-template-columns: repeat(2, ...) }` 규칙 제거.
  - `@media (max-width: 980px)` 분기(line ~6366) 안의 `.ai-progress-board { grid-template-columns: repeat(2, ...) }` 규칙 제거.
  - `@media (max-width: 720px)` 분기(line ~6406) 안의 `.ai-progress-board { grid-template-columns: minmax(0, 1fr) }` 규칙 제거.
  - 위 분기에 함께 들어 있던 `.ai-progress-board` 단일 셀렉터 override 만 정리한다. 같은 미디어쿼리 안의 다른 셀렉터(`.workspace-layout`, `.settings-page`, `.metrics-strip`, `.ai-progress-row-top`, `.ai-progress-actions`, `.ai-progress-config*`, `.ai-progress-current*`, `.ai-progress-row-worker`, `.ai-progress-track`, `.runner-grid`, `.settings-content .runner-grid`, `.settings-nav*`, `.settings-content*` 등)는 절대 건드리지 않는다.
  - 베이스 규칙 `.ai-progress-board { grid-template-columns: repeat(3, minmax(0, 1fr)); }` (line 2383 부근)은 그대로 유지하고 필요한 경우 `min-width` 만 그대로 둔다. 가로 공간 부족 시 카드가 깨지지 않게 컨테이너에 가로 스크롤이 자연스럽게 살도록 그대로 둔다(`overflow` 변경 금지).
- Out of Scope:
  - Non-AI Autoflow 그리드(`.runner-grid`, `.settings-content .runner-grid`, `.metrics-strip`, ticket kanban 등) 의 반응형 동작 변경.
  - `apps/desktop/src/renderer/main.tsx` 또는 React 컴포넌트 구조 변경.
  - 다른 탭 레이아웃, 사이드바, 다이얼로그, 카드 내부 typography 조정.
  - `.ai-progress-board` 베이스 규칙의 grid-template-columns 값 자체 변경(이미 3열로 고정돼 있음).
  - 새로운 미디어쿼리 추가 또는 viewport 단위(`vw`, `clamp`) 도입.
  - `git push`, runner process 관리.
