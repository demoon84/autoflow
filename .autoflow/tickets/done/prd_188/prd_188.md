- In Scope:
  - `apps/desktop/src/renderer/main.tsx` 의 `runnerHeartbeatStale` 단일 boolean 판단을 단계형 helper 로 바꾸거나 보강한다.
  - freshness 기준은 `lastEventAt` 과 `lastAdapterChunkAt` 중 최신 시각을 사용하고, PRD 178 의 `activeStage="adapter_running"` 계약을 유지한다.

  - 진행 중인 LLM 호출 또는 최근 freshness 가 있는 runner 는 destructive `응답 지연` 대신 정보성 `LLM 응답 대기 중` 또는 라벨 없음으로 표시한다.
  - 오래 걸리는 호출은 `LLM 응답 대기 중`, `응답 지연 의심`, `멈춤 가능`처럼 심각도를 분리하고, 각 단계의 시각 톤과 tooltip 을 구분한다.
  - tooltip 은 단계 의미, 경과 시간, timeout 1200초 기준에서 얼마나 가까운지, 정상 worker/planner 호출이 5~10분 걸릴 수 있다는 점을 설명한다.
- Out of Scope:
  - adapter stdout/stderr streaming chunk 수집, `last_adapter_chunk_at` 기록, `active_stage=adapter_running` runtime 기록 재구현. 해당 범위는 `prd_178` / `tickets_177` 에서 완료됐다.
  - runner timeout watchdog, process-tree cleanup, stdin preservation, runner start/stop/restart 정책 변경.
  - self-monitoring monitor stuck 정책(`prd_185`) 변경.
  - 데스크톱 progress card 의 레이아웃 대개편 또는 ticket dialog 동작 변경.
