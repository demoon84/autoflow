# PRD 218 — 통계 탭 처리 시간 카드 추가

## Goal

`autoflow metrics` 가 ticket lead/active 시간과 24h 누적 duration 을 새 key=value 로 emit 하도록 `packages/cli/metrics-project.sh` 를 확장하고, 데스크톱 통계 탭의 strip 영역에 "처리 시간" 카드를 변경 코드량 / 토큰 사용량 카드 옆 세 번째 칸으로 추가한다.

## Background

- order_191. 사용자 요청: 통계에 처리 시간 정보가 없어 추가하고 싶음. 토큰 사용량 카드 오른쪽에 카드 1개 추가.
- 데이터 소스는 이미 존재한다 (새 IPC / 스토리지 추가 불필요):
  - `state.db` 의 `ticket_lifecycle` 테이블에 `lead_seconds` / `active_seconds` / `tick_count` 컬럼이 있고, `autoflow origin status` 의 `=== Average ticket lead time by status ===` 출력이 같은 SQL 을 이미 사용 중이다.
  - `state.db` 가 없는 환경 fallback: `.autoflow/telemetry/runs.jsonl` 의 `duration_ms` 합산 + 각 ticket 의 `## Goal Runtime > Time Used Seconds` 합산.
- 현재 strip 은 `repeat(auto-fit, minmax(152px, 1fr))` 자동 그리드라 카드 1개 추가만으로 자동 3칸 배치된다. styles.css 변경 없이 끝나야 한다.

## Scope

### 1. `packages/cli/metrics-project.sh` — 신규 key=value emit

- 신규 필드 (4개):
  - `autoflow_avg_lead_seconds` — done ticket 평균 lead time (생성→완료, ticket_lifecycle.lead_seconds 평균).
  - `autoflow_avg_active_seconds` — done ticket 평균 active time (worker 실작업, ticket_lifecycle.active_seconds 평균).
  - `autoflow_avg_ticks_per_done_ticket` — done ticket 평균 tick count (ticket_lifecycle.tick_count 평균).
  - `autoflow_duration_total_24h_seconds` — 최근 24h `runs.jsonl` 의 `duration_ms` 합 (초 단위 정수).
- 우선순위: state.db 가 있으면 ticket_lifecycle 에서 집계. 없거나 row 가 0개면 jsonl + ticket markdown fallback.
- 기존 write-time sanity cap (rule 19a 동일 정신) 적용. 이상치 / NaN 은 0 으로 emit.

### 2. `apps/desktop/src/renderer/vite-env.d.ts` — 타입 확장

- `AutoflowMetricSnapshot` 인터페이스에 위 4 필드를 `number` 로 추가.

### 3. `apps/desktop/src/renderer/main.tsx` — UI 카드

- `apps/desktop/src/renderer/main.tsx` 의 strip 영역 (현재 변경 코드량 / 토큰 사용량 카드가 렌더링되는 곳, 약 line 4388~4396 근처) 에 세 번째 카드 "처리 시간" 추가.
- 본문:
  - 큰 숫자 = 평균 active time. 60초 미만은 그대로 초 표기, 60분 미만은 분 1자리 소수 (`14.3분`), 그 이상은 시간 1자리 소수 (`2.4h`).
  - 보조 라인: `lead 16.0분 / 누적 24h 47.2h` 형식. 동일한 단위 자동 포맷 적용.
  - title attribute (hover tooltip) 에 raw seconds 와 sample 수 (예: `n=184`, `lead=960s`, `active=860s`, `24h=170000s`) 정보 노출.
- 기존 변경 코드량 / 토큰 사용량 카드의 layout / 정렬 / 라벨 / class 는 변동 없음.

## Out of Scope

- 새 IPC / preload 추가, 새 storage 도입.
- styles.css 그리드 / strip 레이아웃 변경 (auto-fit 으로 자동 배치되어 불필요).
- `autoflow origin sync` 자동 트리거 변경.
- 토큰 사용량 / 변경 코드량 카드 동작 변경.
