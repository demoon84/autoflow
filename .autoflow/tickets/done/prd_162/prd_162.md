## Core Scope

- Goal: Hermes 패턴의 핵심 인프라를 Autoflow 에 도입. 저장소 분리(in-repo vs agent-local), 폴더 단위 skill, sidecar 통계, validator/cap, CLI 기본 명령. Phase 2~5 의 토대.
- In Scope:
  - **저장소 이중화**:
    - `.autoflow/wiki/skills/<category>/<name>/SKILL.md` — 사람 작성 + 검토 (배포본). Lifecycle 자동 archive 대상 아님.
    - `.autoflow/wiki/skills-local/<category>/<name>/SKILL.md` — agent-created (자동 추출, lifecycle 대상).
    - `.autoflow/wiki/skills-local/.archive/<category>/<name>/` — archive 대상 (삭제 안 함, 복구 가능).
  - **폴더 단위 skill**:
    - 단일 .md 가 아닌 `<name>/SKILL.md` + 부속 자료 (예시 코드, snippet, reference).
    - 부속 파일 cap 1MiB.
  - **frontmatter 표준**:
    ```yaml
    ---
    name: <lowercase-hyphen, ≤64 chars>
    description: <≤1024 chars>
    pattern_type: <ticket_completion | reject_turnaround | blocked_recovery | orchestration_cleanup>
    applies_to:
      module: <path or keywords>
      keywords: [list]
    pinned: false
    created_from:
      prd: <PRD_NNN | null>
      ticket: <ticket_NNN | null>
    created_at: <ISO>
    ---
    ```
  - **Validator + Size cap**:
    - Hermes 패턴 채택: name ≤64 chars, description ≤1024 chars, content ≤100KB (≈36K tokens), file ≤1MiB.
    - YAML 파싱 실패 시 거부, non-empty body 강제.
  - **`.usage.json` sidecar**:
    - 위치: `.autoflow/wiki/skills-local/.usage.json`
    - 구조: `{ "<category>/<name>": { view_count, last_viewed_at, last_used_at, success_count, failure_count } }`
    - Atomic write (tempfile + os.replace).
    - 갱신 실패는 silent (best-effort, 흐름 안 막음).
  - **CLI 기본 명령** (`packages/cli/skill-project.sh` 신설):
    - `autoflow skill list <project-root> <board-dir-name>` — 모든 skill 목록 + 통계
    - `autoflow skill view <project-root> <board-dir-name> <name>` — skill 본문 표시 (preprocessing 포함)
    - `autoflow skill create <project-root> <board-dir-name> --from-ticket <ticket_NNN>` — ticket 본문에서 패턴 추출 + skill 생성
    - `autoflow skill validate <project-root> <board-dir-name> <name>` — frontmatter / size cap 검증
    - `bin/autoflow` dispatch 추가
- Out of Scope:
  - Curator (Phase 2)
  - RAG injection / 활용 (Phase 3)
  - 데스크톱 UI (Phase 4)
  - Security scan / agentskills.io 호환 (Phase 5)
  - 인라인 shell 도입 (보안 위험, 별도 결정)

## Global Rules

- 모든 skill 파일은 frontmatter validator + size cap 통과 필수.
- skill 파일 / sidecar 갱신은 atomic write.
- `.archive/` 는 절대 삭제 안 함 (복구 가능).
- Pinned skill (frontmatter `pinned: true`) 은 모든 자동 transition 우회.
- in-repo (`skills/`) 는 lifecycle 자동 archive 대상 아님.
