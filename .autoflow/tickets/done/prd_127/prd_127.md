# Project PRD

## Project

- ID: prd_127
- Name: telemetry duration_ms 단위 보정
- Title: runner telemetry duration_ms 를 seconds 가 아닌 milliseconds 로 기록
- Goal: `packages/cli/run-role.sh` 의 runner telemetry 기록에서 `duration_ms` 가 epoch seconds 차이를 그대로 저장하는 버그를 고쳐 `.autoflow/telemetry/runs.jsonl` 과 runner timing 집계가 milliseconds 단위 값을 갖게 한다.
- AI: planner
- Status: generated

## Core Scope

- Goal: `run_role_record_worker_tick_telemetry()` 내부의 duration 계산만 좁게 수정해 `telemetry_timestamp_to_epoch()` 가 반환하는 seconds 단위 차이를 `duration_ms` 필드에 저장하기 전 1000배로 변환한다.
- In Scope:
  - `packages/cli/run-role.sh` 에서 `duration_ms=$((end_epoch - start_epoch))` 형태의 seconds 저장을 `duration_ms=$(((end_epoch - start_epoch) * 1000))` 또는 동등한 milliseconds 계산으로 바꾼다.
  - `end_epoch >= start_epoch` 인 경우에만 기존처럼 duration 을 기록하고, timestamp parse 실패나 역전 상태에서는 `duration_ms=0` fallback 을 유지한다.
  - `.autoflow/telemetry/runs.jsonl` 의 신규 runner telemetry 라인이 분 단위 작업에 대해 `minutes * 60 * 1000` 수준의 값을 기록할 수 있게 한다.
  - `packages/cli/wiki-project.sh` 의 `p50_duration_ms` / `p95_duration_ms` / `p99_duration_ms` 집계 헤더와 `autoflow metrics`의 milliseconds 의미를 유지한다.
- Out of Scope:
  - `duration_ms` 필드를 `duration_seconds` 로 rename 하는 schema 변경.
  - `.autoflow/telemetry/runs.jsonl` 과거 라인 backfill.
  - worker 외 planner / wiki / verifier tick telemetry 기록 확대. 해당 범위는 `tickets/inbox/order_122.md` 의 별도 order 로 남긴다.
  - Gemini token usage, `run_with_timeout` stdin 전달, desktop dashboard 레이아웃 변경.
