## Goal

PTY 모드 runner의 worker 카드에서 "실행 중" 배지와 진행 슬라이더가 inprogress ticket 정보를 표시하지 못하는 회귀를 영구적으로 수정한다.

legacy `run-role.sh`가 `active_ticket_id` / `active_ticket_title` / `active_stage`를 state 파일에 직접 썼던 반면, PTY 경로는 이 단계가 없어 UI에 빈 값이 전달된다. `enrichRunnerActiveTicketFromFs` 함수를 `apps/desktop/src/main.js`의 `listRunners` 흐름에 안정적으로 정착시키고, 이후 다른 worker 편집과의 충돌로 인한 재회귀를 방지한다.

## Background

- PTY 모드 실행 시 runner state 파일에 `active_ticket_id` 등의 필드가 기록되지 않아 UI 배지/슬라이더가 공백으로 표시됨
- `enrichRunnerActiveTicketFromFs` 함수가 working tree에 존재하나 stable commit이 없어 다른 worker 편집 시 revert될 위험이 있음
- `apps/desktop/src/main.js`의 `parseRunnerListOutput → enrichRunnerTerminalPreviews` 직후 호출이 인라인 구현된 상태이지만 commit이 누락됨

## Scope

- `enrichRunnerActiveTicketFromFs` 함수의 stable 구현 및 commit 확정
- ticket-owner(PTY) → inprogress/Todo-*.md에서 activeTicketId/activeTicketTitle 파생
- planner(PTY) → inbox/order_*.md 또는 backlog/prd_*.md에서 activeTicketId 파생
- legacy state 파일의 active_ticket_id 값이 이미 있으면 덮어쓰지 않음 (호환)
- 함수 위치: `enrichRunnerTerminalPreviews` 정의 직후 → 향후 편집 충돌 최소화

## Acceptance Criteria (Global)

- PTY 모드 ticket-owner runner가 inprogress 티켓을 처리 중일 때 UI 배지에 ticket ID + Title이 표시됨
- PTY 모드 planner runner가 inbox/backlog에 항목이 있을 때 UI에 해당 항목 ID가 표시됨
- state 파일에 active_ticket_id가 이미 설정된 legacy runner에서는 값이 보존됨
- main.js에 `enrichRunnerActiveTicketFromFs` 함수 정의가 `enrichRunnerTerminalPreviews` 직후에 위치

## Execution Candidates

### Candidate 1: enrichRunnerActiveTicketFromFs stable commit

- Scope: `apps/desktop/src/main.js` 의 `enrichRunnerActiveTicketFromFs` 구현 확인·수정·commit
- Allowed Paths: `apps/desktop/src/main.js`, `apps/desktop/src/renderer/main.tsx`
- Done When:
  - `enrichRunnerActiveTicketFromFs` 함수가 `main.js`에 정의되어 있고 `enrichRunnerTerminalPreviews` 직후 위치에 있음
  - ticket-owner PTY runner가 inprogress Todo-*.md에서 activeTicketId/activeTicketTitle을 파생함
  - planner PTY runner가 inbox order_*.md 또는 backlog prd_*.md에서 activeTicketId를 파생함
  - activeStage가 ticket-owner는 `inprogress`, planner는 `planning`으로 설정됨
  - legacy active_ticket_id가 이미 있으면 덮어쓰지 않음
  - `rg -n "enrichRunnerActiveTicketFromFs|active_ticket_id" apps/desktop/src/main.js` 결과에 함수 정의와 호출 양쪽이 나타남
- Verification: `rg -n "enrichRunnerActiveTicketFromFs|active_ticket_id" apps/desktop/src/main.js`
