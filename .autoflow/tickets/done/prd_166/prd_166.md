## Core Scope

- Goal: Phase 1 인프라 (PRD-162) 위에 Curator (자율 lifecycle 관리) 와 Auto-extraction (이벤트 기반 skill 자동 생성) 도입. 사람 개입 없이 skill 누적 + 정리.
- In Scope:
  - **Curator (Wiki AI 가 owner)**:
    - 7일 주기 + Wiki AI idle 시점 trigger.
    - agent-created skills (`skills-local/`) 만 review 대상 (in-repo `skills/` 는 우회).
    - Lifecycle:
      - 30일 unused → `state: stale` 마킹 (frontmatter 또는 sidecar)
      - 90일 unused → `.archive/` 이동 (절대 삭제 안 함)
      - Pinned 은 모든 transition 우회
    - **결정적 invariant**: "auxiliary client; never touches the main session's prompt cache" — main planner/worker 의 prompt caching 효과 보존. 별도 LLM client instance 사용.
    - 환경변수:
      - `AUTOFLOW_CURATOR_INTERVAL_HOURS` (기본 168 = 7일)
      - `AUTOFLOW_CURATOR_STALE_AFTER_DAYS` (기본 30)
      - `AUTOFLOW_CURATOR_ARCHIVE_AFTER_DAYS` (기본 90)
      - `AUTOFLOW_CURATOR_ENABLED` (기본 1, 0 으로 끔)
    - State: `.autoflow/runners/state/wiki.curator.state` — last_run_at, run_count, paused, last_summary.
  - **Auto-extraction trigger 4개**:
    1. **PRD 완료 + verify pass** (worker 의 `finish-ticket-owner.sh` 직후)
       - pattern_type: `ticket_completion`
       - 추출 source: 완료된 PRD 본문 + ticket 변경 diff + verify 결과
    2. **Reject auto-replan 후 verify pass** (planner 의 `start-plan.sh` 안)
       - pattern_type: `reject_turnaround`
       - 추출 source: 원 reject 사유 + 재계획 후 ticket 변경
    3. **Blocked → 사람 개입 → 풀림** (planner 의 blocked-recovery 감지 시)
       - pattern_type: `blocked_recovery`
       - 추출 source: blocked 시점의 dirty paths + 사람 commit + 재진입 ticket
       - **가장 학습 가치 큼 — 사람만이 풀 수 있는 회복 패턴**
    4. **Orchestration cleanup commit** (planner 의 blocked-dirty-orchestration 직후)
       - pattern_type: `orchestration_cleanup`
       - 추출 source: dirty paths grouping 결정 + commit message
  - **Nudge mechanism (자기 환기)**:
    - planner / worker tick 안에 N tick 마다 또는 PRD 완료 직후 self-prompt: "지금까지 본 패턴 중 skill 화 할만한 것 있나?"
    - flag `skill_extraction_in_progress` 로 재귀 방지.
    - 환경변수: `AUTOFLOW_SKILL_NUDGE_INTERVAL_TICKS` (기본 10).
  - **CLI 보강**:
    - `autoflow skill curator-run --once` — 즉시 curator 실행 (디버그용)
    - `autoflow skill curator-status` — last_run_at / paused / 통계
    - `autoflow skill auto-extract --from-ticket <ticket_NNN> --pattern-type <type>` — 명시적 trigger
- Out of Scope:
  - RAG injection / prompt 주입 (Phase 3, PRD-163)
  - 데스크톱 UI (Phase 4, PRD-164)
  - Security scan (Phase 5, PRD-165)
  - skill clustering / meta-skill (Phase 5, PRD-165)
