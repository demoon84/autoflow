# Project

- ID: prd_155
- Title: Idle tick LLM skip 강화 — 입력 매니페스트 hash 기반 unchanged 시 LLM 호출 생략
- AI: planner/worker
- Status: draft

## Core Scope

- Goal: planner/worker/verifier 의 idle tick (입력이 변하지 않은 tick) 에서 LLM 호출 자체를 생략해 토큰 10-20% 절감. wiki 가 이미 적용 중인 패턴 (`wiki_inputs_unchanged`) 을 다른 runner 로 확장.
- In Scope:
  - 입력 매니페스트 hash 계산:
    - **planner**: `tickets/inbox/`, `tickets/backlog/`, `tickets/reject/`, `tickets/inprogress/` 의 파일 hash sum.
    - **worker**: 자기 worktree base + `tickets/todo/`, `tickets/inprogress/<자기 ticket>` hash.
    - **verifier**: 자기 worktree + `tickets/verifier/` (또는 inprogress 의 verify 대상) hash.
  - 매 tick 시작 시 매니페스트 hash 와 직전 hash 비교:
    - 같으면 LLM 호출 skip + state `last_result=*_inputs_unchanged` 기록.
    - 다르면 LLM 호출 진행.
  - `packages/cli/run-role.sh` 에 이미 `planner_inputs_unchanged` / `ticket_inputs_unchanged` 시그널이 있음 — 이게 LLM 호출 skip 까지 이어지는지 확인 후 보강.
  - wiki 의 `wiki_inputs_unchanged` 처리 패턴 (`packages/cli/wiki-project.sh:1886`) 재사용.
- Out of Scope:
  - 강제 LLM 호출 (사용자 트리거 등) 무시 — 명시적 호출은 skip 안 함.
  - 매니페스트 scope 자동 학습.
