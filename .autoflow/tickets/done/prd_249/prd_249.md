## Core Scope

- Goal: `tickets/inbox/order_220.md` 를 generated PRD 로 승격한다. PTY runner 가 `active_ticket_id` / `active_ticket_title` / `active_stage` state 필드를 갱신하지 않아 UI 에 ticket 정보가 비어있는 문제를 worktree / inprogress 폴더 기반 fs enrichment 로 해결한다.
- In Scope:
  - `apps/desktop/src/main.js`
    - `enrichRunnerActiveTicketFromFs` 함수 또는 동등 로직을 개선해 `git worktree list` 로 worker 의 활성 worktree path 를 추출하고, 해당 path 와 inprogress 폴더를 매칭해 정확한 ticket id/title 을 도출한다.
    - `active_stage` 를 inprogress 폴더 내 ticket 존재 여부 + worktree 활성 상태로 결정한다 (비어있으면 "대기", 1개 이상이면 "구현").
    - inprogress 에 여러 ticket 이 있을 때 가장 최근 worktree 와 매칭되는 ticket 을 선택한다.
    - planner runner 에 대해서도 inbox/backlog 에 처리할 항목이 있으면 "계획", 없으면 "대기" 단계를 반영한다.
    - legacy state 가 `active_ticket_id` 를 이미 박아 둔 경우 그 값을 우선 사용한다.
  - `apps/desktop/src/renderer/main.tsx`
    - enriched `activeTicketId`, `activeTicketTitle`, `activeStage` 를 runner 카드 UI 에 표시한다.
    - PTY mode (`mode=pty`) 와 legacy mode 모두 호환.
- Out of Scope:
  - runner state 파일 포맷 변경.
  - 새 IPC 채널 추가.
  - PTY 스트림 처리 로직 변경.
  - `git push`.
