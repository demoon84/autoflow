## Core Scope

- Goal: Autoflow Desktop 사용자가 앱을 열면 첫 화면이 "대화" 메뉴이고, 거기서 프로젝트와 관련된 단일 대화 스레드를 계속 이어가면서 같은 화면에서 그 결과를 `tickets/inbox/memo_NNN.md` 또는 `tickets/backlog/prd_NNN.md` 로 저장할 수 있게 한다. 대화 히스토리가 잘리거나 초기화되어도 보드 자체와 LLM 위키(`wiki/answers/`)가 컨텍스트 source of truth 가 되도록, 어댑터에는 보드 스냅샷과 관련 wiki answer 발췌가 매번 자동 주입된다.
- In Scope:
  - 사이드바 내비게이션 **첫 자리**에 "대화" 메뉴(`chat`) 추가. 기존 항목(작업/Tickets/Wiki/통계/로그)은 순서를 유지한 채 한 칸씩 뒤로 밀린다.
  - 앱 실행 시 기본 진입 화면을 "대화" 로 변경한다. localStorage `autoflow.activeSettingsSection` 의 초기값을 `chat` 으로 두고, 저장된 값이 없거나 알 수 없는 키일 때도 `chat` 으로 fallback.
  - 대화 화면 구성: 상단 헤더(현재 프로젝트 + 모드 토글 + "Wiki 인용" 토글 + "이전 요약 인계" 토글 + 대화 초기화 버튼) + 본문 메시지 영역 + 하단 입력창 + 저장 액션 영역. 세션 리스트 / 세션 선택 UI 는 두지 않는다(단일 스레드).
  - 의도 모드 토글: `Auto`(기본) / `Memo only` / `PRD only`. 모드는 어댑터 system 프롬프트에 반영되어 질문 깊이가 달라진다.
  - AI 시스템 프롬프트에 두 어시스턴트 역할 계약을 주입:
    - PRD 모드: `.autoflow/agents/spec-author-agent.md` 절차.
    - Memo 모드: `.autoflow/README.md` Memo Intake Rules + `.claude/skills/order` 의 짧은 의도 보존 규칙.
    - 두 모드 모두 한국어 산출 기본, parser 민감 헤더/필드/경로/명령은 영어/원형 유지.
  - **보드 스냅샷 자동 주입(컨텍스트 보강 A)**: 어댑터 호출 시 system 프롬프트에 다음 스냅샷이 매번 자동 첨부된다. 각 섹션은 토큰 한도를 두며 환경변수 `AUTOFLOW_DESKTOP_CHAT_SNAPSHOT_BUDGET` 으로 덮어쓸 수 있다.
    - `wiki/index.md` 의 첫 헤더 + 처음 1~2 단락(또는 헤더 목록).
    - `wiki/project-overview.md` 의 첫 헤더 + 처음 1~2 단락.
    - `wiki/log.md` 의 최근 N(기본 10) 엔트리 헤더.
    - `wiki/answers/` 의 모든 파일에 대해 `(slug, title 한 줄, terms[])` 카탈로그. 본문은 포함하지 않고 검색용 인덱스만.
    - `wiki/{architecture,decisions,features,learnings,sources}/` 의 파일명 목록(섹션당 N=10).
    - 최근 done 티켓 10개의 `(project_key/ticket_id, 제목, 한 줄 요약)`.
    - 활성 inprogress 티켓의 `(ticket_id, 제목)` 목록.
    - 현재 inbox memo 와 backlog PRD 의 `(파일명, 제목)` 목록.
    - 이 스레드의 frontmatter `saved_paths` 누적 절대경로 목록.
  - **관련 wiki answer 자동 발췌(컨텍스트 보강 D)**: 사용자 메시지가 들어오면 매 호출 직전에
    - 최근 사용자 메시지(기본 직전 3건)에서 키워드 토큰을 추출.
    - `wiki/answers/*.md` 의 frontmatter `slug`, `terms[]`, 그리고 파일명에서 키워드 일치 점수를 계산.
    - 상위 K(기본 3, 환경변수 `AUTOFLOW_DESKTOP_CHAT_WIKI_ANSWERS_TOPK`)개 answer 의 frontmatter + `## Answer` 단락 + `## Citations` 를 system 프롬프트의 "Relevant wiki answers" 섹션에 첨부.
    - 매칭이 약하면(점수 임계 미만) 첨부하지 않고 비어있음을 명시.
    - 헤더의 "Wiki 인용" 토글이 OFF 면 자동 발췌가 비활성화되고 카탈로그(목록)만 남는다(default ON).
    - 첨부에 사용된 파일 절대경로는 그 호출의 응답을 화면에 표시할 때 메시지 하단에 "참고: wiki/answers/<slug>.md" 칩으로 노출되어 사용자가 즉시 따라갈 수 있다(클릭은 후속 PRD).
  - **AI 출처 표기 지시**: system 프롬프트에 "Relevant wiki answers 섹션을 참조한 경우 답변 본문 끝에 `Wiki: wiki/answers/<slug>.md` 형태로 출처를 표기하고, 보드 스냅샷의 done/inprogress/inbox/backlog 항목을 인용한 경우 해당 ticket id 또는 파일명을 함께 표기" 지시.
  - **자동 대화 요약(컨텍스트 보강 B)**: 단일 스레드 메시지 수가 임계값(기본 50, `AUTOFLOW_DESKTOP_CHAT_SUMMARY_THRESHOLD`)을 넘으면 다음 호출 직전에 가벼운 요약 호출을 보내 frontmatter `summary` 를 갱신. 이후 호출에서 윈도우 앞 prefix.
  - **초기화 시 요약 인계(컨텍스트 보강 C)**: "대화 초기화" → 확인 → 현재 `desktop-chat.md` 를 `desktop-chat-archive/<YYYYMMDD-HHMMSSZ>.md` 로 이동(요약 포함) → 빈 새 스레드 frontmatter 에 `prior_summary` + `prior_archive_path` 기록. 헤더 "이전 요약 인계" 토글이 ON(default)이면 다음 호출부터 prefix 주입. OFF 면 인계 없음.
  - 본문 하단에 "메모로 저장" / "PRD로 저장" 액션 버튼. AI 가 직전 답변에서 추천한 산출물 형식이 강조 표시되며, 사용자는 언제든 다른 쪽도 누를 수 있다.
  - 저장 액션 → AI 에게 표준 템플릿 그대로의 최종 마크다운을 요청 → 미리보기 다이얼로그(MUI Dialog) → 사용자가 "저장" 클릭 → IPC 가 보드에 파일 작성. AI 답변이 직접 보드 파일을 쓸 수 있는 경로는 없다.
  - 저장 경로:
    - Memo: `.autoflow/tickets/inbox/memo_NNN.md` (CLI `autoflow memo create` 우선, fallback 직접 쓰기. prefix `memo_` 와 CLI 명령 호환 유지).
    - PRD: `.autoflow/tickets/backlog/prd_NNN.md` (CLI `autoflow spec create <project-root> .autoflow --from-file <draft> --raw --save-handoff` 우선, fallback 직접 쓰기).
  - 단일 영속 대화 스레드: 프로젝트당 하나의 파일 `.autoflow/conversations/desktop-chat.md`. 메시지는 시간순 append. frontmatter 필드: `provider`, `model`, `project_root`, `created_at`, `last_active_at`, `mode`, `saved_paths[]`, `summary?`, `prior_summary?`, `prior_archive_path?`.
  - 컨텍스트 윈도우: 어댑터 매 호출에 `(prior_summary? + summary? + 최근 N 메시지 + 보드 스냅샷 + relevant wiki answers?)` 만 보낸다. N 기본 20, 환경변수 `AUTOFLOW_DESKTOP_CHAT_CONTEXT`.
  - 프로젝트 전환: 폴더가 바뀌면 그 프로젝트의 `desktop-chat.md` 와 그 프로젝트 기준 보드 스냅샷·wiki 인덱스로 자동 전환.
  - 단발성 어댑터 호출용 신규 IPC `autoflow:chatSend` + 셸 진입점 `runtime/board-scripts/chat-once.sh`. 기존 `run-role.sh` 의 `prepare_adapter_cli_env` / `run_default_adapter_command` 재사용.
  - 신규 IPC: `autoflow:chatLoad`(스레드 + 보드 스냅샷 + wiki answer 카탈로그 로드), `autoflow:chatAppend`, `autoflow:chatSummarize`, `autoflow:chatReset`, `autoflow:saveMemo`, `autoflow:saveSpec`. 저장 IPC 는 절대경로가 `.autoflow/tickets/inbox/` 또는 `.autoflow/tickets/backlog/` 안에 드는지 검증.
  - 어댑터 호출 중 취소: 기존 `autoflow:cancelInvocation` IPC 재사용.
  - 오류 처리: agent profile 미설정, 어댑터 실패, 저장 실패, 스냅샷 일부 누락(부분 누락은 누락 표시 후 진행) 모두 한국어 오류 + 다음 행동 힌트.
  - MUI Material + Emotion 우선(헤더 토글, 입력창, 다이얼로그). 채팅 버블만 자체 구현(`role="log"`, `aria-live="polite"`, 키보드 포커스, ESC 입력 비우기).
  - 사용자 노출 라벨/안내/오류는 한국어.
- Out of Scope:
  - 다중 대화 세션, 세션 리스트, 세션 검색·이름 변경.
  - memo/PRD 외의 보드 변경(티켓 이동, 러너 제어, 위키 편집, plan 작성, git 커밋·푸시).
  - 응답 토큰 단위 스트리밍.
  - provider 선택 드롭다운.
  - 저장된 memo/PRD 의 후속 자동 처리(plan/todo).
  - `autoflow wiki query --synth` 를 직접 호출하는 deep 모드(별도 LLM 비용). 첫 버전은 정적 키워드 매칭 lite 모드만.
  - wiki answer 본문 자동 첨부 외의 본문 전체 포함(answers 외 디렉토리는 파일명만).
  - 첨부 파일/이미지 업로드, 클라우드 동기화.

## Global Acceptance Criteria

- [ ] 사이드바 첫 자리에 "대화" 메뉴가 보이고, 그 아래에 작업/Tickets/Wiki/통계/로그가 같은 순서로 이어진다.
- [ ] 앱 첫 실행 또는 localStorage 가 비어 있을 때 첫 화면이 "대화" 페이지로 열린다.
- [ ] 대화 화면 진입 시 현재 프로젝트의 `.autoflow/conversations/desktop-chat.md` 가 자동 로드되어 시간순으로 표시된다(파일 없으면 빈 상태 안내).
- [ ] 메시지 송수신이 동작하고, 어댑터에 보내는 컨텍스트는 `(prior_summary? + summary? + 최근 N 메시지 + 보드 스냅샷 + relevant wiki answers?)` 으로 구성된다(N 기본 20).
- [ ] 매 어댑터 호출의 system 프롬프트에 보드 스냅샷이 포함된다(검증: chat-once 호출 트레이스에 wiki/index, wiki/project-overview, wiki/log 최근 N개, wiki/answers 카탈로그(slug+terms), 보조 wiki 디렉토리 파일 목록, done 10개, inprogress, inbox/backlog, saved_paths 섹션이 모두 직렬화되어 있다). 비어있는 섹션은 "(none)" 표기.
- [ ] "Wiki 인용" 토글 ON(default) 상태에서 사용자 메시지 키워드와 매칭되는 wiki answer 가 있을 때 system 프롬프트의 "Relevant wiki answers" 섹션에 상위 K(기본 3) 답변의 frontmatter + `## Answer` + `## Citations` 가 포함된다. 매칭 없으면 비포함이 명시된다.
- [ ] "Wiki 인용" 토글 OFF 시 wiki answer 본문 자동 첨부가 비활성화되고 카탈로그(목록)만 남는다(검증: 트레이스 비교).
- [ ] AI 응답이 wiki answer 또는 보드 항목을 참조한 경우 본문 끝에 `Wiki: wiki/answers/<slug>.md` 또는 ticket id/파일명이 표기된다(샘플 호출로 수동 확인).
- [ ] wiki answer 가 첨부되어 응답이 생성되면 그 메시지 하단에 "참고: wiki/answers/<slug>.md" 칩이 표시된다.
- [ ] 메시지 수가 임계값(기본 50)을 넘으면 frontmatter `summary` 가 자동 갱신된다.
- [ ] 모드 토글 `Auto / Memo only / PRD only` 가 헤더에 보이고 어댑터 system 프롬프트에 반영된다.
- [ ] AI 가 충분한 정보를 모은 시점에 "메모로 저장" 또는 "PRD로 저장" 중 하나가 추천 강조된다.
- [ ] "메모로 저장" → 미리보기 → "저장" → `.autoflow/tickets/inbox/memo_NNN.md` 가 생성된다.
- [ ] "PRD로 저장" → 미리보기 → "저장" → `.autoflow/tickets/backlog/prd_NNN.md` 가 생성되고 모든 템플릿 섹션이 채워진다.
- [ ] 미리보기 "취소" 시 보드에 어떤 파일도 생성되지 않는다.
- [ ] 저장 성공 시 frontmatter `saved_paths` 에 새 절대경로가 누적되고, 이후 호출의 보드 스냅샷에 그 경로가 반영된다.
- [ ] 응답 대기 중 입력창 비활성화, 취소 버튼 클릭 시 즉시 호출 종료 + "취소되었습니다".
- [ ] "대화 초기화" → 확인 → 현재 스레드(요약 포함)가 아카이브로 이동되고 새 스레드 frontmatter 에 `prior_summary` + `prior_archive_path` 가 채워진다.
- [ ] "이전 요약 인계" 토글 ON(default)이면 새 스레드 첫 호출부터 `prior_summary` 가 prefix 로 주입되고, OFF 면 주입되지 않는다.
