# Project PRD

## Project

- ID: prd_019
- Title: Reduce ticket workspace tabs to PRD + 발급 티켓 only
- AI:
- Status: draft

## Core Scope

- Goal: 데스크톱 `티켓 정보` 페이지의 탭 줄을 단순화한다. 현재 7개 탭 (`전체` / `PRD` / `발급 티켓` / `진행 중` / `막힘` / `검증/완료` / `반려`) 중 `전체` 와 단계별 필터 4개 (`진행 중`, `막힘`, `검증/완료`, `반려`) 를 모두 제거하고, **`PRD`** 와 **`발급 티켓`** 두 탭만 노출한다. `발급 티켓` 안에 진행/막힘/검증완료/반려 상태가 모두 포함되도록 필터 제한을 풀어 한 화면에서 티켓의 모든 단계를 본다.
- In Scope:
  - `apps/desktop/src/renderer/main.tsx` 의 `TicketWorkspaceTabKey` 타입에서 `all`, `inprogress`, `blocked`, `closed`, `reject` 키 제거. 남는 키는 `prd` 와 `issued` (= 발급 티켓) 두 개.
  - 탭 정의 배열(현재 7개 항목) 을 `prd`, `issued` 만 있는 2개 배열로 축소.
  - 기본 활성 탭을 `issued` 로 (현재 default 가 `all` 이면 default 변경 필요).
  - `발급 티켓` 탭이 보여주는 티켓 set 을 backlog 외 모든 ticket 파일 (`tickets/{todo, inprogress, done, reject}/tickets_*.md`) 의 합집합으로 정의해, 진행 단계 무관하게 한 줄 리스트로 노출. 각 티켓 카드의 stage badge (`발급됨`, `구현 중`, `완료`, `반려` 등) 는 그대로 표시해 시각적으로 단계를 구분.
  - `localStorage` 에 저장돼 있던 활성 탭 키 (`autoflow.activeTicketWorkspaceTab` 등) 가 제거된 키였을 때, fallback 으로 `issued` 사용.
  - 우측 상단 보조 텍스트 `"PRD와 발급된 티켓 전체"` 는 그대로 의미가 맞으므로 유지.
- Out of Scope:
  - 티켓 카드 내부 디자인 / 정렬 변경.
  - PRD 카드의 표시 정책 변경.
  - 우측 상세 미리보기 패널 변경.
  - 검색 / 필터 입력 추가 (단계별 필터를 빼는 대신 별도 검색 도입은 다른 PRD).
  - 다른 페이지 (`작업 흐름`, `통계`, `Wiki`) 의 탭 / 카운트.

- [ ] 티켓 정보 페이지의 탭 줄에 정확히 두 개 탭 (`PRD` 와 `발급 티켓`) 만 보인다. `전체` / `진행 중` / `막힘` / `검증/완료` / `반려` 가 모두 사라졌다.
- [ ] 페이지 진입 직후 기본 활성 탭이 `발급 티켓` 이고, 거기 표시되는 티켓 리스트에 `발급됨`, `구현 중`, `완료`, `반려` 등 단계가 다른 카드가 한 화면에서 함께 보인다.

- [ ] 이전에 `localStorage` 에 `all` / `inprogress` 등 사라진 키가 저장된 사용자가 페이지를 다시 열 때, 자동으로 `issued` 탭이 선택되고 콘솔 에러 없음.
- [ ] 우측 상단 카운트 라벨이 더 이상 사라진 탭의 카운트를 노출하지 않는다 (예: 헤더에서 `진행 중 (3)` 뱃지가 따로 있다면 함께 제거).

- [ ] `cd apps/desktop && npm run check` 가 통과한다.
- [ ] 시각 회귀: 다른 사이드 페이지 영향 없음.

- 단계별 필터를 다시 원할 가능성에 대비해, 티켓 단계는 카드 badge 로 계속 노출 (제거 X).

- localStorage 키 마이그레이션은 단순 fallback (잘못된 값 → `issued`) 으로 처리, 기존 키 자체 삭제는 다음 페이지 방문 시 새 값으로 자연 덮어쓰기.
