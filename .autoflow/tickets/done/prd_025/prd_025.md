# Project PRD

- Status: draft

## Core Scope

- Goal: 작업 흐름 페이지의 AI 카드 progress bar 가 (1) 런타임에서 실제로 구분 가능한 단계만 보여주고, (2) 활성 dot 의 위치가 라벨/연결선과 정확히 정렬되도록 두 가지를 함께 정리한다. 현재 Impl AI 의 6단계 중 "계획"/"검증" 은 별도 emit 신호가 없어 fiction 이고, 단계가 적게 표시되는 케이스(예: 첫 단계만 활성) 에서 dot 이 컨테이너 가장자리에 잘리거나 라벨과 어긋나 시각적으로 어색하다.
- In Scope:
  - **Impl AI (ticket-owner / Worker) progress 정리**:
    - `ownerFlowStages` 6단계를 **`대기 / 구현 / 완료 / 반려` 4단계** 로 축소.
    - `runnerStageKey` 의 ticket-owner 분기에서:
      - 활성 티켓 없음 → `todo` (대기)
      - active_stage 가 `executing|claimed|inprogress|verifying|ready_to_merge|merging` 중 어느 것이든 → `inprogress` (구현, mini-plan/구현/검증/머지 통합)
      - last_result 에 `committed_via_inline_merge` 또는 `event=adapter_finish.*status=ok` → `done` (완료)
      - fail-like → `reject` (반려)
    - 라벨 meta 영역에 "mini-plan / 구현 / 검증 / 머지 통합" 설명.
  - **Plan AI / Wiki AI progress 유지**:
    - Plan AI 는 prd_014 매핑 정밀화 적용 후 그대로 4단계.
    - Wiki AI 도 4단계 그대로.
  - **Progress dot / 연결선 정렬 수정**:
    - 활성 dot (`.ai-progress-dot`) 이 단계 라벨의 정확한 가운데 위에 오도록 CSS 정렬 보정.
    - 첫 단계가 활성일 때 dot 이 컨테이너 좌측 padding 영역에 걸치지 않도록 `padding-left` 또는 `transform` 보정.
    - 마지막 단계가 활성일 때도 우측 동일.
    - 단계가 4개 이하로 적어졌을 때 (Impl AI 가 6 → 4 로 줄어들면서 발생할 가능성) dot 간 간격이 균등 분배되어 progress 채움(bar fill) 의 시작/끝 좌표가 dot 중앙과 일치.
    - 카드 폭이 좁아져 progress 가 두 줄로 wrap (prd_021 항목) 될 때도 첫 줄/둘째 줄 모두 dot 이 라벨과 정렬됨.
  - **테스트 케이스 시각 검증 매트릭스**:
    - Impl AI 의 4단계 각각이 활성일 때 dot 위치가 라벨과 일치 (1번/2번/3번/4번).
    - Plan AI 의 4단계 각각도 동일 검증.
    - Wiki AI 의 4단계 각각도 동일 검증.
- Out of Scope:
  - 단계 색상 / 아이콘 변경.
  - 연결선(연결 bar) 의 두께/색 변경.
  - 카드 외곽 / 좌측 보더 (prd_023 별도 처리).
  - 시작/정지 버튼 / 모델 셀렉터 (prd_021 별도 처리).
  - 단계 라벨 다국어 변경.
