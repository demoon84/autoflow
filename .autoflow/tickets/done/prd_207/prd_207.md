## Core Scope

- Goal: `apps/desktop/src/renderer/main.tsx` 의 `AiConversationPanel` (line ~6285) 헤더 `<header className="ai-conversation-panel-head">` 안에 binary 상태 인디케이터(컬러 점 + 라벨)를 추가한다. 데이터는 이미 IPC 로 들어오는 `runner.stateStatus` (`running` / `idle` / `stopped` / `blocked` / `needs_user`) 와 `runner.pid` 를 그대로 사용해 새로운 IPC 채널은 만들지 않는다.
- In Scope:
  - `AiConversationPanel` 의 props 에 `runner: AutoflowRunner` (이미 받고 있음) 를 사용해 상태 인디케이터 element 를 헤더에 추가. 표시 규칙:
    - `running` 또는 `pid` 존재: `● 실행 중` (초록)
    - `stopped`: `○ 정지` (회색)
    - `blocked` / `needs_user`: `⚠ 막힘` (주황 또는 빨강)
    - 그 외 (`idle` 등): `◌ 대기` (옅은 회색)
  - 새 CSS 클래스(`.ai-conversation-panel-status`, `.ai-conversation-panel-status-running` 등)를 `apps/desktop/src/renderer/styles.css` 의 기존 `.ai-conversation-panel-head` 블록(line ~3135) 아래에 추가. 색상은 기존 `runner-status-dot` 톤(`runner-status-running` / `runner-status-stopped` / `runner-status-blocked` / `runner-status-idle`) 과 시각적으로 일관되게 맞춘다.
  - `AiConversationPanel` 호출 지점 2곳(`main.tsx` line ~6301 의 panel 사용처와 line ~6967 의 `progress:${runner.id}` 사용처) 에서 실제로는 `AiConversationPanel` 가 한 곳에서만 쓰이고 다른 한 곳은 `ConversationStream` 직접 호출이므로 `AiConversationPanel` 본인 props 만 변경한다. 직접 `ConversationStream` 을 호출하는 곳에는 같은 인디케이터를 wrapper 로 추가하지 않는다 (scope minimal).
  - 접근성: 인디케이터는 `aria-live="polite"` 로 상태 변화 announce, 시각 dot 은 `aria-hidden="true"`, 라벨 텍스트는 screen reader 가 읽을 수 있게 노출.
- Out of Scope:
  - 새 IPC 채널, 새 runner state 필드, main process 변경.
  - PRD 188 (응답 지연 severity 라벨) 영역 변경 — 이번은 binary 시그널만.
  - `ConversationStream` 내부 구현 변경 (ANSI 변환 등).
  - 러너 카드(`runner-row`) 의 `runner-status-dot` (line ~3573) 변경 — 이미 존재하므로 재사용 X.
  - `apps/desktop/src/main/`, electron preload, IPC handler 변경.
  - `git push`, runner process 관리, 새 의존성 추가.
