## Core Scope

- Goal: 최근 대시보드에서 보이는 비현실적인 `tokenUsageCount` 및 변경코드량 카드의 신뢰도를 회복한다. 구현은 "집계 파서가 지금도 잘못 합산하는가" 와 "과거 왜곡 데이터가 현재 누적값을 오염시키는가" 를 분리해서 검증하고, destructive cleanup 없이 현재 운영 값이 안정되도록 보정하는 데 집중한다.
- In Scope:
  - 토큰 사용량 집계 경로 검증:
    - `packages/cli/run-role.sh` 와 `runtime/board-scripts/run-role.sh` 의 adapter 종료 후 usage 추출 경로를 확인하고, Codex `--json` 의 `turn.completed.usage` 만 신뢰하는 현재 계약이 임베디드 JSON(`item.completed` 내부 aggregated JSON) 까지 중복 합산하지 않는지 점검한다.
    - Claude stream-json 의 intermediate event 와 final `result.usage` 처리 경로가 현재 구현에서 분리돼 있는지 확인하고, 필요하면 intermediate skip / final-only 집계를 더 명시적으로 고정한다.
    - `packages/cli/telemetry-project.sh` 와 `packages/cli/metrics-project.sh` 가 읽는 source-of-truth 가 `.autoflow/telemetry/runs.jsonl` 인지, legacy `.autoflow/metrics/telemetry-runs.jsonl` 경로가 집계에 섞이지 않는지 검증한다.
  - 토큰 baseline 보정:
    - 현재 부풀려진 값의 주원인이 과거 telemetry row 라면, raw `runs.jsonl` 를 직접 절단하지 않고 현재 집계에서만 legacy row 를 제외하거나 시간 창을 두는 read-side 보정 1개를 적용한다.
    - 보정 방식은 운영 중인 최신 runner tick 의 `token_input`, `token_output`, `started_at` 또는 동등한 필드를 기준으로 재현 가능해야 한다.
  - 변경코드량 집계 검증:
    - `packages/cli/metrics-project.sh` 의 commit / insertions / deletions / files changed 계산이 local completion commit 기준인지 확인하고, working tree dirty state 나 autoflow 외부 미머지 변경이 섞이지 않는지 검증한다.
    - dirty state 가 metrics 에 반영되지 않아야 한다면 그 정책을 코드 주석 또는 함수명/로직으로 명확히 고정한다. 반대로 반영 대상이라면 어떤 입력을 반영하는지 명시적으로 드러나게 한다.
  - 대시보드 표시 경로 점검:
    - `apps/desktop/src/main.js` 와 `apps/desktop/src/renderer/main.tsx` 에서 token usage / code volume 표시가 위 집계 결과를 그대로 소비하는지 확인하고, 이미 올바른 경우에는 UI 로직을 불필요하게 바꾸지 않는다.
- Out of Scope:
  - `.autoflow/telemetry/runs.jsonl` 과거 row 를 직접 삭제하거나 rewrite 하는 일회성 cleanup.
  - 신규 telemetry schema 도입.
  - `autoflow doctor` 확장이나 별도 cleanup CLI 신설.
  - 집계와 무관한 dashboard UI 레이아웃 변경.
  - `git push`.

## Global Rules

- Allowed Paths are relative to the host project root.
- Verification commands run from the host project root unless a ticket says otherwise.
- `.autoflow/telemetry/runs.jsonl` 의 기존 행은 이번 작업에서 destructive edit 하지 않는다. 현재 값 보정은 read-side 에서 해결한다.
- Codex 경로는 최종 `turn.completed.usage` 만 신뢰하고, intermediate / embedded JSON 은 집계 대상에서 제외한다.
- Claude stream-json 경로는 final `result.usage` 만 집계 대상으로 삼고, partial / intermediate event 는 token 합산 대상에서 제외한다.
- 변경코드량 정책은 local completion commit 기반으로 유지하고, uncommitted dirty state 를 metrics 에 합산하려면 명시적 근거가 있어야 한다. 근거가 없으면 dirty state 는 제외한다.
- Human-readable PRD prose should be Korean by default. Preserve parser-sensitive section names, field names, ids, paths, commands, code, and runtime formats.
