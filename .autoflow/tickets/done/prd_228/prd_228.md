## Core Scope

- Goal: claude 어댑터 cmd 를 stream-json + include-partial-messages 로 바꾸고, post-processor 가 JSONL 의 final `result` 이벤트에서 사용자 메시지를 정확히 추출하며, LiveTerminalView 가 active live_stdout.log 를 동시에 tail 해 JSONL 을 한 줄 요약으로 흘려 보여준다. CLI 가 stream-json 미지원이거나 `AUTOFLOW_CLAUDE_STREAM=0` 일 때 legacy text 모드로 안전하게 fallback 한다.
- In Scope:
  - `packages/cli/run-role.sh` (`runner_claude_base_cmd` 기본 cmd, post-processor):
    - 기본 호출을 `claude -p --dangerously-skip-permissions --permission-mode bypassPermissions --output-format stream-json --include-partial-messages` 로 전환.
    - `claude --help` 에 `stream-json` 토큰이 없거나 env `AUTOFLOW_CLAUDE_STREAM=0` 이면 기존 `--output-format text` 로 자동 fallback.
    - post-processor: live_stdout.log 가 JSONL 이면 마지막 `{"type":"result","subtype":"success","result":"..."}` 의 `result` 필드를 jq/python 으로 추출해 `*_last_message.txt` 에 기록. 추출 실패 시 `type:assistant` 의 `message.content[].text` concat fallback. 둘 다 실패하면 raw stdout 그대로 남김 (legacy 보존).
    - JSONL 이 아니거나 stream 모드 미사용이면 기존 raw text 처리 유지.
  - `runtime/board-scripts/run-role.sh`: 위 변경을 동일하게 sync.
  - `packages/cli/wiki-project.sh`: wiki adapter cmd 가 같은 변경 적용 (또는 wiki 는 동일 fallback 정책). wiki 의 stream-json 전환은 출력 cap 정책과 충돌 없도록 동일 패턴으로.
  - `packages/cli/runners-project.sh`, `runtime/board-scripts/runners-project.sh`: 도움말/표시 문자열 sync.
  - `apps/desktop/src/renderer/main.tsx`:
    - `useLiveStdoutText` (또는 동등) 훅이 `runner.lastStdoutLog` (state `last_stdout_log`) 를 함께 1초 tail 하도록 확장.
    - persistent `<runner>.log` + active `live_stdout.log` 두 stream 을 merge: persistent 먼저, active live 뒤. fingerprint 는 path+offset 으로.
    - JSONL 라인 → 한 줄 요약 변환:
      - `system/init` → `[init] tools=N model=...`
      - `text` → text 원문
      - `tool_use` → `[tool] <name> <truncated input summary>`
      - `tool_result` → `[result] <preview 80 chars>`
      - `result/success` → `[done] <result preview>`
      - `message_delta` → text fragment 그대로 append
      - 기타 → drop (또는 dim 옵션)
    - 기존 ANSI 컬러 처리 (`colorizeLogChunk`) 유지.
  - `apps/desktop/src/main.js`:
    - token usage 추출이 stream-json 의 마지막 `result` 이벤트 `usage.{input_tokens,output_tokens,total_tokens,cache_read_input_tokens}` 를 우선 사용하고, 기존 정규식 fallback 도 유지 (Codex/Gemini 호환).
  - `AGENTS.md`: rule 19b (output cap) / 19c (realtime) 인접 위치에 stream-json 기본/`AUTOFLOW_CLAUDE_STREAM=0` rollback 옵션 한 줄 추가.
- Out of Scope:
  - codex / gemini / opencode adapter 의 stream 출력 전환 (별도 PRD).
  - 새 IPC 채널, 새 의존성 추가.
  - xterm fit/cols, scrollback 50000, debounce, chunk flush 동작 변경.
  - shell sanity gate (zero-diff + Done When) / output_truncated marker / consecutive_timeout_count 카운터 의미 변경.
