# PRD 213 — SKILL 제거 Phase 2: CLI subcommand 제거

## Goal

`autoflow skill ...` CLI subcommand 와 curator-run 보조 script, `wiki query --rag` 의 SKILL view_count 자동 증가 로직, package manifest 의 skill 자산 entry 를 모두 제거한다. 이 phase 이후 `autoflow skill` 호출은 unknown command 로 떨어지고, RAG retrieval 도 SKILL view counter 를 건드리지 않는다.

## Background

- order_190 phase 2. phase 1(prd_212) 가 자동 호출 site 를 끊은 뒤 CLI 자체를 떼어낸다.
- `bin/autoflow` 의 `skill)` case 가 `packages/cli/skill-project.sh` 로 dispatch.
- `runtime/board-scripts/curator-run.sh` 는 skill curator lifecycle script — SKILL 폐기 시 무의미.
- `packages/cli/wiki-project.sh` 가 `--rag` 결과에서 SKILL path 를 만나면 `view_count` / `last_viewed_at` 를 +1 (rule 18e) — SKILL 폐기 시 의미 없음.
- `packages/cli/package-board-common.sh` 의 `managed_board_asset_entries` 에 `wiki/skills/README.md`, `wiki/skills-local/README.md`, `wiki/skills/skill-template.md` 등 skill 관련 자산 등록.

## Decision

CLI 표면 전체를 한 번에 떼어낸다. 호출자(phase 1 에서 hook 제거, phase 3 에서 desktop IPC 제거)가 모두 정리된 뒤이므로 회귀 위험 낮음. 단, 본 PRD 는 phase 1(prd_212) 가 `Stage: done` 이 된 뒤 todo 로 promote 되어야 안전 — planner 가 직렬로 진행한다.

## Scope

- `bin/autoflow`: `skill)` case 와 그에 딸린 도움말/usage 라인 제거. dispatcher unknown command 분기는 그대로.
- `packages/cli/skill-project.sh`: 파일 삭제.
- `runtime/board-scripts/curator-run.sh`: 파일 삭제.
- `packages/cli/wiki-project.sh`: `--rag` 결과의 SKILL `.usage.json` view_count 증가 로직(`skill_view_increments` 카운터 포함) 제거. 다른 RAG 출력 형식(`result.*`, `chunk_*`) 은 변경하지 않는다.
- `packages/cli/package-board-common.sh`: `managed_board_asset_entries` 의 `wiki/skills/README.md`, `wiki/skills-local/README.md`, `wiki/skills/skill-template.md` 등 skill 관련 entry 제거. 다른 asset entries 유지.

## Out of Scope

- `.autoflow/wiki/skills/` 실제 컨텐츠 / `skills-local/` 디렉토리 (phase 6).
- order/autoflow skill 의 RAG 호출 (phase 4).
- `/skill-this` skill 디렉토리 (phase 5).
- AGENTS.md rule 본문 (phase 6).

## Allowed Paths

- `bin/autoflow`
- `packages/cli/skill-project.sh`
- `packages/cli/wiki-project.sh`
- `packages/cli/package-board-common.sh`
- `runtime/board-scripts/curator-run.sh`

## Done When

- [ ] `./bin/autoflow skill list` 또는 임의 `autoflow skill <sub>` 가 unknown command 메시지로 떨어진다.
- [ ] `./bin/autoflow help` (또는 인자 없는 호출) 출력에 `skill` 단어가 0 hit.
- [ ] `packages/cli/skill-project.sh` 와 `runtime/board-scripts/curator-run.sh` 가 존재하지 않는다.
- [ ] `packages/cli/wiki-project.sh` 에 `view_count` / `skill_view_increments` 가 0 hit.
- [ ] `packages/cli/package-board-common.sh` 의 `managed_board_asset_entries` 에 `skills` 가 0 hit.
- [ ] `bash -n bin/autoflow packages/cli/wiki-project.sh packages/cli/package-board-common.sh` 통과.
- [ ] `bash .autoflow/scripts/board-guard.sh` PASS.

## Verification

- Command: `bash -n bin/autoflow packages/cli/wiki-project.sh packages/cli/package-board-common.sh && ./bin/autoflow skill list 2>&1 | grep -qi unknown && grep -nE "skill_view_increments|view_count" packages/cli/wiki-project.sh; test $? -ne 0 && bash .autoflow/scripts/board-guard.sh`
