## Problem

PTY 재시작 / worker abandon / 작업 중 크래시로 발생하는 stuck 상태(stale ownership lock, atomic 위반, 빈 worktree 좀비 등)를 현재는 사용자가 수동으로 정리해야 한다. 매 tick 자동 점검/복구로 사람 개입 빈도를 줄인다.

## Goal

Planner AI(`plan-to-ticket-agent`)가 매 tick 5가지 mechanical 점검을 수행하고 안전한 복구만 자동으로 처리한다. mechanical하게 판단 불가한 케이스만 `needs_user`로 park하고 플로우는 계속 유지한다.

## Scope

### 점검 항목 (매 tick, mechanical 검사만)

1. **Stale ownership lock 정리**: ticket의 `Result: pending ticket-owner by <id>:<pid>:...` — `kill -0 <pid>`로 dead 판정 시 lock 라인 삭제 + audit 1줄
2. **Atomic 위반 (inprogress 2건 이상)**: 가장 많이 진행된 1건 유지, 나머지 `todo/`로 이동 (worktree 보존)
3. **빈 inprogress + 빈 worktree 좀비**: worktree 변경 0 + Stage=todo + 30분 이상 경과 → todo 이동 + `git worktree remove --force`
4. **고아 worktree**: `git worktree list`에 매칭 inprogress ticket 없고 변경 없음 → `git worktree remove` + audit
5. **Stage/폴더 mismatch**: inprogress ticket의 Stage가 `todo` + worktree 변경 0 + 30분 경과 → todo 이동

### 구현 방법

`planner-janitor.js` 신규 스크립트 또는 `start-plan.sh`/`board-guard.sh`에 inline 추가. planner tick이 이 janitor를 자동 호출한다. `plan-to-ticket-agent.md`에 새 janitor 책임 1 섹션 추가.
