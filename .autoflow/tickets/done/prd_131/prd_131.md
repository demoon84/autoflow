## Core Scope

- Goal: 현재 3-runner/4-role topology 에서 더 이상 참조하지 않는 `owner-1`, `planner-1`, `wiki-1` 계열 state 파일과 `*.state.<random>` 임시 잔재를 제거하고, 같은 유형의 stale 파일이 다시 누적되지 않도록 runner cleanup 도구에 state cleanup을 추가한다.
- In Scope:
  - `.autoflow/runners/config.toml` 에 enabled runner id 가 suffix 없는 `planner`, `worker`, `wiki`, `verifier` 인 것을 기준으로 legacy `owner-1.state`, `planner-1.state`, `wiki-1.*`, `wiki-1.semantic-lint.pages.d/` 를 stale 로 다룬다.
  - `.autoflow/runners/state/wiki.state.CWxbDK`, `.autoflow/runners/state/wiki.state.wLVnDN` 처럼 `runner-common.sh` 의 atomic state write temp 패턴(`*.state.XXXXXX`)으로 보이는 잔재를 제거한다.
  - `packages/cli/cleanup-runner-logs.sh` 를 확장하거나 같은 CLI 경로에서 호출되는 좁은 helper를 추가해 legacy suffix state와 old atomic temp state 파일을 cleanup 대상으로 포함한다.
  - root cause 위치를 코드로 확인해 필요하면 `.autoflow/scripts/runner-common.sh` 의 `runner_write_state()` temp cleanup 방어를 보강한다. 단, 현재 활성 `planner.state`, `worker.state`, `wiki.state`, `verifier.state` 는 삭제하지 않는다.
  - `packages/cli/wiki-project.sh` 는 실제로 `wiki.state.<random>` leak 원인으로 확인된 경우에만 수정한다. 원인이 `runner-common.sh` atomic write라면 `wiki-project.sh` 변경 없이 조사 결과를 ticket Notes/Result 에 남긴다.
- Out of Scope:
  - runner start/stop/restart, process kill, background process cleanup.
  - 현재 활성 suffix 없는 runner state 파일 삭제 또는 runner status 값 재작성.
  - `.autoflow/wiki/` content 변경, wiki synthesis, semantic lint 결과 재생성.
  - `.autoflow/runners/state/wiki-baseline.history`, `wiki.*` 현재 baseline/debounce/input manifest 파일 삭제.
  - 오래된 completion log 압축 정책 변경.
