## Core Scope

- Goal: Shorten the static natural-language runtime output emitted by `start-ticket-owner.sh` after a ticket is prepared, without changing ticket claim, worktree setup, run file creation, or finish/verify script behavior.
- In Scope:
  - Replace the long `next_action=` value in `.autoflow/scripts/start-ticket-owner.sh` with a concise cue such as `ticket-owner-agent.md flow for ticket %s`.
  - Remove the `routing_verify=`, `routing_pass=`, and `routing_fail=` emissions from `.autoflow/scripts/start-ticket-owner.sh`.
  - Apply the same output-shape change to `runtime/board-scripts/start-ticket-owner.sh` so the current board runtime and source runtime stay aligned.
  - Preserve all machine-readable keys needed for ticket claiming and execution context, including `ticket`, `ticket_id`, `owner`, `stage`, `source`, worktree fields, `implementation_root`, `run`, `done_target`, `reject_target`, `board_root`, and `project_root`.
- Out of Scope:
  - Do not edit `packages/cli/run-role.sh`; it is cited only because it includes runtime output in adapter prompts.
  - Do not alter ticket-owner claim selection, inprogress resume behavior, worktree creation, verification run file setup, done/reject target paths, or finalization scripts.
  - Do not remove `next_action=` entirely.
  - Do not implement or verify any product behavior beyond the runtime output change.
