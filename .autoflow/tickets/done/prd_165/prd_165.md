## Core Scope

- Goal: Phase 1~4 의 skill 시스템을 production-grade 로 끌어올리는 고도화. 보안 가드 강화, 외부 표준 호환, 고급 기능 (clustering / deterministic mode).
- In Scope:
  - **Security scan (`tools/skills_guard.py` 패턴 채택)**:
    - 자동 추출된 skill 본문 검사:
      - 악성 명령 패턴 (rm -rf /, fork-bomb, curl | sh 등)
      - Secret leak (API key, password, token 형태 정규식)
      - Allow-list 외 shell 명령 (인라인 shell 활성화 시 — 본 PRD 에서 결정)
    - 위반 감지 시 skill 생성 거부 + warning 로그 + check_NNN.md (order_135) 와 연동해 사람 검토 큐에 적재.
    - 환경변수: `AUTOFLOW_SKILL_SECURITY_SCAN_ENABLED` (기본 1).
  - **agentskills.io 호환 (검토 후 채택 결정)**:
    - 표준 frontmatter 매핑 (`name`, `description` 같은 핵심 필드는 이미 호환).
    - Autoflow 고유 필드 (`pattern_type`, `applies_to`) 는 `metadata.autoflow.*` 네임스페이스 하위.
    - import/export CLI: `autoflow skill import <agentskills_url>`, `autoflow skill export <name>` (agentskills format 변환).
    - 호환 시 cross-tool 공유 가능 (Hermes / 다른 agent 와 skill 교환).
    - **Plan AI 가 호환 여부 최종 결정** (호환 비용 vs 단순성 트레이드오프).
  - **Skill clustering / Meta-skill**:
    - 비슷한 skill (keyword overlap > 70%, 같은 pattern_type, 같은 applies_to) 묶음 자동 감지.
    - Curator (Phase 2) 가 7일 review 시 cluster 후보 list 생성.
    - **Meta-skill**: cluster 안 skill 들의 공통 pattern 을 추출한 상위 skill (Wiki AI auxiliary client 사용).
    - 사람 검토 후 meta-skill 채택 / 기각 (데스크톱 UI 통해).
  - **Deterministic mode (장기 비전)**:
    - skill 매칭 신뢰도가 매우 높은 케이스 (success_rate ≥ 95%, success_count ≥ 10) 는 LLM 호출 없이 skill 의 `Recommended Procedure` 를 그대로 실행.
    - 환경변수: `AUTOFLOW_SKILL_DETERMINISTIC_MODE_ENABLED` (기본 0, opt-in).
    - 임계값:
      - `AUTOFLOW_SKILL_DETERMINISTIC_MIN_SUCCESS_RATE_PERCENT` (기본 95)
      - `AUTOFLOW_SKILL_DETERMINISTIC_MIN_SUCCESS_COUNT` (기본 10)
    - 결정적 실행 결과가 fail 이면 자동 disable + low_confidence 마킹 → 다음 호출은 LLM 으로 fallback.
- Out of Scope:
  - 기본 인프라 (Phase 1)
  - Curator + Auto-extraction (Phase 2)
  - RAG injection / 활용 (Phase 3)
  - 데스크톱 UI 기본 (Phase 4)
