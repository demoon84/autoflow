## Project

- ID: prd_050
- Name: Gate and trim wiki semantic lint prompts
- Title: Add change-gate and per-page truncation to wiki semantic lint
- Goal: Reduce recurring Wiki AI adapter token usage by skipping unchanged semantic lint runs and sending shorter wiki page context when lint does run.
- AI: planner-1
- Status: generated

## Core Scope

- Goal: Add a deterministic unchanged-input gate to `run_semantic_lint` and reduce the per-page wiki content slice sent to the Wiki AI adapter.
- In Scope:
  - Fingerprint the wiki markdown inputs used by semantic lint with a stable file manifest, such as path, mtime, and size.
  - Skip `run_semantic_lint` adapter prompting when the wiki input fingerprint matches the last successful semantic lint run.
  - Store the successful semantic lint fingerprint in a runner-aware or board-local state file that is durable across ticks but does not touch `.autoflow/wiki/` content.
  - Reduce the per-page semantic lint prompt slice from `sed -n '1,220p'` to a shorter bounded slice, around the first 80 lines per page.
  - Add or extend a smoke test proving an unchanged second semantic lint call skips the adapter and that a changed wiki page triggers the adapter again.
- Out of Scope:
  - Do not change wiki synthesis behavior, wiki query behavior, non-semantic lint behavior, ticket lifecycle behavior, or runner topology.
  - Do not modify `.autoflow/wiki/` generated knowledge pages.
  - Do not introduce AI synthesis into Plan AI or Ticket Owner ticks.
  - Do not implement digest headers or recent-diff prompting unless it is simpler than the bounded per-page slice and satisfies the same acceptance criteria.

## Global Acceptance Criteria

- [ ] `run_semantic_lint` computes a wiki input fingerprint from the markdown files it would send to the adapter.
- [ ] After a successful semantic lint adapter run, the fingerprint is recorded in board/runner state.
- [ ] A later semantic lint run with the same wiki input fingerprint returns a skipped status without invoking the adapter.
- [ ] Changing a wiki markdown page changes the fingerprint and allows semantic lint to invoke the adapter again.
- [ ] The semantic lint prompt no longer includes the first 220 lines of every wiki page; each page is capped to roughly the first 80 lines or a comparably smaller deterministic summary.
- [ ] Existing `semantic_status=skipped_no_adapter`, `semantic_status=failed`, `semantic_status=ok`, and `semantic_finding.*` output contracts remain parseable.
