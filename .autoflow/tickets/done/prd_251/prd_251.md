## Core Scope

- Goal: `tickets/inbox/order_222.md`를 generated PRD로 승격한다. Phase 1은 claude CLI만 대상으로 하며, codex/gemini 지원은 후속 PRD로 분리한다.
- In Scope:
  - `apps/desktop/src/main/runner-pty-manager.js` 또는 `apps/desktop/src/main/`에 새 helper 파일
    - PTY spawn 시점에 `~/.claude/projects/$(echo $PROJECT_PATH | tr '/' '-')/` 디렉토리의 현재 JSONL 파일 mtime 스냅샷 기록.
    - 이후 신규 생성되는 JSONL 파일을 `fs.watch`/`chokidar`(이미 사용 중이면) 또는 setInterval poll로 추적.
    - JSONL 파일에서 `{"type":"message_delta","usage":{...}}` 또는 `{"type":"result",...,"usage":{...}}` 라인을 파싱해 `input_tokens`, `output_tokens`, `cache_read_input_tokens`, `cache_creation_input_tokens`를 누적.
    - 누적값을 `cumulative_tokens`, `last_turn_tokens`로 runner state JSON에 기록. `token_source=session_log` 필드 추가.
    - IPC로 renderer에 `runner-token-update` 이벤트 전송.
    - PTY stop 시 watcher dispose.
  - `apps/desktop/src/main.js`
    - token watcher 초기화 / 정리 훅 연결.
    - 기존 `last_token_usage_source`, `token_budget_*` 필드가 존재하면 그 포맷과 호환 유지.
  - `apps/desktop/src/preload.js`
    - `runner-token-update` IPC 이벤트를 renderer에 expose (이미 있으면 skip).
  - `apps/desktop/src/renderer/main.tsx`
    - `runner-token-update` 이벤트를 수신해 해당 runner 카드의 토큰 표시를 갱신.
    - token 추출 실패 시 `0 tokens` 또는 `-` fallback 유지 — UI 깨지지 않음.
- Out of Scope:
  - codex, gemini 세션 로그 파싱 (후속 PRD).
  - PTY stdout 직접 파싱.
  - interactive PTY에서 `--output-format stream-json` 사용.
  - runner state 파일 포맷의 breaking change.
