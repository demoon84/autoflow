## Core Scope

- Goal: `tickets/inbox/order_129.md`의 요청을 통계 페이지 정보 위계, 용어, 단위, 차트 의미 정비 작업으로 승격한다.
- In Scope:
  - `ReportingDashboard` 상단 요약을 핵심 KPI 4개 중심으로 재구성한다: 완료 티켓, 검증 통과율, 막힌 항목, 토큰 사용량.
  - "인수인계", "AI 산출물", "완료 커밋"처럼 사용자가 즉시 해석하기 어려운 용어는 통계 화면 맥락에 맞게 직관적 표현으로 바꾸거나 보조 정보로 낮춘다.
  - 카드와 차트의 단위 표기를 `개`, `줄`, `토큰`, `%`, `커밋`처럼 일관되게 노출하고, 큰 숫자는 화면 표시에서 `formatCompactCount`를 일관되게 사용한다.
  - 서로 다른 단위가 같은 bar 길이로 비교되던 기존 차트를 재구성한다. 특히 토큰 수와 산출물 수를 같은 "AI 사용량" bar로 비교하지 않고, 변경 파일 수와 라인 수를 같은 "코드 영향" bar로 비교하지 않게 한다.
  - 통계 페이지의 시간 컨텍스트를 명확히 한다. 누적 지표와 최근 스냅샷/이력 기반 지표를 카드 또는 차트 meta/detail에서 구분한다.
  - 데이터가 0인 카드와 차트에는 해당 지표를 채우는 다음 이벤트를 한 줄 안내로 표시한다.
  - hover/title 또는 기존 로컬 UI 패턴으로 정확한 숫자와 지표 정의를 확인할 수 있게 한다.
- Out of Scope:
  - `packages/cli/metrics-project.sh`, `.autoflow/telemetry/runs.jsonl`, metrics snapshot schema, runner telemetry 기록 방식 변경.
  - PRD-129에서 완료한 토큰 집계 경로 재수정 또는 과거 telemetry backfill.
  - 통계 페이지 밖의 AI 진행 현황, 티켓, LLM 위키, essential 대시보드 레이아웃 변경.
  - 새 metrics 필드 추가/삭제. 필요한 값은 기존 `board.metrics`, `board.metricsHistory`, `board.tickets`, `board.runners`에서 계산한다.
  - `git push`.

## Global Acceptance Criteria

- [ ] `ReportingDashboard` 상단 primary KPI는 완료 티켓, 검증 통과율, 막힌 항목, 토큰 사용량 4개가 먼저 보이며, 보조 지표는 별도 보조 카드/차트 영역으로 내려간다.
- [ ] 통계 화면에서 사용자가 보는 주요 라벨은 내부 용어보다 상태 이해 중심이다. 예: `인수인계` 단독 라벨은 남기지 않고 `PRD 처리` 또는 `전달된 요청`처럼 바꾼다.
- [ ] 통계 화면의 숫자 표시는 `formatCompactCount` 또는 기존 formatter를 일관되게 사용하며, detail/meta/title에서 정확값과 단위(`개`, `줄`, `토큰`, `%`, `커밋`)가 확인된다.
- [ ] 기존 "AI 사용량" 차트처럼 토큰 수와 산출물 수를 같은 bar scale로 비교하는 UI가 남지 않는다.
- [ ] 기존 "코드 영향" 차트처럼 변경 파일 수와 추가/삭제 라인 수를 같은 bar scale로 비교하는 UI가 남지 않는다. 파일 수는 별도 숫자 지표로, 추가/삭제 라인은 같은 단위의 split/stacked 표현으로 확인된다.
- [ ] `rejectCount`, `runnerBlocked`, blocked/needs_user 티켓 신호를 조합한 "막힌 항목" 지표가 상단 KPI에 표시되고, 0보다 크면 주의 톤을 사용한다.
- [ ] `metricsHistory` 또는 현재 snapshot만 있는 경우를 모두 처리하며, 추세/기간 meta가 `이번 7일`, `최근 스냅샷`, `전체 누적` 중 실제 의미에 맞게 표시된다.
- [ ] 데이터가 0인 검증, 토큰, 코드 영향, 막힘 관련 영역은 `검증이 완료되면 채워집니다`, `러너 실행 후 채워집니다`처럼 다음 이벤트를 설명하는 fallback 문구를 표시한다.
- [ ] `apps/desktop/src/renderer/main.tsx`와 `apps/desktop/src/renderer/styles.css` 밖의 product 파일은 수정하지 않는다.
- [ ] `npm run desktop:check` exits 0.
