## Core Scope

- Goal: 데스크톱 앱 메인 프로세스가 V8 heap OOM 또는 RSS 급증으로 갑자기 종료되기 전에 메모리 사용량을 주기적으로 감시하고, 임계치 도달 시 기존 종료 정리 경로를 거쳐 `app.relaunch()` + `app.exit(0)` 으로 graceful self-restart 를 수행한다.
- In Scope: `apps/desktop/src/main.js` 안에 메모리 점검 interval, 환경변수 기반 설정, RSS/heapUsed OR 임계치 판단, restart cool-down, restart 직전 1줄 로그, 기존 detached runner 종료 정리 경로와의 연결을 추가한다.
- Out of Scope: 메모리 누수의 근본 원인 분석/수정, 외부 watchdog 또는 L3 health monitor runner, renderer UI 변경, BrowserWindow launch/dev 플로우 재설계.

## Global Acceptance Criteria

- [ ] `apps/desktop/src/main.js` 에서 `AUTOFLOW_DESKTOP_MEMORY_CEILING_DISABLED=1` 일 때 메모리 감시 interval 이 시작되지 않는다.
- [ ] `AUTOFLOW_DESKTOP_MEMORY_CEILING_MB` 기본값 1500, `AUTOFLOW_DESKTOP_MEMORY_CHECK_INTERVAL_SECONDS` 기본값 30, `AUTOFLOW_DESKTOP_MEMORY_RESTART_COOLDOWN_SECONDS` 기본값 300 을 읽고, 유효하지 않은 값은 기본값으로 처리한다.
- [ ] `process.memoryUsage()` 의 `rss` 또는 `heapUsed` 중 하나가 임계치 MB 이상이면 restart 사유와 `rss`/`heapUsed` MB 값을 포함한 1줄 로그를 남긴다.
- [ ] 임계치 도달 시 기존 앱 종료 정리 경로를 재사용해 detached runner 정리를 시도한 뒤 `app.relaunch()` 와 `app.exit(0)` 을 순서대로 호출하는 코드 경로가 존재한다.
- [ ] restart trigger 후 cool-down 기간 안에는 같은 메모리 임계치 조건으로 즉시 중복 restart 를 요청하지 않는다.
- [ ] `npm run desktop:check` 가 exit 0 으로 통과한다.
