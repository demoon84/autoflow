## Project

- ID: prd_049
- Name: Demote runtime bookkeeping notes
- Title: Move low-value append_note runtime bookkeeping out of AI Notes context
- Goal: Reduce recurring ticket-owner adapter prompt noise by keeping decision-relevant AI notes in `## Notes` while moving repetitive runtime bookkeeping events to a capped `## Runtime Log` or equivalent runtime-only field.
- AI: planner-1
- Status: generated

## Core Scope

- Goal: Change board runtime note handling so repeated machine bookkeeping from `append_note` call sites no longer grows the AI-facing `## Notes` section on every ticket-owner tick.
- In Scope:
  - Add or reuse a helper in `common.sh` for runtime-only ticket events that writes outside `## Notes`, preferably to a capped `## Runtime Log` section or single last-machine-action field.
  - Route the low-value bookkeeping events called out by the memo away from AI-facing `## Notes`, including worktree dependency hydration, auto-block/recovery bookkeeping, auto-resume finish-pass bookkeeping, reject archival/replan bookkeeping, and ticket-owner prepared resume/todo entries.
  - Keep decision-relevant AI findings, verification outcomes, blockers, merge blockers, and implementation progress notes in `## Notes`.
  - Keep source runtime and installed board script copies synchronized.
- Out of Scope:
  - Do not change ticket lifecycle, stage transitions, claim semantics, verification semantics, merge/finalization behavior, or wiki update behavior.
  - Do not remove runtime bookkeeping entirely; it must remain inspectable in a runtime-only location.
  - Do not alter adapter prompt assembly outside what naturally follows from moving runtime-only lines out of `## Notes`.
  - Do not modify product/UI code.
