# Project PRD

## Meta

- Project Key: prd_096
- Title: Planner 오케스트레이터 역할 표시
- Status: populated

## Goal

데스크톱 AI 대쉬보드에서 `planner-1` 의 역할이 단순 "Plan AI"(계획만 담당)처럼 보이는 라벨/안내 문구를 정리해, 현재 3-runner 토폴로지에서 Plan AI가 동시에 보드 오케스트레이터(메모/PRD 승격, reject 재계획, 회복 지시) 역할도 한다는 사실이 사용자에게 분명히 드러나도록 한다. 변경은 사람이 읽는 라벨/카피/도움말에 한정하며, runner id (`planner-1`), runtime role 키 (`planner`/`plan`), parser 계약, 보드 runtime 출력은 전혀 바꾸지 않는다.

## Core Scope

### In Scope

- `apps/desktop/src/renderer/main.tsx` 의 `runnerRoleLabels` 에서 `planner` / `plan` 키의 한국어 표기를 오케스트레이터 의미가 드러나도록 정렬한다 (예: `Plan AI` → `Plan AI (오케스트레이터)` 같이 기존 "Plan AI" 식별성은 유지하면서 오케스트레이터 역할을 병기). `ticket-owner`(Impl AI), `wiki-maintainer`(Wiki AI) 라벨은 회귀 없이 그대로 둔다.
- `apps/desktop/src/renderer/main.tsx` 의 `displayProgressRoleLabel` 에서 `planner`/`plan` 분기가 반환하는 `"Planner"` 짧은 라벨을 한국어 + 오케스트레이터 의미가 보이는 표기로 정리한다 (예: `오케스트레이터` 또는 `오케스트레이터 (Planner)`). `worker` / `위키봇` 분기는 유지한다.
- `apps/desktop/src/renderer/main.tsx` 의 AI 설정/대쉬보드 화면 카피 중 Plan AI 역할만 단순 노출하는 부분을 보정한다: `runner-page-toolbar` 우측의 `Plan AI / Impl AI / Wiki AI` 안내 카피와, runner 카드가 비었을 때의 빈 상태 문장(`Plan AI(planner) / Impl AI(ticket-owner) / Wiki AI(wiki-maintainer) runner가 추가되면 여기에 표시됩니다.`) 을 Plan AI의 오케스트레이터 역할을 함께 드러내는 문구로 바꾼다.
- `apps/desktop/src/renderer/main.tsx` 의 `plannerFlowStages` meta(예: `backlog/reject 감시`, `PRD 분해 / 재계획`, `todo 생성 완료`) 가 이미 오케스트레이션 의미를 일부 담고 있으므로, 라벨/메타가 다른 수정과 일관되게 보이도록 필요 시 표현만 미세 조정한다 (역할 구조/키/스테이지 키 변경 금지).
- `apps/desktop/src/renderer/styles.css` 에서 위 라벨이 길어졌을 때 runner 카드/툴바/empty state 의 한 줄 영역(예: `runner-title-line`, `ticket-workspace-tab-copy`, `ai-progress-empty`) 이 줄바꿈/잘림으로 깨지면 wrap, ellipsis, `min-width: 0`, `flex-wrap`, `gap` 같은 최소 보정만 추가한다. 새 클래스/그리드 구조/색상 토큰은 도입하지 않는다.

### Out of Scope

- runner id (`planner-1`, `owner-1`, `wiki-1`), runtime role 키 (`planner`/`plan`/`ticket-owner`/`owner`/`wiki-maintainer`/`wiki`), runner state 파일 이름 변경.
- AGENTS.md / CLAUDE.md / `.autoflow/agents/*` / `protocols/*` / `reference/*` / scaffold 문서 변경.
- 보드 runtime 스크립트 출력 형식, parser 계약, key=value 토큰, ticket 필드 라벨 변경.
- `Impl AI`(`ticket-owner`/`owner`) 와 `Wiki AI`(`wiki-maintainer`/`wiki`) 의 표시 문구 자체 변경.
- 사이드바 메뉴 항목, 라우팅, 탭 구성, 페이지 레이아웃, 컴포넌트 추가/제거.
- runner enable/select 토글, 시작/중지 버튼, 키보드 포커스 흐름 변경.
- MUI ↔ shadcn 마이그레이션 확장, 신규 lucide 아이콘 도입.
- 새로운 테스트/smoke 추가.

## Global Acceptance Criteria

- [ ] AI 설정/대쉬보드에서 `planner-1` 카드의 역할 라벨이 "Plan AI" 만 단독 노출하지 않고, 오케스트레이터 역할이 함께 드러나는 한국어 표기로 보인다.
- [ ] `displayProgressRoleLabel` 이 planner role 에 대해 반환하는 짧은 라벨이 `"Planner"` 영문 단독에서 오케스트레이터 의미가 보이는 한국어(혹은 한국어 + 영문 보조) 표기로 바뀌어 진행 상태 카드/뱃지에 일관되게 노출된다.
- [ ] AI 설정 페이지 우측의 역할 안내 카피와 runner 가 없는 빈 상태 문구가 Plan AI 의 오케스트레이터 역할을 함께 설명한다.
- [ ] runner id (`planner-1`), runtime role 키 (`planner`/`plan`), runner state 파일 이름, board runtime 출력(`autoflow run planner` 등)에는 변화가 없다.
- [ ] `Impl AI` / `Wiki AI` 카드의 라벨, 진행 상태 표시, 안내 문구는 회귀 없이 그대로 유지된다.
- [ ] 라벨 길이가 늘어나도 runner 카드/툴바/빈 상태 영역에서 텍스트가 카드 밖으로 넘치거나 우측 시작/중지 버튼과 겹치지 않는다.
- [ ] `cd apps/desktop && npx tsc --noEmit` 가 통과한다.
- [ ] `cd apps/desktop && node scripts/check-syntax.mjs` 가 통과한다.
