- AI: planner

## Core Scope

- Goal: `.autoflow/runners/logs/` 의 무한 누적과 readBoard / metrics-project 부하를 막기 위해 소비자 없는 로그 영구 저장 경로를 닫고, loop tail rotation 을 도입하고, 누적된 644MB 잔재를 1회성으로 정리한다. 향후 telemetry layer (PRD-B) 가 들어오기 전에도 안정 상태가 약 30MB 로 수렴하게 한다.
- In Scope:
  - `packages/cli/run-role.sh` 의 `persist_run_artifact "$prompt_file" "prompt"` / `"$preflight_output" "runtime"` / `"$dry_run_output" "dry-run"` 호출과 그 경로를 state 에 기록하는 라인을 제거하거나 no-op 으로 바꾼다.
  - `packages/cli/run-role.sh` 의 stderr persist 경로를 stderr 파일 크기가 0 byte 인 경우 skip 하도록 분기한다.
  - `packages/cli/runners-project.sh` 의 `<runner>.loop.{stdout,stderr}.log` write 경로에 1MB cap 도달 시 `.1` -> `.2` 로 밀고 오래된 `.2` 는 삭제하는 rotation 을 추가한다.
  - tick 시작 직전에서 같은 runner 의 `_live_stdout.log` / `_live_stderr.log` / 타임스탬프 포함 `_last_message.txt` 중 mtime 이 `AUTOFLOW_LIVE_LOG_STALE_AGE_SECONDS` (기본 3600) 이상이고 동일 runner 의 active loop PID 가 살아있지 않은 잔재를 삭제하는 sweep 을 추가한다.
  - 신규 1회성 스크립트 `packages/cli/cleanup-runner-logs.sh` 를 추가한다. 인자: `<project-root> <board-dir-name>`. 동작: `*_stdout.log`, `*_stderr.log`, `*_prompt.log`, `*_runtime.log`, `*_dry-run.log`, `*_live_*.log`, 타임스탬프 포함 `*_last_message.txt` 를 삭제한다. `*.loop.{stdout,stderr}.log{,.1,.2}` 는 보존한다. `deleted_count=`, `freed_bytes=` key=value 출력 후 exit 0.
  - `bin/autoflow` 의 dispatcher 에 `cleanup-runner-logs` 서브커맨드 라우팅을 추가한다.
- Out of Scope:
  - `.autoflow/telemetry/` 도입, `runs.jsonl`, `failures.jsonl` (PRD-B).
  - Wiki AI synthesis 입력 변경 (PRD-C).
  - `metrics-project.sh` telemetry 기반 재작성 (PRD-D).
  - desktop renderer UI 변경.
  - 메모리/heap watchdog (별건 prd_119 / tickets_118).
