## Core Scope

- Goal: 데스크톱 앱 종료 이후 runner 4개가 detached 상태로 계속 ticking 중인 상황에서, 다음 앱 실행이 기존 runner pid/state를 안전하게 인식하고 UI monitoring/control을 복구하며 중복 loop-worker 생성을 방지한다.
- In Scope:
  - `apps/desktop/src/main.js`의 앱 시작 및 runner list/control 경로가 `.autoflow/runners/state/*`의 `pid`, `status`, `mode`, `enabled` 값을 읽어 살아 있는 loop runner를 기존 세션으로 attach 하도록 정리한다.
  - 데스크톱에서 `start` 요청을 보낼 때 CLI가 `result=already_running`을 반환하면 실패나 새 spawn으로 취급하지 않고 성공 상태로 반영하며, runner list cache를 즉시 refresh 한다.
  - 앱 종료 정책은 기본값을 "detached runner 유지"로 두고, 사용자가 명시적으로 선택한 경우에만 graceful stop을 호출한다. 메모리 ceiling relaunch처럼 앱 자체가 restart를 수행하는 내부 경로는 기존 cleanup 의도를 보존하되 일반 창 닫기/Cmd+Q 정책과 구분한다.
  - `apps/desktop/src/renderer/main.tsx`에 기존 runner 재연결 상태와 종료 정책 선택을 사용자에게 노출한다. UI는 shadcn 스타일의 로컬 컴포넌트와 `lucide-react` 아이콘을 우선 사용한다.
  - 앱 시작 시 이전 데스크톱 세션이 clean shutdown marker 없이 끝난 경우, runner를 죽이거나 재시작하지 않고 UI/로그에 "desktop exit was not clean; detached runners reattached" 수준의 증거를 남긴다.
  - smoke test를 추가해 fake board/state + fake `autoflow` CLI fixture에서 alive pid가 있는 runner를 list/start 했을 때 `already_running`이 success로 처리되고, 중복 start 호출 수가 0 또는 1회로 제한되는지 검증한다.
- Out of Scope:
  - 현재 호스트의 detached runner process를 planner나 ticket owner가 직접 kill/stop/restart 하는 작업.
  - macOS `log show` / DiagnosticReports 자동 수집을 제품 코드에 넣는 작업. 이번 ticket은 앱 내부 상태 marker와 runner state 재연결까지만 다룬다.
  - runner loop, adapter timeout, token budget, IPC lock, fork-bomb 원인 후보 전체 재설계.
  - `.autoflow/runners/state/` 운영 파일 직접 rewrite, 기존 runner state 삭제, worktree cleanup, `git push`.

## Global Acceptance Criteria

- [ ] With a fixture board containing enabled loop runners whose state files have alive `pid` values, desktop runner list/start logic treats those runners as reconnected/attached and does not spawn duplicate `loop-worker` processes.
- [ ] If the desktop sends `autoflow runners start <runner>` and stdout contains `status=ok` plus `result=already_running`, the renderer sees a successful running runner state and runner list cache refreshes before the next displayed status.
- [ ] A normal desktop close/Cmd+Q path preserves detached runners by default; graceful stop is called only after an explicit user-selected close policy, and that policy is visible in the renderer UI.
- [ ] A previous desktop session without a clean shutdown marker is reported as an unclean desktop exit with detached runner reattach evidence; the app does not kill, restart, or delete runner state as part of that report.
- [ ] Existing memory ceiling relaunch behavior remains separate from user close policy and still has a bounded cleanup timeout before `app.relaunch()`.
- [ ] `npm --prefix apps/desktop run check`, `bash -n packages/cli/runners-project.sh`, and `bash tests/smoke/desktop-detached-runner-reconnect-smoke.sh` exit 0.
