## Core Scope

- Goal: 실시간 감시에서 발견된 token usage stale 상태를 복구해 데스크톱 통계와 runner 카드가 최근 adapter 호출의 token 입력/출력 값을 다시 표시하도록 한다.
- In Scope:
  - `packages/cli/run-role.sh` 와 `runtime/board-scripts/run-role.sh` 의 adapter 완료 경로가 성공/실패/timeout tick 이후 token 입력/출력 값을 `.autoflow/telemetry/runs.jsonl` 에 기록하는지 확인하고, 누락된 finalize/record path 를 보정한다.
  - `packages/cli/telemetry-project.sh` 의 `record` / `token-usage` 경로가 `token_input` / `token_output` 을 양수 integer 로 보존하는지 fixture 로 검증한다.
  - `packages/cli/metrics-project.sh` 가 `.autoflow/telemetry/runs.jsonl` 를 source 로 사용해 `autoflow_token_usage_count` 와 `autoflow_token_report_count` 를 산출하는지 확인하고, stale `.autoflow/metrics/token-cache.tsv` 나 오래된 `.autoflow/metrics/telemetry-runs.jsonl` 언급이 현재 집계를 가리지 않도록 정리한다.
  - `apps/desktop/src/main.js` 의 `readRunnerTokenUsage` / `aggregateLiveTokenUsage` 가 telemetry totals 를 우선하고, token-cache fallback 과 live log additive path 가 최신 telemetry 값을 덮어쓰지 않도록 보정한다.
  - `tests/smoke/metrics-token-usage-smoke.sh` 또는 동등한 smoke test 에 PRD_129 회귀 fixture 를 추가한다: stale token-cache 와 fresh telemetry row 가 함께 있을 때 metrics/desktop aggregation 이 fresh telemetry 를 양수로 집계해야 한다.
- Out of Scope:
  - 과거 24시간 누락 telemetry row backfill.
  - token 비용 환산, chart redesign, ABA 분석 대시보드 추가.
  - 외부 모니터링 스크립트 수정.
  - runner / OS process 직접 정리, `git push`.
