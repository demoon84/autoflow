## Core Scope

- Goal: `RUNNER_QUOTA_KEYWORD_PATTERN` 기반의 단일 quota toast 판별을 rate-limit 전용 패턴과 quota-exhausted 전용 패턴으로 분리하고, LiveTerminalView 하단 sticky toast copy 를 신호 유형에 따라 다르게 보여준다.
- In Scope:
  - `apps/desktop/src/renderer/main.tsx`
    - `rate limit`, `too many requests`, `429` 를 감지하는 패턴과 `usage limit`, `quota exceeded`, `resource_exhausted`, `model_capacity_exhausted` 를 감지하는 패턴을 분리한다.
    - toast signal 계산부가 두 유형 중 어떤 신호인지 식별하고 fingerprint/retryAfter 를 계속 제공하도록 정리한다.
    - rate limit 일 때 제목은 `API 속도 제한`, 본문은 `일시적 제한입니다. 잠시 후 자동으로 재시도됩니다.` 를 기본으로 하고, retry-after 가 있으면 이어서 표시한다.
    - quota 소진 일 때 제목은 `사용 한도 소진`, 본문은 `{agent} 한도가 소진됐습니다. 모델을 교체하거나 한도 회복을 기다려주세요.` 로 표시한다.
    - 기존 dismiss(X) 버튼과 동일 fingerprint 동안 재표시 억제 동작은 유지한다.
- Out of Scope:
  - `apps/desktop/src/renderer/styles.css` 또는 다른 데스크톱 스타일 조정.
  - runner state / adapter / backend 의 rate-limit 재시도 로직 변경.
  - 새 IPC, 새 의존성, quota 계산 방식 추가.
