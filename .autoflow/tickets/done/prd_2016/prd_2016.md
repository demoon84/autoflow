## Core Scope

- Goal: Reduce per-tick token spend of the Wiki AI loop and move our wiki closer to the upstream LLM-Wiki pattern, by (a) sending only changed pages to the semantic-lint adapter, (b) emitting cheap orphan / broken-link / missing-frontmatter findings without any adapter call, (c) capping the lint prompt with an explicit byte budget, and (d) persisting good `wiki query --synth` answers back into the wiki so explorations compound.
- In Scope:
  - Per-page fingerprint cache for `wiki lint --semantic` so unchanged pages are excluded from the adapter prompt while still preserving the existing whole-wiki "skipped_unchanged" early-exit. State lives under `.autoflow/runners/state/<runner>.semantic-lint.fingerprint.d/` (one file per page).
  - Deterministic pre-check pass inside `run_semantic_lint` (and reachable via `wiki lint` without `--semantic`) that emits, without calling any adapter, the keys `lint_orphan.<n>.page=`, `lint_broken_link.<n>.page=` + `lint_broken_link.<n>.target=`, and `lint_missing_frontmatter.<n>.page=`. These run on every tick regardless of fingerprint state and run before any adapter call.
  - Explicit prompt byte budget controlled by env `AUTOFLOW_WIKI_LINT_PROMPT_BYTES` (default `32768`). When the assembled prompt would exceed the budget, the script trims pages from the tail and emits a `semantic_truncated=true` line plus `semantic_dropped_pages.<n>=<board-relative path>` lines in the output. The default applies in all callers, including `run-role.sh` and the wiki-1 loop runner.
  - New `--save-as <slug>` flag on `wiki-project.sh query --synth` that writes the answer to `.autoflow/wiki/answers/<slug>.md` with YAML frontmatter (`kind: synth_answer`, `created`, `updated`, `terms`, `citations`) and the synth answer body. The synth call still emits the same `synth_*` key=value lines on stdout. Re-running with the same slug updates `updated:` and the body in place rather than duplicating the file.
  - `.autoflow/wiki/answers/` directory (new) with a `README.md` describing it as the home for filed-back synthesis answers, and an `answers/` group added to the human-readable navigation of `.autoflow/wiki/index.md` outside any `AUTOFLOW:BEGIN`/`AUTOFLOW:END` managed block.
  - Update `.autoflow/agents/wiki-maintainer-agent.md` and `.autoflow/reference/wiki.md` to describe the deterministic pre-checks, the per-page fingerprint behavior, the prompt-byte budget env, and the `--save-as` flow. `grep -n AUTOFLOW_WIKI_LINT_PROMPT_BYTES .autoflow/agents/wiki-maintainer-agent.md` returns at least one hit after this PRD lands.
  - Add a debug-only env hook `AUTOFLOW_WIKI_LINT_DEBUG_PROMPT_PATH=<file>` that, when set, copies the assembled adapter prompt to that file so the verifier can confirm only changed pages are included.
- Out of Scope:
  - Full YAML frontmatter retrofit across existing wiki pages. Only `wiki/answers/*.md` get frontmatter in this PRD; existing pages remain as-is and are surfaced via `lint_missing_frontmatter.*` so a follow-up PRD can fix them.
  - A new immutable `raw/` sources directory and any `wiki ingest` command — that is a separate PRD.
  - Per-page digest sidecar files. The byte-budget trim is enough for now.
  - Any change to `update-wiki.sh` (deterministic baseline path stays untouched).
  - Any change to the desktop UI.

## Notes

- Token cost rationale: today every wiki edit invalidates the whole-wiki fingerprint, sending all ~30 page heads (~2.4k lines) to the adapter on every changed tick. After this PRD only the changed page + its inbound-link neighbors go in the prompt, the rest is dropped, and a hard byte budget truncates pathological cases. Deterministic orphan / broken-link / missing-frontmatter checks are emitted for free, removing whole categories of work from the adapter.
- Risk: the per-page fingerprint state directory is new under `.autoflow/runners/state/`. If wiki-1's runner id changes, the cache is invalidated and the next run reverts to "everything changed". Acceptable — it is a cache, not a source of truth.
- Risk: `--save-as` writes inside `.autoflow/wiki/answers/` which is not currently in the wiki-1 agent's expected output set. The agent doc update in this PRD must list it explicitly so future maintainers do not delete it as orphan content.
- Risk: a too-tight `AUTOFLOW_WIKI_LINT_PROMPT_BYTES` value can cause every page to be dropped. The default `32768` was chosen so the current 30-page wiki (≈ 17 KB at 80 lines/page) fits comfortably, with headroom for growth.
- Follow-up PRDs to consider after this one (out of scope here): (a) full frontmatter retrofit across existing pages once `lint_missing_frontmatter.*` is wired; (b) raw-sources directory + `wiki ingest <file>` command for the upstream LLM-Wiki ingest workflow; (c) per-page digest sidecar to further shrink prompts.
- Heartbeat: planner-1 and owner-1 already loop on a 1-minute tick. After this PRD is saved, the planner loop turns it into a `tickets/todo/` ticket on the next tick, and the worker loop claims it. No additional heartbeat is needed.
