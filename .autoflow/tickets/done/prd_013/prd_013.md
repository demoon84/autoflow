# Project PRD

## Project

- Title: Hoist code-volume + token-usage stats above the 작업 흐름 ticket board
- AI:
- Status: draft

## Core Scope

- Goal: 데스크톱 사이드 메뉴의 `작업 흐름` 페이지에서 TicketBoard ("PRD 바") 위로, `통계` 페이지에 이미 노출되는 두 지표 — Autoflow 누적 변경 코드량과 누적 토큰 사용량 — 을 한 줄짜리 stat strip 으로 항상 노출한다. 사용자가 작업 흐름을 볼 때마다 별도 페이지로 이동하지 않고 진행 비용/규모를 즉시 인지할 수 있게 한다.
- In Scope:
  - `apps/desktop/src/renderer/main.tsx` 의 `progress` (작업 흐름) settingsSection 분기에서, `<section className="board-section ... 코덱스 작업 흐름">` 위로 새 stat strip 컴포넌트를 삽입.
  - 표시 항목 (board.metrics 에 모두 존재):
    1. **변경 코드량**: `autoflow_code_files_changed_count` (변경 파일 수) + `autoflow_code_insertions_count` / `autoflow_code_deletions_count` (라인 +/-) — 한 칸에 압축 표기 (예: `+13,897 / −1,861 라인 · 파일 75`)
    2. **토큰 사용량**: `autoflow_token_usage_count` (compact 포맷, 예: `33.4M`) + `autoflow_token_report_count` (실행 로그 N개) — 두 번째 칸
  - `통계` 페이지의 `ReportingDashboard` 와 같은 데이터 소스(`board.metrics`) + 같은 포맷터(`formatCount`, `formatCompactCount`)를 재사용해 두 페이지 사이의 숫자가 항상 일치하도록 한다.
  - shadcn-first 규칙 준수: 새 컴포넌트는 `Badge`, `Card`, 또는 단순 `<div>` + Tailwind 유틸로 구성. modal/popover/tooltip 같은 인터랙션은 사용하지 않음.
  - 스타일: `apps/desktop/src/renderer/styles.css` 에 stat strip 클래스(예: `.workflow-stat-strip`) 추가. 기존 `.report-metric-grid` 보다 가벼운 한 줄짜리 톤.
- Out of Scope:
  - 통계 페이지 자체의 레이아웃 변경 (하단 차트, BoardSearch, MetricsHistory 등은 그대로).
  - 신규 메트릭 수집(예: 시간당 토큰, AI 별 분리). 현재 metrics 에 있는 값만 노출.
  - 토큰 비용을 통화 환산하기.
  - `티켓 정보` / `AI 관리` / `Wiki` / `통계` 등 다른 사이드 페이지에 동일 strip 추가.

## Notes

- 기존 ReportingDashboard 의 `codeImpactData`, `aiUsageData` 구성을 함수로 추출해 두 페이지가 공유하면 drift 위험을 더 줄일 수 있다 (선택적 리팩터, 본 티켓에서 함께 처리해도 무방).
- strip 높이는 한 줄(약 48px)로 제한해 TicketBoard 가 화면을 잡아먹는 일이 없어야 한다.
- 다국어/숫자 포맷은 기존 `formatCount` / `formatCompactCount` 그대로 사용 (한국어 포맷에 맞춰져 있음).
- 토큰 정보가 0 인 경우(`autoflow_token_usage_count=0`, `autoflow_token_report_count=0`) 에는 strip 의 토큰 칸을 회색 톤으로 표시해 "데이터 없음" 임을 시사.
