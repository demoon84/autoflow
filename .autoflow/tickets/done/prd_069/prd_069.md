## Core Scope

- Goal: 데스크톱 "대화" 화면을 claude.ai 스타일의 읽기 좋은 메시지 레이아웃으로 다듬고, 같은 화면에서 이미지 첨부(드래그&드롭/파일 선택)가 가능하게 한다. 첨부된 이미지는 보드의 conversations attachments 폴더에 저장되어 영속 thread, memo/PRD 미리보기, 어댑터 프롬프트 모두에서 일관되게 보인다.
- In Scope:
  - **메시지 레이아웃 재설계**: 메시지 영역에 max-width(기본 760px) + 가운데 정렬, 사용자 메시지는 우측 정렬 + 작은 회색 둥근 버블, AI 메시지는 좌측 정렬 + 박스 없이 흐르는 본문 + 아바타.
  - **아바타**: 사용자=이니셜 원형 (이름 첫 글자, fallback "나"), AI=`Sparkles` 아이콘 또는 `assets/agent-icons/claude.png` 같은 도메인 자산을 둥근 배경에 올린 형태. 아바타 크기 28~32px, 본문과 8px 간격.
  - **상대 시간 표시**: ISO timestamp 대신 "방금"/"N분 전"/"오늘 오후 6:56"/"어제"/"YYYY-MM-DD" 룰. 호버하면 풀 timestamp 가 title tooltip 으로 노출.
  - **메시지 spacing**: 메시지 사이 16~20px gap, 첫 메시지/마지막 메시지 padding 의 시각 균형, 빈 상태 일러스트 가운데 정렬.
  - **AI 응답 본문**: MarkdownViewer 가 코드 블록/링크/리스트를 렌더하고, 코드 블록은 모노폰트 + 옅은 배경.
  - **wiki 인용 칩**: 현재의 outline 칩을 라이트 톤으로 다듬고 메시지 아래 한 줄에 정렬.
  - **이미지 첨부**: 입력창 좌측에 paperclip 아이콘 버튼 + 입력창 영역 전체에 drag&drop. 파일 선택 시 입력창 위에 작은 미리보기 chip(썸네일+파일명+x 버튼) 노출.
  - **첨부 IPC 와 저장**:
    - `autoflow:chatPickImages` — Electron `dialog.showOpenDialog` 를 띄우고 이미지(png/jpg/jpeg/gif/webp/svg) 만 필터로 선택, 절대경로 배열을 반환.
    - `autoflow:chatAttachImages({sourcePaths})` — 선택된 파일을 `.autoflow/conversations/desktop-chat-attachments/<YYYYMMDDTHHMMSSZ>-<safeName>` 로 복사하고, 복사된 보드 상대 경로 배열을 반환.
    - 두 IPC 모두 확장자 화이트리스트와 파일 크기 제한(기본 8MB, env `AUTOFLOW_DESKTOP_CHAT_IMAGE_MAX_BYTES`) 을 검증한다. 절대경로가 attachments 디렉토리 밖이면 거부.
  - **메시지 본문 형식**: 첨부된 이미지가 있는 사용자 메시지는 markdown 이미지로 thread 에 저장된다. 예: `![attached](conversations/desktop-chat-attachments/<file>)\n\n<본문>`. memo/PRD 미리보기 본문도 같은 markdown 그대로 보존된다.
  - **렌더링**: 메시지 본문에 이미지 markdown 이 있으면 본문 영역에서 max-width 320px / max-height 240px 의 둥근 thumbnail 로 표시. 클릭하면 MUI Dialog 로 원본 크기(또는 뷰포트 한도) 까지 확장된 미리보기.
  - **어댑터 hint**: 첨부가 있는 사용자 메시지를 어댑터 프롬프트에 직렬화할 때 본문 끝에 한 줄 `[Attached image: <board-relative path>]` 를 붙여 multi-modal 어댑터가 인지하거나 텍스트 어댑터가 최소한 첨부 사실을 알 수 있게 한다.
  - **빈 상태**: 메시지 0개일 때 가운데에 작은 안내(아이콘 + 한 줄). 현재 톤 유지하되 spacing 만 정리.
  - **MUI 우선 정책**: 모달/툴팁/스낵바/IconButton 모두 MUI Material 컴포넌트 위에서 스타일만 확장. 메시지 버블은 도메인 특수 자체 구현 유지(접근성 ARIA `role="log"`, `aria-live="polite"` 보존).
  - 사용자 노출 라벨/안내/오류는 한국어.
- Out of Scope:
  - 비이미지 첨부(pdf, zip, 텍스트 등).
  - 이미지 편집/크롭.
  - 어댑터별 multi-modal protocol 정식 지원(첫 버전은 path text hint 만, claude/codex 가 이를 어떻게 해석하는지는 어댑터 측 책임).
  - 응답 토큰 단위 스트리밍.
  - provider 선택 드롭다운(별도 후속 PRD).
  - 다중 대화 세션, 세션 검색.

## Global Acceptance Criteria

- [ ] 채팅 메시지 영역에 max-width 가 적용되어 가운데 정렬되고, 사용자 메시지는 우측 정렬 회색 둥근 버블 + 좌측 본문, AI 메시지는 좌측 정렬 본문 + 좌측 아바타로 시각적으로 명확히 구분된다.
- [ ] 모든 메시지 헤더의 시각 표기는 상대 시간("방금"/"N분 전"/"오늘 오후 HH:mm"/"어제"/"YYYY-MM-DD") 이고, 호버 시 풀 ISO timestamp 가 title tooltip 으로 노출된다.
- [ ] 사용자/AI 아바타가 메시지 옆에 표시된다(사용자 이니셜 원형, AI 아이콘 또는 Claude 자산).
- [ ] 입력창 좌측에 paperclip 아이콘 버튼이 있고, 클릭하면 파일 다이얼로그가 열려 이미지 확장자(png/jpg/jpeg/gif/webp/svg) 만 선택 가능하다.
- [ ] 입력창 영역에 이미지 파일을 드래그&드롭하면 첨부 chip 으로 추가되고, x 버튼으로 제거할 수 있다.
- [ ] 전송 시 첨부된 파일이 `.autoflow/conversations/desktop-chat-attachments/` 로 복사되고, 메시지 본문은 그 상대 경로의 markdown 이미지로 thread (`desktop-chat.md`) 에 저장된다.
- [ ] 메시지 본문에 이미지 markdown 이 있으면 max-width 320px 둥근 thumbnail 로 렌더되고, 클릭하면 MUI Dialog 안에서 원본 비율로 큰 미리보기가 열린다.
- [ ] 어댑터 프롬프트에 첨부가 있을 때 사용자 메시지 본문 뒤에 `[Attached image: <보드 상대 경로>]` 라인이 붙어 직렬화된다(트레이스 또는 dry-run 으로 확인).
- [ ] 확장자 외 파일 또는 기본 8MB 초과 파일을 첨부하면 한국어 오류와 함께 거부된다.
- [ ] memo/PRD 미리보기로 저장된 본문에 이미지 markdown 이 있으면 그대로 보존된다(미리보기 다이얼로그에서 그대로 보임).
- [ ] `npm --prefix apps/desktop run check` 통과.
- [ ] `bash tests/smoke/desktop-chat-spec-intake-smoke.sh` 통과(추가 검증: `chatPickImages` / `chatAttachImages` IPC 노출, attachments 디렉토리 경로 prefix 검증 코드 존재, 어댑터 hint 직렬화 패턴, 메시지 헤더에 ISO timestamp 가 직접 출력되지 않음을 확인).
