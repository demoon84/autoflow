## Core Scope

- Goal: `packages/cli/run-role.sh` 의 Gemini adapter 실행 경로와 `apps/desktop/src/main.js` 의 token usage parser 를 좁게 보강해 Wiki runner token usage 가 desktop metrics 경로에 누락되지 않게 한다.
- In Scope:
  - Gemini CLI help 에 `--show-tokens` 옵션이 없고 `--output-format text|json|stream-json` 옵션만 있는 현재 환경을 기준으로, Gemini adapter 가 usage metadata 를 얻을 수 있으면 이를 표준 marker (`total_tokens=`, `input_tokens=`, `output_tokens=` 등) 로 stdout 에 남긴다.
  - Gemini CLI JSON/stream-json 출력에서 `usageMetadata.promptTokenCount`, `usageMetadata.candidatesTokenCount`, `usageMetadata.totalTokenCount` 또는 동등한 camelCase/snake_case key 가 나오면 desktop parser 가 이를 token usage 로 합산한다.
  - Gemini CLI 가 usage metadata 를 노출하지 않는 경우에도 wiki runner stdout 마지막에 보수적인 fallback marker 를 남겨 `token-cache.tsv` 신규 wiki 행의 token 값이 0 으로 고정되지 않게 한다.
  - 기존 Claude / Codex / OpenCode token marker 파싱과 live log aggregation 동작을 유지한다.
  - token 산출 방식과 fallback 여부를 ticket Notes 또는 verification evidence 에 남긴다.
- Out of Scope:
  - Gemini API SDK 직접 호출 모드 추가.
  - runner timeout 정책, wiki debounce 정책, wiki synthesis 내용 변경.
  - `.autoflow/metrics/token-cache.tsv` 과거 행 backfill.
  - telemetry `runs.jsonl` duration/token schema rename.
  - desktop renderer 화면 구조, 카드 레이아웃, runner lifecycle UI 변경.
