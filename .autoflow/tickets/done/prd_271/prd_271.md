## Goal

PTY 모드 운영 중 `wiki.state` 등 runner state 파일이 핵심 필드(`id`, `status`, `mode`, `pid`, `role`, `agent`)를 잃고 token 5필드만 남아 UI가 stopped로 오인하는 문제를 구조적으로 해결한다. `createClaudeTokenWatcher`의 race condition fix, `writePtyRunnerStateFile`의 defensive merge, Electron `powerMonitor` sleep/wake 처리, 주기적 state self-heal을 적용한다.

## Scope

### Allowed Paths

- `apps/desktop/src/main.js`
- `apps/desktop/src/main/runner-pty-manager.js`

### Global Acceptance Criteria

- token watcher publish() 가 spawn 완료 후에만 발화
- writePtyRunnerStateFile 신규 파일 생성 시 spawn 메타 자동 보강
- powerMonitor suspend/resume 처리 존재
- state self-heal 로직 존재 (AUTOFLOW_STATE_SELFHEAL_MIN, 기본 5분)
- `rg -n "powerMonitor|writePtyRunnerStateFile|createClaudeTokenWatcher" apps/desktop/src/main.js` 모두 결과 있음

## Execution Candidates

### Candidate 1: race fix + defensive merge + powerMonitor + self-heal 통합 적용

1. **race fix**: `createClaudeTokenWatcher` 첫 publish()를 spawn의 writePtyRunnerStateFile 완료 후에 발화하도록 setTimeout 또는 Promise 동기화.
2. **defensive merge**: `writePtyRunnerStateFile`이 새 파일 생성 시 `ptyRunnerMeta.get(runnerId)` spawn 정보(role/agent/mode/status/pid/started_at) 자동 채움.
3. **powerMonitor 통합**: Electron `powerMonitor` `suspend`/`resume` 이벤트 처리 — suspend 시 fs.watch 디바운스 잠시 정지, resume 시 PTY children kill-0 검증 후 state 파일 rewrite.
4. **state self-heal**: 매 N분 (`AUTOFLOW_STATE_SELFHEAL_MIN`, 기본 5) `__autoflowPtyManager.list()`와 state 파일 비교 후 핵심 필드 누락 시 자동 보강.
