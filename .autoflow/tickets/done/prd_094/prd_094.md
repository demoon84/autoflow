# Project PRD

## Meta

- Project Key: prd_094
- Title: Worker 행 라벨·토큰 사용량 flex 중앙 정렬
- Status: populated

## Goal

데스크톱 앱 AI 진행 보드의 Worker 행에서 아이콘, Worker 라벨, 토큰 사용량 텍스트, 우측 제어(시작/중지 버튼)가 같은 행에 어긋나지 않게 정렬되도록 라벨/토큰 사용량 영역을 flex로 묶고 세로 중앙 정렬을 적용한다. 긴 토큰 사용량 문자열이 들어와도 행 높이와 좌우 간격이 안정적으로 유지되어야 한다.

## Core Scope

### In Scope

- worker progress row(`ai-progress-row-worker`) 안에서 라벨(아이콘 + Worker 텍스트)과 토큰 사용량(`ai-progress-token-usage`)을 flex 컨테이너로 묶어 가로 정렬한다.
- 묶은 flex 컨테이너가 우측 제어(`ai-progress-actions`)와 세로 중앙 기준으로 정렬되도록 row-top 그리드의 `align-items` 또는 worker-row 전용 override를 적용한다.
- 토큰 사용량 텍스트가 길어져도 라벨이 잘리거나 행이 비대칭으로 늘어지지 않도록 wrap/ellipsis/`min-width: 0` 처리를 정리한다.
- 기존 비-worker progress row(track 표시 row)의 시각/동작은 유지한다.

### Out of Scope

- 진행 단계 트랙(`ai-progress-track`) 자체의 레이아웃 변경.
- Worker 행 외 헤더/구성 패널(`ai-progress-config`) 정렬 변경.
- 토큰 사용량 텍스트 포맷/문구 변경.
- runner enable/select 동작이나 시작/중지 버튼 핸들러 수정.
- MUI ↔ shadcn 마이그레이션 작업 확장.
- 테스트 추가, 신규 컴포넌트 도입.

## Global Acceptance Criteria

- [ ] Worker 행에서 아이콘, Worker 라벨, 토큰 사용량 텍스트, 우측 제어가 모두 같은 시각적 baseline 또는 세로 중앙 기준으로 정렬되어 어긋나 보이지 않는다.
- [ ] 라벨과 토큰 사용량이 flex 컨테이너로 묶여 있고, worker 행에서는 두 요소가 가로 행으로 자연스럽게 이어진다.
- [ ] 긴 토큰 사용량 문자열이 들어와도 Worker 행의 높이가 갑자기 두 배 이상으로 커지거나 좌우 패딩이 불안정해지지 않는다.
- [ ] 토큰 사용량이 비어 있을 때도 라벨 위치와 우측 제어 위치가 어긋나지 않는다.
- [ ] 비-worker progress row(planner/owner 등 track 노출 row)의 기존 정렬과 그리드 레이아웃은 회귀 없이 유지된다.
- [ ] runner enable/select 토글, 시작/중지 버튼 동작과 키보드 포커스 흐름은 그대로다.
