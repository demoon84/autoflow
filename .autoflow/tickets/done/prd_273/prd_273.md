## Goal

`update-wiki.sh` 에 두 가지 자동화를 추가한다. (1) 30일 이상 갱신 없는 learnings 파일을 wiki AI 가 archive 또는 superseded 마크하는 TTL 점검 루틴. (2) ticket 본문에서 동일 failure_class / root cause 가 `AUTOFLOW_WIKI_PATTERN_THRESHOLD`(기본 3)회 재발 시 패턴 draft 페이지를 자동 생성하는 루틴. 마지막으로, PRD_272 의 sources 정리 전/후 RAG 검색 품질을 비교한 증거 보고서 1쪽을 작성한다.

## Background

PRD_272 로 구조 정리가 완료된 뒤, 실제 update-wiki.sh 코드 수준에서 새 노이즈가 쌓이지 않도록 막고 장기 유용성을 보장하는 자동화 레이어가 필요하다. TTL 미적용 시 stale 페이지가 누적되어 RAG 결과를 다시 오염시킬 수 있다.

## Scope

- In Scope: `update-wiki.sh` TTL 점검 루틴, 패턴 합성 루틴, RAG 품질 보고서, runtime 미러
- Out of Scope: 구조 정리(sources 이동, index.md 재작성)는 PRD_272 에서 완료
