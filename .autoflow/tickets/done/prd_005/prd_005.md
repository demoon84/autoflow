# Project Spec

## Meta

- PRD Key: prd_005
- Title: Rename user-facing "spec" terminology to "PRD" (UI + docs + CLI alias)
- Status: populated

## Goal

Autoflow 의 핵심 산출물 명칭을 사용자 노출 면에서 "spec / 스펙" → "PRD" 로 통일한다. 데스크톱 UI, 에이전트 문서, 호스트 가이드(README/AGENTS/CLAUDE), CLI 헬프, 스캐폴드 미러까지 한국어/영어 사용자 텍스트를 PRD 로 정렬하고, `autoflow prd` 명령을 alias 로 추가해 사용자가 신/구 어떤 이름으로도 호출할 수 있게 한다. 파일 경로·내부 함수명·기존 CLI 서브커맨드는 backward-compat 을 위해 그대로 유지한다.

## Core Scope

### In Scope

- 데스크톱 UI 텍스트: `apps/desktop/src/renderer/main.tsx` 의 사용자 노출 "스펙" → "PRD" (예: 카드 라벨 `"스펙"`, `"${specTotal}개 스펙"` 등). 기존 도움말 spec(prd_004)이 적용된 시점이면 도움말 문구도 PRD 기준으로 일치.
- 에이전트 문서 (`.autoflow/agents/*.md` + `scaffold/board/agents/*.md` 미러):
  - `spec-author-agent.md`: 문서 내 본문에서 "spec" → "PRD" 변경. 파일명은 유지(시스템 트리거가 이 경로를 참조).
  - `ticket-owner-agent.md`, `plan-to-ticket-agent.md`, `todo-queue-agent.md`, `verifier-agent.md`, `wiki-maintainer-agent.md`: "spec" 단어가 산출물(=PRD)을 가리키는 문맥에서 PRD 로 수정. `spec-handoff`, `spec_id`, `start-spec.sh` 같은 식별자/파일명은 그대로.
- 호스트 가이드: `AGENTS.md`, `CLAUDE.md`, `README.md`, `scaffold/board/AGENTS.md`, `scaffold/board/README.md` — "spec" 이 산출물을 가리키는 문장을 "PRD" 로 다듬는다. `/af`·`/autoflow` 트리거 설명도 "spec handoff" → "PRD handoff" 로 표현.
- 레퍼런스 템플릿:
  - `reference/project-spec-template.md`, `reference/feature-spec-template.md` 의 1행 제목 `# Project Spec` / `# Feature Spec` 을 `# Project PRD` / `# Feature PRD` 로 변경. 파일명·섹션 키(`## Meta`, `## Goal` 등)는 유지(파서 호환).
- CLI alias 추가:
  - `bin/autoflow` 와 `bin/autoflow.ps1` 에 `prd` 디스패치 추가 → `packages/cli/spec-project.sh` / `spec-project.ps1` 로 위임. 출력/플래그 동일.
  - 헬프 텍스트는 `autoflow prd create` 를 1순위 표기, `autoflow spec create` 는 "(legacy alias)" 로 함께 노출.
- CLI 출력 메시지: `packages/cli/spec-project.sh` 의 사람-가독성 출력(예: `Spec saved at ...`, 헬프 usage 헤더)에서 "Spec" → "PRD" 로 표현. `status=ready_for_input` 같은 머신 키는 변경 없음.
- 스캐폴드 미러 동기화: `scaffold/board/agents/`, `scaffold/board/AGENTS.md`, `scaffold/board/README.md`, `scaffold/board/reference/` 를 `.autoflow/` 변경과 `diff -q` 동일하게 유지.
- 스킬 문서: `.claude/skills/autoflow/SKILL.md`, `.claude/skills/af/SKILL.md`, `.codex/skills/autoflow/SKILL.md`, `.codex/skills/af/SKILL.md`, `integrations/claude/skills/...`, `integrations/codex/skills/...` 의 사용자 노출 텍스트 동일 원칙으로 PRD 표현.

- 함수/변수명 변경: shell, JS/TS, agent runtime 의 `spec_file`, `spec_id`, `specTotal`, `extractSpecAllowedPaths` 등 그대로.

## Global Rules

- 사용자용 텍스트만 PRD 로 변경 — 파일명/식별자/머신 키는 절대 변경하지 않는다.
- `autoflow spec ...` 호출은 변경 후에도 동일하게 동작해야 한다.
- `autoflow prd ...` 가 추가되어 같은 동작을 보장한다.
- 기존 owner smoke (`tests/smoke/ticket-owner-smoke.sh`) 회귀 없음 — 특히 `status=created` 같은 머신 출력은 그대로.
- 한국어 UI 문구는 한국어 기본, AI/계약 문서는 영어 기본 (AGENTS.md rule 15).
