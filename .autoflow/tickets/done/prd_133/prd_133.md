## Core Scope

- Goal: `.autoflow/logs/` 의 outcome markdown 로그가 무한 누적되지 않도록 `packages/cli/cleanup-runner-logs.sh` 의 recurring cleanup 경로에 outcome log retention을 추가한다.
- In Scope:
  - `packages/cli/cleanup-runner-logs.sh` 가 기존 `.autoflow/runners/logs/` 와 `.autoflow/runners/state/` cleanup에 더해 `.autoflow/logs/` outcome markdown 로그도 처리한다.
  - verifier outcome 로그(`verifier_<id>_*_pass.md`, `verifier_<id>_*_fail.md`)는 filename timestamp 기준으로 hot root 보존 한도와 age cutoff를 적용한다. 기본 hot root 한도는 root `.md` 로그가 대략 50~100개 수준으로 수렴하도록 정하고, 오래되거나 한도를 초과한 파일은 `.autoflow/logs/archive/YYYY-MM/` 아래로 압축 또는 이동한다.
  - archive 된 outcome 로그는 filename timestamp 기준 90일 경과 시 삭제한다.
  - legacy/deprecated 패턴인 `coordinator_*_blocked.md`, `owner_*_blocked.md`, `branch-cleanup_*` 디렉터리는 현재 enabled runner topology에서 유효한 active runner 산출물이 아니므로 cleanup 실행 시 root에서 제거하거나 archive로 이동한다.
  - `manual_worktree_merge_*.md` 는 filename timestamp 기준 30일 retention을 적용한다. 30일 미만 파일은 유지할 수 있고, 30일 경과 파일은 archive 또는 삭제한다.
  - filename timestamp 파싱은 mtime에 의존하지 않는다. checkout으로 mtime이 갱신되어도 `20260426T...`, `20260427_...`, `20260503_...` 형태의 basename timestamp를 기준으로 판단한다.
  - cleanup output은 기존 `deleted_count=` / `freed_bytes=` key를 유지하고, 필요하면 outcome 전용 `outcome_archived_count=` / `outcome_deleted_count=` 같은 추가 key를 뒤에 붙인다.
- Out of Scope:
  - `.autoflow/wiki/` content 직접 수정 또는 wiki synthesis 실행.
  - ticket worktree cleanup, runner/OS process 관리, runner start/stop/restart.
  - 새로운 desktop UI.
  - verifier pass/fail 판정 로직 변경.
  - `.autoflow/logs/archive/` 내부의 장기 보관 포맷을 UI에서 탐색하는 기능.

## Global Acceptance Criteria

- [ ] `packages/cli/cleanup-runner-logs.sh` 가 `.autoflow/logs/` 의 verifier outcome markdown 로그를 filename timestamp 기준 hot root 한도와 age cutoff로 archive 또는 압축한다.
- [ ] cleanup 1회 실행 후 `.autoflow/logs/` root 에 남는 `*.md` 파일 수가 100개 이하로 수렴한다. archive 디렉터리 내부 파일은 이 root count에서 제외한다.
- [ ] cleanup 1회 실행 후 `.autoflow/logs/` root 에 `coordinator_*_blocked.md`, `owner_*_blocked.md`, `branch-cleanup_*` 디렉터리가 남아 있지 않다.
- [ ] `manual_worktree_merge_*.md` 는 filename timestamp 기준 30일 미만이면 보존 가능하고, 30일 이상이면 root에서 archive 또는 삭제된다.
- [ ] archive 된 outcome 로그는 filename timestamp 기준 90일 경과 시 삭제되는 경로가 구현되어 있다.
- [ ] `bin/autoflow cleanup-runner-logs "$PWD" .autoflow` 가 기존 `deleted_count=` / `freed_bytes=` 출력과 exit 0 계약을 유지한다.
- [ ] cleanup 명령은 두 번 연속 실행해도 두 번째 실행이 exit 0이며, 이미 정리된 root outcome 로그를 다시 실패시키지 않는다.
- [ ] `bash -n packages/cli/cleanup-runner-logs.sh` 가 exit 0으로 통과한다.
