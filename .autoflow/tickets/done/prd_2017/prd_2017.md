## Core Scope

- Goal: Close the biggest LLM-Wiki essay gap in our wiki — the missing **immutable raw-sources layer** plus an **ingest workflow** — without burning adapter tokens on unchanged sources. Adding the same source twice must call no adapter; adding a new or modified source must produce a YAML-frontmatter summary page under `wiki/sources/` with citations back to the raw file.
- In Scope:
  - New `.autoflow/wiki-raw/` directory + `README.md` describing it as the immutable raw-sources layer (the LLM reads from it, never writes back).
  - New `.autoflow/wiki/sources/` directory + `README.md` describing it as the wiki-side summary layer derived from `wiki-raw/`.
  - New `wiki-project.sh ingest [project-root] [board-dir-name] <source-file> [--slug SLUG] [--no-summary] [--runner RUNNER_ID]` action that:
    1. Computes the source's sha256.
    2. Resolves `<slug>` from `--slug` or by sanitising the source filename to `[a-z0-9_-]+`.
    3. Writes `wiki-raw/<slug>.md` with a YAML frontmatter prefix (`kind: raw_source`, `slug`, `original_path`, `ingested_at`, `updated_at`, `sha256`) followed by the source's verbatim body. If the file already exists with the same sha256, it is left alone (`ingest_status=skipped_unchanged_source`); a body change refreshes `updated_at` and the body but preserves `ingested_at`.
    4. Per-source fingerprint cache at `runners/state/<runner_id>.ingest.sources.d/<safe_slug>` containing the sha256 at the time of the last successful summary. If it matches the current sha256 and `wiki/sources/<slug>.md` already exists, no adapter is invoked (`ingest_summary_status=skipped_unchanged`).
    5. Otherwise, calls the wiki adapter with a prompt capped at `AUTOFLOW_WIKI_INGEST_PROMPT_BYTES` bytes (default `16384`). The body is truncated from the tail with an explicit `...[truncated]...` marker if it exceeds the budget. The prompt asks for a fixed key=value response (`source_title`, `source_one_line`, `source_summary`, `source_entity.<n>.{name,kind,note}`, `source_concept.<n>.{name,note}`).
    6. Writes `wiki/sources/<slug>.md` with YAML frontmatter (`kind: source_summary`, `slug`, `created`, `updated`, `raw_source`, `entities`, `concepts`) and a body containing `# title`, `## One-liner`, `## Summary`, `## Entities`, `## Concepts`, `## Source` (citation back to `wiki-raw/<slug>.md`). Re-running with a changed source preserves `created:` and refreshes `updated:`.
    7. Records the new fingerprint on success only.
  - `--no-summary` flag that copies into `wiki-raw/` but skips the adapter call (useful for batch ingest when the operator wants to summarize later).
  - Debug hook `AUTOFLOW_WIKI_INGEST_DEBUG_PROMPT_PATH` that copies the assembled adapter prompt to that path, mirroring the lint debug hook.
  - Lint extension: the existing `wiki/sources/*.md` pages are exempt from `lint_missing_frontmatter.*` because they are guaranteed to carry frontmatter. Orphan detection still runs but a `[[sources/<slug>]]` reference from `wiki/index.md` (added automatically when the first source is summarized) prevents orphans.
  - Index update: when `wiki/sources/<slug>.md` is created (not on update), append `- [[sources/<slug>]] (...)` under a new outside-managed-block `sources/` group in `wiki/index.md` if the entry does not already exist.
  - Update `.autoflow/agents/wiki-maintainer-agent.md` and `.autoflow/reference/wiki.md` to describe the new layer, the ingest workflow, and the env vars.
- Out of Scope:
  - Removing or renaming the existing `wiki/answers/` directory. Sources and answers coexist as two distinct file-back layers (sources = summaries of raw documents; answers = persisted synth answers to queries).
  - Image / binary handling. v1 is markdown / text only. Binary sources can be added in a follow-up that records only the metadata + a textual description.
  - Frontmatter retrofit on existing wiki pages — still deferred.
  - Any change to `update-wiki.sh` deterministic baseline.
  - Desktop UI changes.

## Conversation Handoff

- Source: 2026-04-29 `/autoflow` chat — user request to autonomously continue improving the wiki along the LLM-Wiki essay's pattern, prioritising what's most token-efficient and most faithful to the essay's idea.
- Summary: After `prd_2016` shipped per-page semantic-lint diff + synth save-back, the largest remaining gap is the essay's "raw sources" layer and "ingest" operation. This PRD adds an immutable `wiki-raw/` directory, a `wiki/sources/` summary layer, and a deterministic-first `wiki ingest <file>` command whose adapter call is gated by a per-source SHA cache and capped by an explicit byte budget. Together with `prd_2016`'s file-back of synth answers, the wiki now has the three operations the essay calls out (ingest / query / lint).

## Notes

- Token cost rationale: the essay's ingest workflow is the operation most prone to runaway token spend (each source can be long, and naively re-summarizing on every tick is wasteful). The per-source SHA cache means an unchanged corpus runs to completion without ever calling the adapter; only NEW or MODIFIED sources cost adapter tokens. The prompt byte budget bounds even those calls.
- Risk: slug collisions between unrelated sources. Mitigated by requiring `[a-z0-9_-]+` and rejecting whitespace, plus letting the operator pass `--slug` explicitly to disambiguate.
- Risk: wiki-raw file size. We keep verbatim body. For large sources the budget only affects what is sent to the adapter, not what is stored. If wiki-raw bloats, a follow-up can add a separate retention policy (out of scope).
- Risk: index.md auto-mutation. We append a sources/ navigation entry only on first creation of a given slug, outside any AUTOFLOW:BEGIN/END managed block. Idempotency check: do not append if the line is already present.
- Heartbeat: the existing 3-runner topology continues unchanged. Plan AI / Impl AI / Wiki AI are not required to actually run for this PRD's verification — the new `ingest` action is fully invokable from the CLI.
