## Core Scope

- Goal: `packages/cli/metrics-project.sh` 의 verifier pass/fail 집계를 raw file count 에서 reset/archive-safe latest outcome 집계로 바꿔 desktop UI 와 `autoflow metrics` 가 현재 시스템 건강을 왜곡하지 않게 한다.
- In Scope:
  - `metrics-project.sh` 가 `.autoflow/logs/verifier_*_{pass,fail}.md` outcome 로그를 ticket id 별 최신 outcome 기준으로 집계한다. 같은 ticket id 에 fail 이 여러 번 있고 이후 pass 가 있으면 최종 pass 1건만 반영한다.
  - `.autoflow/logs/archive/`, `.autoflow/logs/tickets-reset_*`, `.autoflow/logs/start-blocked-reset_*` 같은 reset/archive 하위 경로의 outcome 로그는 current metrics 에 포함하지 않는다.
  - root `.autoflow/logs/` 에 legacy `verifier_003_*_fail.md` 같은 파일이 남아 있어도 같은 ticket id 의 최신 outcome 또는 archive-safe 필터링 때문에 `verifier_fail_count` 가 raw legacy fail 개수만큼 증가하지 않는다.
  - 기존 metrics key (`verifier_pass_count`, `verifier_fail_count`, `verifier_total`, `verification_pass_rate_percent`) 와 `--write` JSON snapshot key 는 유지한다.
  - 현재 완료되지 않은 `prd_133` / `tickets_132` 의 `.autoflow/logs/` retention 구현과 충돌하지 않도록 이 PRD 는 metrics 계산만 다룬다. 로그 파일 이동/삭제는 `tickets_132` 범위다.
- Out of Scope:
  - `.autoflow/logs/` 파일 이동, 삭제, archive 생성.
  - `packages/cli/cleanup-runner-logs.sh` 기능 변경.
  - retry limit, verifier pass/fail 판정, ticket finalization 정책 변경.
  - desktop UI 변경.
  - `.autoflow/wiki/` 직접 수정 또는 wiki synthesis 실행.
