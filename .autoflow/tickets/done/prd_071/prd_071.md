## Core Scope

- Goal: `prd_068`의 단일 영속 대화 화면, `prd_069`의 채팅 UI, `prd_070`의 provider/model 선택 계약 위에 스트리밍 전송 경로를 추가한다. 기존 `autoflow:chatSend` 단발 호출과 `chat-once.sh --agent/--model/--reasoning` 계약은 유지하고, streaming 지원 어댑터만 opt-in 경로로 chunk 를 보낸다. 미지원 어댑터는 기존 단발 호출 결과를 하나의 final chunk 로 보내는 fallback 으로 동작한다.
- In Scope:
  - `runtime/board-scripts/chat-once.sh` 에 `--stream` 옵션을 추가하거나, 같은 인자 계약을 공유하는 `runtime/board-scripts/chat-stream.sh` 를 새로 만든다. 어느 쪽이든 기존 단발 호출 사용자는 깨지지 않아야 한다.
  - scaffold copy(`scaffold/board/scripts/...`)도 runtime script 와 같은 streaming entrypoint/옵션을 갖도록 동기화한다.
  - 어댑터별 streaming 분기:
    - codex: `codex exec --stream` 또는 현재 CLI 가 제공하는 동등 streaming 모드를 사용하고, 출력 이벤트를 chunk 로 정규화한다.
    - claude: `claude -p --output-format json` 라인 단위 출력을 파싱해 text delta 를 chunk 로 정규화한다.
    - opencode/gemini: 현재 설치 CLI 에 안정적인 streaming 옵션이 확인되면 adapter branch 로 추가하고, 확인되지 않으면 기존 non-stream 호출 결과를 final chunk 로 내보낸다.
  - Main process IPC:
    - 신규 invoke 채널 `autoflow:chatSendStream({...})` 를 추가한다.
    - 호출 시작 시 `invocationId` 를 반환하거나 renderer 가 전달한 id 를 등록한다.
    - adapter stdout/stderr 를 읽으며 renderer `webContents.send("autoflow:chatStreamChunk", event)` 로 chunk, done, error 이벤트를 push 한다.
    - 기존 `autoflow:cancelInvocation` 으로 streaming child process 도 즉시 중단할 수 있어야 한다.
  - Preload/type:
    - `chatSendStream` API 와 `onChatStreamChunk`/unsubscribe API 를 노출한다.
    - `vite-env.d.ts` 에 chunk event 타입을 추가한다.
  - Renderer ChatView:
    - 전송 즉시 assistant placeholder 메시지를 만들고, chunk 가 올 때마다 마지막 assistant 메시지 본문을 in-place 로 grow 시킨다.
    - 스트리밍 중 전송 버튼은 비활성화하고 취소 버튼은 활성화한다.
    - done 이벤트 후에만 `.autoflow/conversations/desktop-chat.md` thread 파일에 최종 assistant 본문을 한 번 append 한다.
    - error/cancel 시 현재까지 받은 본문과 상태 메시지를 사용자가 이해할 수 있게 표시하고, 입력 상태를 복구한다.
  - 기존 `chatSend` 단발 경로는 호환 fallback 으로 남긴다. streaming 미지원 provider, streaming script 실패, 또는 사용자가 streaming 을 끈 경우 기존 경로로 동작할 수 있어야 한다.
  - smoke test 는 streaming IPC 노출, chunk listener cleanup, script syntax, codex/claude branch, fallback branch, thread append-on-done 규칙을 정적으로 검증한다.
  - 사용자 노출 라벨, 오류 안내, 진행 상태 문구는 한국어로 작성한다.
- Out of Scope:
  - provider/model 선택 UX 재설계.
  - 이미지 첨부의 multi-modal 정식 streaming protocol.
  - 다중 대화 세션, 아카이브 검색, saved_paths 정책 변경.
  - `autoflow wiki query --synth` 같은 deep wiki 호출.
  - ticket owner runner terminal streaming 변경.

## Global Acceptance Criteria

- [ ] 기존 `autoflow:chatSend` 단발 호출은 계속 동작하고, 새 streaming 경로는 `autoflow:chatSendStream` 로 별도 노출된다.
- [ ] preload 가 `chatSendStream` 과 `onChatStreamChunk` listener cleanup API 를 노출하고, `vite-env.d.ts` 에 chunk event 타입이 정의된다.
- [ ] main process 는 streaming 호출마다 `invocationId` 를 추적하고, adapter output 을 `autoflow:chatStreamChunk` 이벤트로 renderer 에 push 한다.
- [ ] chunk event 는 최소한 `invocationId`, `type`(`chunk`/`done`/`error`/`cancelled`), `content?`, `error?` 를 포함한다.
- [ ] codex branch 는 `codex exec --stream` 또는 동등 streaming 옵션을 사용하고, claude branch 는 `claude -p --output-format json` 라인 단위 출력을 text delta 로 파싱한다.
- [ ] opencode/gemini 가 안정적인 streaming 옵션을 제공하지 않는 경우 기존 단발 호출 결과를 final chunk 로 보내며 사용자는 빈 화면 대신 fallback 진행 상태를 본다.
- [ ] Renderer 는 사용자 메시지 전송 직후 assistant placeholder 를 만들고, chunk 수신 때마다 마지막 assistant 메시지를 in-place 로 갱신한다.
- [ ] 스트리밍이 끝난 뒤에만 최종 assistant 본문이 `.autoflow/conversations/desktop-chat.md` 에 한 번 저장된다. partial chunk 는 thread 파일에 중복 append 되지 않는다.
- [ ] 취소 버튼은 streaming child process 를 중단하고, renderer 는 취소 상태를 한국어로 표시하며 입력을 다시 활성화한다.
- [ ] adapter 오류는 현재까지 받은 본문을 보존한 채 한국어 오류와 다음 행동 힌트를 보여준다.
- [ ] provider/model 선택, wiki 인용, 요약 인계, 이미지 첨부, memo/PRD 저장 기능은 기존 smoke 범위에서 계속 통과한다.
- [ ] `npm --prefix apps/desktop run check` 가 통과한다.
- [ ] `bash tests/smoke/desktop-chat-spec-intake-smoke.sh` 가 통과한다. smoke 는 streaming IPC 노출, chunk listener cleanup, script branch, fallback branch, thread append-on-done 패턴을 정적으로 확인한다.
