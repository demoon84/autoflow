## Core Scope

- Goal: 작업 흐름 페이지 상단의 stat strip (변경 코드량 / 토큰 사용량) 을 두 가지 다듬는다. 첫째, stat strip 우측 끝선이 하단 PRD 핀 바 우측 끝선과 정확히 일치하도록 컨테이너 정렬을 통일한다. 둘째, 토큰 사용량 표기를 compact 단위(`34M`, `421K`) 에서 천 단위 구분자가 들어간 raw number(`33,381,501`, `421,384`) 로 바꿔 정확한 누적량을 한눈에 본다.
- In Scope:
  - **정렬 (alignment)**:
    - `apps/desktop/src/renderer/styles.css` 의 stat strip 컨테이너와 `.workflow-pin-strip` 의 좌/우 인셋(`padding-left`/`padding-right`, 컨테이너 폭, 자식 grid/flex 분배) 통일.
    - 필요 시 `apps/desktop/src/renderer/main.tsx` 의 TicketBoard 헤더에서 stat strip + PRD 핀 바를 같은 wrapper(`<div className="workflow-header-strip">`) 로 묶어 동일 정렬 컨텍스트.
    - viewport resize 시에도 끝선 정렬 유지.
  - **토큰 표기**:
    - `apps/desktop/src/renderer/main.tsx` 의 stat strip 우측 카드 토큰 값 포맷터를 `formatCompactCount` (예: `33.4M`) 에서 `formatCount` (예: `33,381,501`) 로 교체.
    - `통계` 페이지 ReportingDashboard 의 `aiUsageData` 에서 `formatCompactCount` 를 쓰는 부분도 동일하게 raw 표기로 바꿔 두 페이지의 표기가 동기화되도록 한다.
    - 보조 텍스트 (`실행 로그 N개`) 는 그대로.
- Out of Scope:
  - stat strip 의 좌측 카드 (변경 코드량) 의 표기 — 이미 `+14,015 / -1,868 라인 · 변경 파일 77` 형태라 raw 숫자 정책에 부합.
  - PRD 핀 바 / 반려 핀 바 내부 디자인.
  - 다른 페이지 (`티켓 정보`, `Wiki`) 의 토큰/숫자 표기.
  - compact 포맷터 자체 제거 — `formatCompactCount` 함수는 다른 차트 라벨에서 계속 쓰일 수 있어 유지.

- raw 숫자가 한국어 환경에서 가독성 좋도록 `formatCount` 가 이미 `Intl.NumberFormat` 을 사용해 `33,381,501` 형식으로 출력 (locale 자동). 별도 포맷터 추가 불필요.

- 토큰 숫자가 매우 클 때(`1,000,000,000+`) 한 줄에서 잘리지 않도록 카드 폭/폰트 크기 점검.
