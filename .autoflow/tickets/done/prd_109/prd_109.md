## Core Scope

- Goal: `apps/desktop/src/main.js`의 메인 `BrowserWindow` 생성 옵션에서 최소 가로 폭을 `1200`으로 상향하고, 현재 코드/스타일 기준에서 1200 미만 데스크톱 폭을 전제로 남은 반응형 분기가 실제로 필요한지 점검할 수 있게 범위를 명확히 한다.
- In Scope:
  - `apps/desktop/src/main.js`의 메인 `BrowserWindow` 생성 옵션에서 `minWidth`를 `1040`에서 `1200`으로 올린다.
  - 같은 파일 안에 다른 `new BrowserWindow(...)` 호출이 없는지 확인하고, 동일 앱 메인 윈도우 경로에만 변경을 적용한다.
  - 초기 `width: 1320`과 `minHeight: 720`는 별도 요구가 없으면 유지한다.
  - 창 상태 복원 또는 이후 resize 경로가 있더라도 Electron의 최소 폭 제약을 우회하지 않음을 코드 기준으로 확인한다.
  - `apps/desktop/src/renderer/styles.css`의 `@media (max-width: 1120px)` / `@media (max-width: 820px)` 등 1200 미만 폭 분기가 현재 데스크톱 메인 윈도우 기준에서 dead branch인지, 또는 dialog/overlay 등 다른 viewport 상황 때문에 여전히 필요한지 점검한다. 실제 충돌이 확인될 때만 좁게 수정한다.
- Out of Scope:
  - 데스크톱 앱의 전체 반응형 레이아웃 재설계.
  - `width`, `height`, `minHeight` 기본값 재조정.
  - 다른 Electron 창 또는 별도 utility window 추가.
  - workflow pin strip, AI dashboard 카드, dialog 폭 등 renderer 스타일의 광범위한 재배치.
  - retry-limit reject(`reject_003`, `reject_071`, `reject_074`) 재시도, dirty-root finalization blocker 해결, runner/runtime 계약 변경.
