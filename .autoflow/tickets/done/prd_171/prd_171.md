## Core Scope

- Goal: `tickets/inbox/order_151.md` 가 보고한 worker stuck root cause를 runtime 레벨에서 해결한다. ticket 의 `Allowed Paths` 에 자기 ticket markdown 또는 `verify_NNN.md` 가 포함되어 있더라도 worker 의 메타데이터 갱신만으로 같은 ticket 이 영구 blocked 되지 않아야 한다.
- In Scope:
  - ticket-owner dirty overlap 검사에서 현재 claim/resume 중인 ticket markdown(`.autoflow/tickets/inprogress/tickets_NNN.md`)과 해당 verification markdown(`.autoflow/tickets/inprogress/verify_NNN.md`)을 self-refresh metadata path 로 분류한다.
  - dirty overlap 이 self-refresh metadata path 만으로 구성된 경우 `ticket_stage_blocked` 로 막지 않고 계속 진행한다. 구현상 필요하면 Stage / Last Updated / Goal Runtime / Notes 중 런타임이 같은 tick 에 기록하는 메타데이터 갱신만 허용한다.
  - 실제 제품 파일 또는 ticket 범위 밖 board 파일 dirty overlap 은 기존처럼 block / planner orchestration 대상으로 유지한다.
  - `packages/cli/run-role.sh` 의 worker tick 진입 / active recovery state 갱신이 stale `ticket_stage_blocked` 를 self-refresh-only 상황으로 반복 보존하지 않도록 조정한다.
  - `packages/cli/runners-project.sh` 의 stop / cleanup 경로에 worktree-aware orphan runner 처리 설계를 추가한다. worktree 내부에서 시작된 `loop-worker worker` 가 board script path mismatch 로 실패 상태에 남는 경우를 정상 stop 경로에서 정리할 수 있어야 한다.
  - installed board template 인 `runtime/board-scripts/*` 와 현재 sidecar `.autoflow/scripts/start-ticket-owner.sh` 동작을 대칭으로 유지한다.
- Out of Scope:
  - planner adapter 가 현재 실행 중인 runner / OS process 를 직접 kill 하거나 재시작하는 작업.
  - broad worktree cleanup 정책, `git worktree remove` 정책 전면 재작성, Desktop UI 변경.
  - unrelated `ticket_stage_blocked` 원인 전체 해결. 제품 파일 dirty overlap, shared head conflict, missing worktree 는 기존 recovery flow 를 따른다.
  - `git push`.
