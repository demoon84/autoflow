- AI:
- Status: draft

## Core Scope

- Goal: 데스크톱 작업 흐름 페이지의 PRD 핀 라벨이 누적 PRD 수만 보여주는 현재 표기 (`PRD 14건`) 에서, 아직 처리되지 않고 backlog 에 남은 대기 PRD 수를 함께 노출해 (`PRD 14건 · 대기 N건`) 사용자가 잔여 작업량을 즉시 인지할 수 있게 한다.
- In Scope:
  - `apps/desktop/src/renderer/main.tsx` 의 PRD 핀 정의 (라인 ~4014-4024) 의 `pinTitle` / `layerHeading` 문자열 변경.
  - 표기 규칙:
    - 대기 PRD (`backlogSpecs.length`) > 0 일 때: `PRD ${total}건 · 대기 ${pending}건`
    - 대기 PRD 0 일 때: `PRD ${total}건` (현재 표기 유지, 깔끔함)
  - 같은 표기를 `pinTitle` 과 `layerHeading` 양쪽에 동일하게 적용해 펼쳤을 때 헤더와 일관됨.
  - 구분자는 가운데점(`·`) 사용 — slash(`/`) 보다 KR/EN 혼용 환경에서 시각적으로 가벼움. (사용자가 `/` 를 명시 선호하면 옵션으로 수용.)
- Out of Scope:
  - 반려(`rejectFiles`) 핀의 라벨 변경.
  - 핀 클릭 시 펼쳐지는 layer 내부 정렬/필터링 변경.
  - 메트릭 (`spec_backlog_count` 등) 신규 수집 — 이미 `board.tickets.backlog` 에 있는 데이터로 충분.
  - 다른 페이지(`통계`, `티켓 정보`)의 PRD 카운트 표기.
