  - 기존 `Priority:` 출력은 `prd_146` / `Todo-145`의 입력 metadata 결과로 보존하되, 이 PRD에서는 priority validation이나 queue sorting을 새로 구현하지 않는다.

  - 네 order skill 모두 가능한 경우 `autoflow order create` CLI를 우선 사용하고, order intake 중 PRD/todo 생성, 구현, 검증, commit, push를 하지 않는다는 경계를 유지한다.
- Out of Scope:
  - `autoflow order create --priority` 옵션 추가, priority keyword inference, explicit priority precedence는 `prd_146` / `Todo-145` 범위다.
  - queue priority sorting은 `prd_145` / `Todo-144` 범위다.
  - 기존 inbox order frontmatter backfill, `autoflow order normalize` 신설, 과거 order 일괄 마이그레이션은 제외한다.
  - Desktop markdown preview가 과거 frontmatter order를 숨기거나 변환하는 fallback UI 보정은 제외한다.
  - PRD, ticket, check 파일 형식 표준화는 제외한다.
