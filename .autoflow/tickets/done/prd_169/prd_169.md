# Project PRD

## Project

- ID: prd_169
- Name: worker last_result self-reset after cleanup
- Priority: high
- Title: worker.state.last_result=`ticket_stage_blocked` 잔류 자가 reset
- Goal: PROJECT_ROOT 의 dirty path 가 모두 정리되고 blocked-dirty orchestration cleanup 이 끝난 뒤에도 `worker.state.last_result` 에 `ticket_stage_blocked` 가 stale 상태로 남아 monitor / desktop UI 가 worker 를 계속 "blocked" 으로 표시하는 문제를 끊는다. worker tick 자가 reset 과 planner cleanup 후 명시적 reset 을 동시에 적용해 user-visible "blocked" 표시가 실제 상태와 동기화되도록 한다.
- AI: planner
- Status: generated

## Core Scope

- Goal: `tickets/inbox/order_148.md` 가 보고한 잔류 last_result 표면을 두 곳에서 동시에 정리한다 — (A) worker tick 시작 시 자가 reset, (B) planner blocked-dirty cleanup 종료 시 worker.state 의 stale `ticket_stage_blocked` 명시적 비우기.
- In Scope:
  - `packages/cli/run-role.sh` 의 `ticket` 역할 tick 진입부에 `worker.state` 의 `last_result` 가 `ticket_stage_blocked` 이고 PROJECT_ROOT 가 해당 ticket Allowed Paths 기준 clean 이면 `last_result` 를 빈 값(또는 `idle`) 으로 reset 하는 자가 회복 단계를 추가한다.
  - `.autoflow/scripts/start-plan.sh` 와 `runtime/board-scripts/start-plan.sh` 의 blocked-dirty orchestration 종료부(또는 `blocked-auto-recover` 직전 분기) 에 `worker.state` 의 `last_result` 가 `ticket_stage_blocked` 면 명시적으로 비우는 후처리 호출을 추가한다.
  - 두 경로 모두 race-safe 하게 동작 (worker / planner 동시 tick 시 last_result 만 다루고 다른 state field 는 건드리지 않는다).
  - sidecar(`.autoflow/scripts/*`) 와 install 템플릿(`runtime/board-scripts/*`) 동작 일치.
- Out of Scope:
  - prd_168 가 다루는 check-ledger 자기참조 live-lock fixpoint guard.
  - worker 가 active ticket 을 새로 claim 하거나 stage 를 재계산하는 로직 변경.
  - planner / worker runner 또는 OS process 강제 종료, `git push`, retry / cleanup 정책 변경.
  - desktop UI 의 worker status 표시 로직 자체 변경 (state 가 정확해지면 UI 는 자연 회복).
  - 다른 sticky last_result 값(e.g. `adapter_timeout`, `adapter_timeout_fallback`) 에 대한 self-reset.
