## Core Scope

- Goal: `tickets/inbox/order_156.md`가 보고한 `ANTHROPIC_API_KEY` 부재 late-block 패턴을 planner promote 단계에서 먼저 감지한다.
- In Scope:
  - `start-plan.sh` backlog-to-todo 경로에서 PRD의 `## Verification` `Command:`와 명시 secret metadata(`Requires Secrets` 또는 `requires_secrets`)를 읽어 필요한 환경변수 이름을 수집한다.
  - 필요한 환경변수 중 현재 process 환경에 값이 없는 항목이 있으면 todo ticket을 생성하지 않고, backlog PRD의 `Status`를 `needs_user_secret`로 갱신하며 `## Notes`에 missing secret evidence를 1회만 기록한다.
  - blocked 결과는 key=value output으로 `source=needs-user-secret`, `missing_secrets=...`, `recovery_state=needs_user`, `failure_class=needs_user_decision`을 노출해 runner/UI가 이유를 표시할 수 있게 한다.
  - secret이 주입된 뒤 같은 PRD가 다음 planner tick에서 다시 promote될 수 있도록, `needs_user_secret` PRD도 missing secret이 더 이상 없으면 정상 backlog-to-todo 경로로 진행한다.
  - spec author 계약과 PRD template에 `Requires Secrets` 작성 가이드를 추가해 live API, cloud, provider credential 의존을 명시하도록 한다.
  - sidecar `.autoflow/scripts/*`와 install template `runtime/board-scripts/*`의 planner secret preflight 동작을 동일하게 유지한다.
- Out of Scope:
  - 별도 `tickets/needs_secret/` 폴더 또는 `autoflow secret resume` CLI 신설.
  - 임의 본문 전체에서 secret token을 광범위하게 스캔하는 기능. 이번 ticket은 false positive를 줄이기 위해 `Verification.Command`와 명시 metadata만 gate로 사용한다.
  - 실제 secret 값을 저장하거나 로그에 출력하는 기능.
  - Anthropic API adapter 자체 구현 또는 live API 검증.

## Global Rules

- Allowed Paths are relative to the host project root.
- Verification commands run from the host project root unless a ticket says otherwise.
- Secret preflight must never print secret values; output and notes may contain only variable names.
- Secret token examples in PRD prose must not by themselves block promotion unless they appear in `Verification.Command` or explicit secret metadata.
- Planner runtime remains a safety-kernel tool: it may park a PRD and emit evidence, but Impl AI still owns implementation and verification for the generated ticket.
- Do not create `tickets/needs_secret/` in this ticket.
- Do not run live Anthropic/OpenAI/AWS calls in verification.
- Do not `git push`.
