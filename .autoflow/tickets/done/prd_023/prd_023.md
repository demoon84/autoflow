# Project PRD

## Project

- ID: prd_023
- Title: Remove left-border color accents from AI progress cards and all workflow pin bars
- AI:
- Status: draft

## Core Scope

- Goal: 작업 흐름 페이지 상단의 시각 노이즈를 줄인다. AI 진행 카드의 단계 별 좌측 색 보더(`구현 중` 녹색, `반려` 빨간색 등) 와 PRD 핀 바 / reject 핀 바 / 기타 workflow pin variant 의 두꺼운 좌측 색 보더를 모두 제거해, 카드와 핀의 외곽이 일관된 단일 톤(`1px solid var(--border)`) 으로 보이도록 한다. 단계 정보는 카드 안 dots 와 핀 안 텍스트 / 카운트 / 아이콘으로 충분히 전달된다.
- In Scope:
  - **AI 진행 카드 좌측 보더** (`apps/desktop/src/renderer/styles.css`):
    - `.ai-progress-plan`, `.ai-progress-todo`, `.ai-progress-inprogress`, `.ai-progress-verifier`, `.ai-progress-done`, `.ai-progress-reject` 의 `border-left-color: var(--*)` 선언 제거.
    - 어디서든 `.ai-progress-row` 또는 그 변형에 `border-left-width` 가 따로 더 두껍게 설정돼 있다면 함께 일반화 (모든 변이 1px).
  - **Workflow 핀 바 좌측 보더** (`apps/desktop/src/renderer/styles.css`):
    - `.workflow-pin` 의 `border-left-width: 3px` 를 `border-width: 1px` 와 동일하게 정렬.
    - `.workflow-pin-default` 의 `border-left-color: var(--primary)` 와 hover/focus 색 override 제거.
    - `.workflow-pin-destructive` 의 `border-left-color: var(--destructive)` 및 동일 hover/focus 제거.
    - 향후 추가될 수 있는 다른 variant (예: 사용자가 언급한 "TODO 핀바") 도 같은 정책을 따르도록 base `.workflow-pin` 에 `border-left-color: inherit` 또는 `var(--border)` 만 정의해 변이가 색을 깔지 못하게 한다.
    - 핀 안의 background tint (예: `background: color-mix(in srgb, var(--primary) 4%, var(--card))`) 도 함께 제거해 카드/핀이 모두 plain `var(--card)` 배경으로 통일.
  - **Pin 의 시각 정보 보존**:
    - 각 핀의 좌측 아이콘 색상은 유지 (`PRD` 는 ClipboardCheck, `반려` 는 TriangleAlert 등). 색 강조는 아이콘 색만 담당.
    - 카운트 라벨 (`PRD 16건 · 대기 N건`, `반려 N건 보류 중`) 은 그대로.
- Out of Scope:
  - 카드 / 핀의 hover 인터랙션 변경 (보더 색 외 효과 유지).
  - 다른 페이지(`통계`, `티켓 정보`, `Wiki`) 의 카드 좌측 보더.
  - reject / done / blocked badge, status pill 색상 변경.
  - markdown blockquote (`.markdown-viewer blockquote`) 좌측 보더 — 별도 컨텍스트라 영향 없음.
