- Title: 데스크톱 대화 헤더 provider/model 선택
- Goal: 데스크톱 "대화" 화면에서 사용자가 codex / claude / opencode / gemini 중 사용할 AI provider 와 model 을 직접 선택하고, 선택값이 대화 스레드 frontmatter 에 보존되며, 선택한 어댑터 CLI 가 없을 때 한국어 안내와 빠른 전환 버튼으로 즉시 우회할 수 있게 한다.
- AI: ticket-owner
- Status: draft

## Core Scope

- Goal: `prd_068`로 만든 단일 영속 대화 화면과 `prd_069`로 다듬은 채팅 UI 위에 provider/model 선택 UX 를 추가한다. 기본 provider/model 은 기존 runner profile 또는 현재 `desktop-chat.md` frontmatter 에서 가져오고, 사용자가 바꾸면 `.autoflow/conversations/desktop-chat.md` frontmatter 의 `provider` / `model` / `reasoning?` 에 저장한다. 이후 `autoflow:chatSend` 와 요약 호출은 저장된 선택값을 사용한다.
- In Scope:
  - 대화 화면 헤더에 provider 드롭다운 또는 segmented control 을 추가한다. 선택지는 `codex`, `claude`, `opencode`, `gemini` 이며 현재 선택값과 설치 여부를 함께 보여준다.
  - provider 선택 옆에 model/effort 표시 또는 model 선택 컨트롤을 둔다. 기존 Desktop runner 설정 화면의 agent/model normalization 패턴과 model option 목록을 재사용한다.
  - 신규 IPC `autoflow:listChatAdapters` 를 추가해 설치된 어댑터, PATH 확인 결과, 기본 runner profile 의 agent/model/reasoning, 현재 `desktop-chat.md` frontmatter 선택값을 반환한다.
  - 신규 IPC `autoflow:setChatAgent` 를 추가해 선택한 provider/model/reasoning 을 `.autoflow/conversations/desktop-chat.md` frontmatter 에 저장하고, thread 파일이 없으면 기존 chat thread 포맷으로 생성한다.
  - `autoflow:chatLoad` 결과에 현재 chat agent 선택과 adapter 목록을 포함해 최초 렌더 시 헤더가 즉시 맞는 값을 보여준다.
  - `autoflow:chatSend` 와 `autoflow:chatSummarize` 는 명시 options 가 없으면 frontmatter 의 provider/model/reasoning 을 사용하고, 그것도 없으면 default runner profile 을 fallback 으로 사용한다.
  - 선택한 provider CLI 가 PATH 에 없거나 `chat-once.sh` 가 `reason=agent_cli_not_found:<agent>` 를 반환하면 채팅 화면에 한국어 inline 안내를 표시하고, 설치된 다른 어댑터로 빠르게 전환하는 버튼을 보여준다.
  - 전환 버튼은 `autoflow:setChatAgent` 로 frontmatter 를 갱신하고, 사용자가 같은 메시지를 다시 보낼 수 있도록 입력 상태를 보존한다.
  - 기존 `chat-once.sh --agent/--model/--reasoning` 계약은 유지한다. provider 선택을 위해 별도 런타임 스크립트를 새로 만들지 않는다.
  - 사용자 노출 라벨, 오류 안내, 빈 상태 문구는 한국어로 작성한다.
- Out of Scope:
  - `.autoflow/runners/config.toml` 의 default runner profile 을 직접 수정하는 기능.
  - provider 별 멀티모달 정식 프로토콜, 응답 토큰 스트리밍, 다중 대화 세션.
  - 대화 화면의 메시지 레이아웃, 이미지 첨부, memo/PRD 저장 플로우 재설계.
  - adapter CLI 설치 자동화 또는 PATH 자동 수정.

- [ ] 현재 선택 provider/model/reasoning 이 `.autoflow/conversations/desktop-chat.md` frontmatter 의 `provider` / `model` / `reasoning` 에 저장되고, 앱 재진입 또는 `chatLoad` 후 같은 값으로 복원된다.
- [ ] `autoflow:listChatAdapters` 가 네 provider 의 PATH 설치 여부, default runner profile fallback, 현재 chat frontmatter 선택값을 반환한다.
- [ ] `autoflow:setChatAgent` 가 thread 파일이 없을 때도 기존 chat thread frontmatter 포맷을 만들어 선택값을 저장한다.
- [ ] `autoflow:chatSend` 와 `autoflow:chatSummarize` 는 명시 agent/model/reasoning options 가 없으면 frontmatter 선택값을 사용하고, frontmatter 가 비어 있으면 default runner profile 을 사용한다.

- [ ] 미설치 안내에는 설치된 다른 provider 로 전환하는 버튼이 있고, 클릭하면 선택값 저장 후 헤더 상태가 즉시 갱신된다.

- [ ] 기존 memo/PRD 저장, wiki 인용 토글, 요약 인계 토글, 이미지 첨부 UI 는 기존 smoke 범위에서 계속 통과한다.
