# Project

- ID: prd_167
- Title: Runner 정지 흐름을 graceful "중지 예약" + 단계별 알림으로 전환
- AI: planner/worker
- Status: draft

## Core Scope

- Goal: 데스크톱 앱과 CLI 의 runner 정지 동작을 즉시 SIGKILL 에서 graceful "중지 예약" 패턴으로 전환해 진행 중인 LLM 호출과 partial state 손실을 방지하고, 사용자에게 "중지 예약됨 → 멈춤 완료" 두 단계 알림을 제공한다. 강제 종료는 명시적 옵션(`--force` / 확인 다이얼로그)으로만 노출한다.
- In Scope:
  - **State 스키마 확장 (`runner.state`)**:
    - `stop_pending=true|false` (기본 false)
    - `stop_requested_at=<ISO8601>` (예약 시각)
    - `stop_mode=graceful|force` (기본 graceful)
    - `stop_requested_by=user|system` (출처)
  - **CLI 동작 변경 (`packages/cli/runners-project.sh` `stop_runner()` 외)**:
    - 기본 `stop` 동작을 graceful 로 전환: `stop_pending=true`, `stop_requested_at` 마킹, PID 살려둔 채 SIGKILL 미실행, `last_event_at` 갱신.
    - `--force` 플래그 신설: 즉시 SIGKILL (현재 로직 유지), `stop_mode=force`, `last_stop_reason=user_force`.
    - tick 시작 시 (`packages/cli/run-role.sh`) `stop_pending` 검사 → true 면 tick skip + 자기 자신 graceful 종료 (`status=stopped`, `last_stop_reason=graceful_stop_completed`).
    - LLM 호출 중에는 호출 응답 또는 `AUTOFLOW_AGENT_TIMEOUT_SECONDS` timeout 까지 자연 진행 후 finalize.
    - `AUTOFLOW_GRACEFUL_STOP_MAX_WAIT_SECONDS` (기본 300) 초과 시 SIGTERM → 30s 후 SIGKILL fallback (1원칙 보존, 무한 대기 방지). 이 경로의 `last_stop_reason=graceful_stop_max_wait_force`.
  - **데스크톱 UI 변경 (`apps/desktop/src/renderer/main.tsx`, `apps/desktop/src/main.js`, `apps/desktop/src/renderer/styles.css`)**:
    - stop 버튼 클릭 → `controlRunner(id, "stop")` 기본 graceful.
    - 버튼 상태: `중지` → `중지 예약 중...` (스피너 + 비활성 톤) → `시작` (재활성).
    - graceful pending 상태에서 추가 클릭 시 확인 다이얼로그 "현재 LLM 호출이 진행 중입니다. 강제 종료할까요?" → Yes 면 force stop.
    - 토스트 두 단계: 클릭 직후 `"<runner-id> 중지 예약됨 (현재 작업 끝나면 자동 멈춤)"`, state `status=stopped` 감지 시 `"<runner-id> 멈춤 완료"`.
    - Electron native `Notification` API 로 OS-level 알림 옵션 (default off, 환경변수/settings 토글).
  - **일괄 정지 정책**:
    - 일반 stop-all 은 모든 runner 에 graceful 적용.
    - emergency stop / `autoflow halt --all` (order_138 계열) 은 force 유지 (1원칙 자기 보호).
  - **정책/문서 갱신**:
    - `AGENTS.md` 에 graceful stop 기본 정책, `last_stop_reason` 확장값(`graceful_stop_completed`, `user_force`, `graceful_stop_max_wait_force`), `--force` flag 의미 추가.
- Out of Scope:
  - start / restart 등 다른 runner control 의 동작 변화 (즉시 동작 유지).
  - 기존 `last_stop_reason` 마커 정리(PRD-135) 와 emergency circuit breaker(order_139) 의 자기 보호 로직 변경.
