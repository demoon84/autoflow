## Core Scope

- Goal: `apps/desktop/src/renderer/styles.css` 의 `.ai-progress-board` 기본 desktop grid 를 `repeat(4, minmax(0, 1fr))` 기준으로 바꾸고, 기존 `data-runner-count="3"` / `data-runner-count="4"` 전용 grid-area 분기로 생기던 2열 또는 3열 특수 배치를 제거하거나 단순화한다.
- In Scope:
  - `.ai-progress-board` 가 일반 desktop 폭에서 4개 column 을 만든다.
  - `data-runner-count="3"` 상태에서 worker 카드가 세로 span 하지 않고 planner / worker / wiki 순서의 1행 3칸 배치가 유지된다.
  - `data-runner-count="4"` 상태에서 4개 카드가 2x2가 아니라 1행 4칸으로 배치된다.
  - 카드 내부 control, active ticket footer, runner label, log 영역 기능은 유지하고, 필요한 경우 padding / min-width / text overflow CSS 만 좁게 조정한다.
  - viewport 폭이 좁은 경우 media query 로 2칸 또는 1칸 fallback 을 제공한다.
  - `apps/desktop/src/renderer/main.tsx` 는 기존 `data-runner-count` / `data-runner-role` 사용부 정리가 CSS 만으로 부족할 때만 좁게 수정한다.
- Out of Scope:
  - runner lifecycle, adapter/model/reasoning selector 동작, ticket open action, log streaming, token usage 계산 변경.
  - AI 진행 카드 내부 정보 구조 재설계 또는 새 UI 문구 추가.
  - workflow pin strip, ticket detail layer, Wiki/Statistics 화면 변경.
  - shadcn component 추가 또는 MUI/Emotion 계층 변경.
