- In Scope:
  - `.autoflow/scripts/start-plan.sh` 와 `runtime/board-scripts/start-plan.sh` 의 dirty inventory 산정에서 **planner 가 직전 cleanup 에서 새로 만든 `.autoflow/tickets/check/check_NNN.md` 만 단독으로 dirty 인 경우** 를 별도 케이스로 인식한다 (no-op cleanup).

  - 동일 ticket 의 누적 orchestration cleanup commit 이 임계치(예: 5건) 를 넘으면 fixpoint guard 를 발동해 ticket 을 `Recovery State: needs_user` 로 escalate (start-plan.sh 가 자동으로 set).

  - 현재 board sidecar (`.autoflow/scripts/*`) 와 설치 템플릿 (`runtime/board-scripts/*`) 동작 일치.
- Out of Scope:
  - 이미 누적된 `Todo-162` 의 16건 check ledger 정리 또는 ticket 강제 done 이동 (별도 board housekeeping).
  - worker `last_result=ticket_stage_blocked` self-reset (이는 `order_148` follow-up 에서 별도 PRD 로 다룬다).
  - planner runner 또는 OS process 강제 종료, `git push`, destructive cleanup, retry 정책 변경.
