# Project

- ID: prd_154
- Title: Output max_tokens cap — 비정상 긴 LLM 응답으로 인한 토큰 폭주 방지
- AI: planner/worker
- Status: draft

## Core Scope

- Goal: LLM 어댑터 호출에 output max_tokens cap 을 도입해 비정상적으로 긴 응답이 토큰을 과소비하지 않도록 안전망 추가 (5-10% 절감).
- In Scope:
  - 환경변수 신설:
    - `AUTOFLOW_PLANNER_MAX_OUTPUT_TOKENS` (기본 8000)
    - `AUTOFLOW_WORKER_MAX_OUTPUT_TOKENS` (기본 16000)
    - `AUTOFLOW_VERIFIER_MAX_OUTPUT_TOKENS` (기본 4000)
    - `AUTOFLOW_WIKI_MAX_OUTPUT_TOKENS` (기본 2000)
  - `packages/cli/run-role.sh` 의 어댑터 호출에 max_tokens 인자 추가:
    - claude: `--max-tokens N` (CLI 지원 확인 후 적용. 미지원 시 truncate 후처리)
    - codex: 동등 옵션
    - gemini: `--max-output-tokens` (gemini CLI 지원)
  - 응답이 cap 도달 (truncated) 시 stdout 마지막에 `output_truncated=true` marker 1줄 + .autoflow/runners/logs/*.log 에 흔적.
  - cap 도달이 잦으면 정책 조정 트리거 — 운영 후 cap 자동 상향 검토.
- Out of Scope:
  - 응답 truncation 후 ' continuation' 자동 호출 (위험, 후속).
  - input cap (PRD-153).
  - 모델별 동적 cap (현재는 정적 기본값).
