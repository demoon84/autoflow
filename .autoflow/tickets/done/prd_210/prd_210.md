# PRD 210 — 데스크톱 러너 라벨 한국어 → 영어 (Worker / LLM Wiki)

## Goal

데스크톱 사이드바 / 러너 카드 / 티켓 메타데이터에서 노출되는 한국어 라벨 `워커` / `LLM위키` / `LLM 위키` 를 영어 `Worker` / `LLM Wiki` 로 일괄 변경한다. suffix(`AI` 등) 는 붙이지 않으며 `Planner` 는 이미 영어라 변경하지 않는다. 한국어 정책(AGENTS rule 15a) 의 예외로, 사용자가 명시 요청한 위치만 영어로 표기한다.

## Background

- order_188 사용자 요청: "Worker, LLM Wiki 영어로 표기".
- 과거 2026-04-30 에 영어로 한 번 갔다가 2026-05-02 에 한국어로 회귀한 이력 있음. 이번엔 다시 영어, suffix 없이.
- AGENTS rule 16 의 worker normalize 규칙(영어 `worker`)과 일치한다.

## Scope

- `apps/desktop/src/renderer/main.tsx` 의 사용자 노출 한국어 라벨만 변경.
  - line 206 `label: "LLM 위키"` → `"LLM Wiki"`
  - line 4802 `label: "진행 중", description: "워커가 처리중"` → description 만 `"Worker가 처리중"`
  - line 5155 `["워커", item.aiLabel]` → `["Worker", item.aiLabel]`
  - line 6557 `singleton ? "LLM위키" : "LLM위키-1"` → `"LLM Wiki" : "LLM Wiki-1"`
  - line 6558/6559 `LLM위키-` prefix → `LLM Wiki-`
  - line 6576 `return "워커";` → `return "Worker";`
  - line 6577 `return "LLM위키";` → `return "LLM Wiki";`
- 기타 grep 으로 발견되는 동일 라벨이 사용자 노출 경로에 있으면 함께 변경.

## Out of Scope

- runtime role key, runner state 파일명, config id, ticket field 같은 storage/parser 식별자(원형 유지).
- AGENTS.md / 다른 문서의 한국어 정책 본문.
- worker-N suffix 가 있는 multi-runner 표기 규칙(rule 16) 자체는 그대로 유지.

## Allowed Paths

- `apps/desktop/src/renderer/main.tsx`

## Done When

- [ ] `apps/desktop/src/renderer/main.tsx` 안에서 사용자 노출 라벨로서의 `워커` / `LLM위키` / `LLM 위키` 문자열이 0건이다 (`rg -n "워커|LLM위키|LLM 위키" apps/desktop/src/renderer/main.tsx` 결과 0줄).
- [ ] 같은 파일에 `"Worker"` 와 `"LLM Wiki"` 라벨이 등장한다 (settingsNavigation knowledge 항목, runner display 매핑 함수, 티켓 메타데이터 라벨 모두 포함).
- [ ] 변경은 사용자 노출 라벨에만 한정되어 있고 role key / state 파일명 / runner id 같은 storage 식별자는 그대로다.

## Verification

- Command: `rg -n "워커|LLM위키|LLM 위키" apps/desktop/src/renderer/main.tsx | wc -l` → `0` 이어야 한다.

## Notes

- order_188 본문 보존: 사용자가 직접 후보 라인을 짚어 줬으므로 그 라인을 우선 처리하고 추가 위치는 grep 으로 확인.
- 한국어 정책(rule 15a) 의 예외 케이스로 처리. 다른 화면 한국어 표기는 건드리지 않는다.
