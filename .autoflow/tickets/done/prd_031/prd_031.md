## Core Scope

- Goal: The desktop app should no longer expose a separate AI management menu/page. Runner monitoring and controls should remain available through the existing workflow page controls.
- In Scope:
  - Remove the `AI 관리` entry from the desktop navigation/menu.
  - Remove or disable routing/rendering for the dedicated AI management page so it cannot be opened from normal desktop navigation.
  - Preserve runner start/stop, model, reasoning, and save controls that were previously hoisted into the workflow page.
  - Keep workflow, ticket board, PRD/inbox workspace, wiki, logs, and metrics navigation unchanged except for layout changes caused by removing the AI management entry.
  - Keep user-facing UI copy in Korean.
- Out of Scope:
  - Do not change runner runtime scripts, runner config format, adapter behavior, or board lifecycle semantics.
  - Do not remove runner controls from the workflow page.
  - Do not redesign the workflow page or ticket workspace beyond what is required to remove the AI management page.
  - Do not change product code outside the desktop source tree.
