# Project PRD

## Project

- ID: prd_017
- Title: Restrict Claude reasoning dropdown to medium/high
- AI:
- Status: draft

## Core Scope

- Goal: 데스크톱 UI 의 Claude 어댑터 reasoning 드롭다운 옵션을 현재 5단계 (`low`, `medium`, `high`, `xhigh`, `max`) 에서 **`medium`, `high`** 두 단계로 축소한다. `xhigh` / `max` 는 Anthropic API 키 사용자 한정 옵션이라 Claude.ai 구독 환경에서 선택 시 매 tick 실패하고, `low` 는 Plan/Impl 로 의미 있는 결과를 내기 어려워 실용성이 낮다. 기본값은 `high`.
- In Scope:
  - `apps/desktop/src/renderer/main.tsx` 의 `runnerAgentReasoningOptions.claude` 배열을 `["medium", "high"]` 로 교체.
  - `runnerOptionLabels.claude` (또는 동등한 라벨 맵) 에서 `low`/`xhigh`/`max` 한국어 라벨 항목이 더 이상 필요 없다면 그대로 두거나 정리 (다른 어댑터에서 재사용 중이면 유지).
  - 기존 config.toml 에 `low|xhigh|max` 가 저장돼 있던 runner 의 호환성 처리: 드롭다운 노출 옵션과 별개로, 저장된 값은 폼이 마운트될 때 `normalizeRunnerSelections` 로 자동 가까운 값(`low|xhigh|max` → `high` 로 보정) 매핑.
  - `runnerAgentReasoningOptions.codex`, `runnerAgentReasoningOptions.opencode`, `runnerAgentReasoningOptions.gemini` 는 변경하지 않음.
- Out of Scope:
  - CLI / scripts 의 reasoning 검증 (현재도 `claude` CLI 가 자체 검증).
  - API 키 감지 후 max 자동 노출 (필요해지면 별도 PRD).
  - codex / opencode / gemini 의 reasoning 옵션.
  - reasoning 라벨 텍스트 (`Medium`, `High`) 변경.

## Main Screens / Modules

- Module: 데스크톱 reasoning 드롭다운 옵션 정의
- Path: `apps/desktop/src/renderer/main.tsx` (`runnerAgentReasoningOptions`)
- Module: 저장값 정규화 (옵션 축소 후 잘못된 값 fallback)
- Path: `apps/desktop/src/renderer/main.tsx` (`normalizeRunnerSelections` / `runnerReasoningChoices`)

## Global Rules

- Allowed Paths are relative to the host project root.
- Verification commands run from the host project root.

- 기존 보드의 config.toml 을 임의로 다시 쓰지 않음 — 사용자가 드롭다운에서 새 값 선택 후 저장할 때만 갱신.

## Global Acceptance Criteria

- [ ] AI 관리 페이지에서 `claude` 어댑터를 선택하면 reasoning 드롭다운에 `Medium`, `High` 두 옵션만 보인다.
- [ ] 다른 어댑터 (`codex`, `opencode`, `gemini`) 의 reasoning 드롭다운은 기존과 동일하다.
- [ ] 저장돼 있던 runner 의 reasoning 이 `low` / `xhigh` / `max` 인 경우, 폼에 진입할 때 드롭다운이 가장 가까운 허용 값(`high`)을 보여주고 사용자가 저장하기 전까지는 config.toml 자동 변경 없음.
- [ ] 새 runner 를 claude 로 추가하면 reasoning 기본값이 `high` 다.
- [ ] `cd apps/desktop && npx tsc --noEmit` 가 0 errors 로 통과한다.
- [ ] `cd apps/desktop && npm run check` 가 통과한다.
- [ ] 시각 회귀: 다른 어댑터 / 다른 페이지에 영향 없음.

## Notes

- 향후 API 키 사용자에 한해 `xhigh`/`max` 를 노출하려면 `runnerAgentReasoningOptions.claude` 를 `process.env.ANTHROPIC_API_KEY` 또는 부모 데스크톱 main 에서 전달된 플래그에 따라 동적으로 분기. 본 티켓에서는 비분기 정적 배열로 단순화.
- `low` 가 없어지면 일부 사용자가 비용 절감용으로 low 를 쓰던 케이스를 잃지만, claude high 의 출력이 충분히 빠르고 토큰 비용 차이는 max 대비 미미하므로 트레이드오프 수용.
