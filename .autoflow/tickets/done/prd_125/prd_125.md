## Core Scope

- Goal: `run_with_timeout ... run_adapter_with_identity codex exec ... < "$prompt_file"` 경로에서 watchdog wrapper 가 stdin 을 잃지 않도록 `run_with_timeout` 내부 child 실행 방식을 수정한다.
- In Scope:
  - `packages/cli/run-role.sh` 의 `run_with_timeout` 함수에서 timeout/watchdog 동작은 유지하면서 child 프로세스가 caller stdin 을 읽을 수 있게 한다.
  - Codex direct path (`AUTOFLOW_CODEX_DISABLE_PTY=1`) 와 custom adapter command path 처럼 prompt file 을 stdin 으로 넘기는 호출이 prompt body 를 그대로 받는지 확인한다.
  - 기존 timeout 발생 시 exit code `124`, SIGTERM 후 kill-after SIGKILL, child exit code 전달 동작을 유지한다.
  - macOS BSD 환경에서 별도 `timeout` / `gtimeout` 의존성을 추가하지 않는다.
- Out of Scope:
  - adapter timeout 정책값 (`AUTOFLOW_AGENT_TIMEOUT_SECONDS`, `AUTOFLOW_AGENT_KILL_AFTER_SECONDS`) 변경.
  - Codex / Claude / OpenCode / Gemini prompt 구성 내용 변경.
  - runner lifecycle, telemetry, metrics, stdout retention 구현 변경.
  - `packages/cli/run-role.sh` 외 제품 파일 수정.
