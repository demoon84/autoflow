- Goal: 데스크톱 Wiki 화면에서 좌측 목록 상단의 작은 `Wiki / 목록` 헤더와 미리보기 토글을 제거하고, 우측 상세 패널이 첫 진입부터 항상 보이도록 복구한다.
- AI: Codex
- Status: draft

## Core Scope

- Goal: 사용자가 Wiki 메뉴에 들어갔을 때 좌측 목록 영역 위의 불필요한 헤더가 보이지 않고, 우측 상세 패널이 토글 없이 항상 노출되는 상태를 만든다.
- In Scope:
  - `knowledge-page-toolbar` 안의 `Wiki` 배지와 `목록` 라벨 헤더 영역을 제거한다.
  - `미리보기 열기` 버튼과 우측 패널 닫기 버튼처럼 Wiki 상세 패널을 숨기는 일반 UI 경로를 제거한다.
  - `isWikiPreviewOpen` 상태, `setIsWikiPreviewOpen` 호출, `knowledge-preview-pane--hidden` 렌더 경로를 제거하거나 비활성화해 상세 패널이 항상 렌더되게 한다.
  - Wiki 항목, handoff 항목, query 결과를 선택하면 기존처럼 같은 우측 패널에 본문이 표시되게 유지한다.
  - 선택 항목이 없는 첫 진입 상태에서는 기존 `LogPreview` 빈 상태 메시지가 우측 패널에 보이면 된다.
  - 관련 CSS에서 Wiki 전용 hidden/toggle/닫기 버튼 스타일을 정리하되, 다른 페이지의 동일 클래스 사용 여부를 확인해 회귀를 피한다.
- Out of Scope:
  - Wiki query, wiki synthesis, board 파일 읽기, markdown renderer IPC 동작은 바꾸지 않는다.
  - 로그 페이지, runner 화면, ticket workspace, statistics, workflow 등 다른 상세/미리보기 UI를 재설계하지 않는다.
  - `.autoflow/wiki/` 문서는 직접 수정하지 않는다.
  - 전체 desktop layout 리팩터링이나 shadcn 전환 범위 밖의 UI 컴포넌트 교체는 하지 않는다.

## Global Acceptance Criteria

- [ ] Wiki 메뉴 첫 진입 시 좌측 목록 상단에 작은 `Wiki` 배지와 `목록` 라벨 헤더가 더 이상 보이지 않는다.
- [ ] Wiki 메뉴 첫 진입 시 우측 상세 패널이 닫힌 상태가 아니라 항상 노출된다.
- [ ] 선택 항목이 없는 상태에서는 우측 상세 패널에 기존 빈 상태 메시지가 보이며, 패널 자체는 숨겨지지 않는다.
- [ ] `WikiList`, `HandoffList`, `WikiQueryPanel` 결과를 선택하면 우측 상세 패널이 닫히지 않고 선택한 markdown 본문을 표시한다.
- [ ] `미리보기 열기` 버튼과 Wiki 상세 패널 닫기 버튼이 일반 화면 경로에서 제거되어 사용자가 상세 패널을 접을 수 없다.
- [ ] `apps/desktop/src/renderer/main.tsx` 와 `apps/desktop/src/renderer/styles.css` 에서 Wiki 전용 `isWikiPreviewOpen`, `setIsWikiPreviewOpen`, `knowledge-preview-pane--hidden`, `knowledge-preview-open-toggle`, `log-preview-close` 경로가 남아 있지 않거나 더 이상 Wiki 패널 숨김에 쓰이지 않는다.
- [ ] 좁은 폭에서는 기존 반응형 규칙 안에서 좌우 영역이 사용 가능하게 유지되고, old hidden class 때문에 우측 패널이 `display: none` 이 되지 않는다.
- [ ] 다른 페이지의 `runner-page-summary`, `LogPreview`, 상세 패널 동작은 이번 Wiki 전용 변경으로 회귀하지 않는다.
- [ ] 구현은 Allowed Paths 두 파일 안에만 머문다.
