# Project

- ID: prd_158
- Title: Prompt caching 도입 — Anthropic API 직호출 전환으로 반복 prefix 캐싱 (30-50% 절감)
- AI: planner/worker
- Status: draft

## Core Scope

- Goal: claude CLI (`claude -p`) 가 stateless 라 prompt caching 미적용. Anthropic API 직호출로 전환하면 system prompt + agent guideline + 보드 컨텍스트 prefix 가 cache hit → 30-50% 토큰 절감.
- In Scope:
  - **Anthropic SDK 도입**:
    - 새 어댑터 추가: `agent = "anthropic-api"` (claude CLI 와 별개)
    - SDK: Node 또는 Python (Autoflow 가 Node 기반이라 `@anthropic-ai/sdk` 권장)
    - 인증: 환경변수 `ANTHROPIC_API_KEY` (기존 claude CLI 설정과 동일)
  - **Prompt 분할 + caching marker**:
    - Static prefix (system prompt + agent guideline): cache_control { type: "ephemeral" }
    - Dynamic suffix (current tick 입력): no caching
    - 모델: claude-sonnet-4-5 또는 사용자 선택 (config.toml `model` 매핑)
  - **마이그레이션 전략**:
    - planner / verifier 가 우선 사용 (claude CLI 사용 중인 runner)
    - 기존 claude CLI 어댑터 유지 (fallback). config.toml `agent = "claude"` 면 CLI, `agent = "anthropic-api"` 면 SDK.
  - **모니터링**:
    - cache_creation_input_tokens / cache_read_input_tokens 를 telemetry 에 기록
    - cache hit rate 통계 (PRD-129 와 정합)
- Out of Scope:
  - codex / gemini 어댑터 (별도 PRD — provider 다름)
  - cache TTL 자동 학습 (5분 ephemeral 만 사용)
  - multi-tier caching (1h cache 등)
