## Core Scope

- Goal: 3-runner 토폴로지의 runner id 를 카운터 접미사 없는 역할 친화 슬러그로 재명명한다. config / heartbeat / 데스크톱 UI / docs 의 모든 하드코딩된 id 를 일관되게 갱신하고, 기존 runner state/log 파일은 새 id 기준으로 이주(또는 신규 생성)되도록 한다.
- In Scope:
  - `planner-1` → `planner`
  - `owner-1` → `worker` (role 은 `ticket-owner` 그대로 유지)
  - `wiki-1` → `wiki-maintainer` (id 와 role 이 같은 문자열이 됨)
  - `.autoflow/runners/config.toml` 의 id 필드
  - `.autoflow/automations/heartbeat-set.toml` 의 owner_workers / planner_workers / wiki_workers / [thread_ids] / notes
  - 데스크톱 UI 의 하드코딩된 id-to-display 매핑 (main.tsx 의 `merge-1` / `wiki-maintainer-1` / `wiki-1` / `coordinator-1` 케이스 정리, 신규 슬러그가 "Plan AI" / "Impl AI" / "Wiki AI" 로 보이도록 보강)
  - AGENTS.md / CLAUDE.md 의 id 인용 부분
  - runner state 파일 마이그레이션 (기존 `planner-1.state`, `owner-1.state`, `wiki-1.state` 가 있으면 새 id 파일명으로 이동)
- Out of Scope:
  - role 명 자체 변경 (`planner` / `ticket-owner` / `wiki-maintainer` 그대로)
  - 토폴로지 변경 (여전히 3개 loop runner)
  - 기존 로그 파일 백필 (`runners/logs/*` 의 옛 id 로그는 historical 로 남김)
  - PowerShell 변형 추가 (현재 `.ps1` 는 pass-through shim 이라 자동 적용)

## Notes

- `wiki-maintainer` 는 id 와 role 이 같은 문자열이 된다. 스크립트는 두 키를 별도로 조회하므로 충돌 없음을 확인했지만, 데스크톱 UI 의 `^wiki-maintainer-\d+$` 같은 정규식은 매치되지 않으므로 신규 슬러그용 분기(또는 role-기반 라벨 fallback)를 추가한다.
- `worker` 는 의도적으로 역할명(`ticket-owner`)과 다르게 둔다 — 미래에 동일 role 을 여러 worker (`worker-1`, `worker-2`) 로 늘릴 때를 위한 명명 공간 확보 목적.
- Runner state 마이그레이션은 mv 로 처리하고, 마이그레이션이 끝난 뒤 옛 id 파일이 남아 있지 않은지 확인한다.
