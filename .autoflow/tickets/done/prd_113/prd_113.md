# PRD: 로그 페이지 리스트 머리글 영역 제거

- ID: prd_113
- Title: 로그 페이지 리스트 머리글 영역 제거
- Status: backlog
- Created: 2026-05-02
- Source: memo_073

## Goal

로그 페이지(logs section)의 좌측 패널 상단에 표시되는 머리글 영역(`로그` 제목, 로그 개수 카운트, 터미널 아이콘)을 제거하여, 패널 진입 시 곧장 로그 목록이 보이도록 개선한다. 이는 이전 `memo_041`에서 수행한 헤더 압축 작업을 넘어, 사용자 요청에 따라 해당 영역을 완전히 제거하는 작업이다.

## Context

사용자는 로그 페이지에서 불필요한 헤더 공간을 줄이고 목록을 더 넓게 보기를 원한다. 카운트 정보는 목록 하단의 "전체 N건 모두 보기" 버튼 등을 통해 여전히 유추 가능하므로 상단 헤더의 중복 정보를 제거해도 기능상 문제가 없다.

## Done When

- [ ] `apps/desktop/src/renderer/main.tsx` 내 로그 페이지 섹션에서 `<div className="section-heading compact log-list-heading">...</div>` 블록이 제거됨.
- [ ] 로그 목록(`LogList`)이 좌측 패널 상단부터 곧바로 렌더링됨.
- [ ] 로그 항목 선택, 미리보기 패널 표시 등 기존 로그 페이지 기능에 회귀가 없음.
- [ ] `apps/desktop/src/renderer/styles.css`에서 더 이상 사용되지 않는 `.log-list-heading`, `.log-heading-copy`, `.log-count-text` 스타일 정의가 (다른 곳에서 쓰이지 않을 경우) 제거됨.
- [ ] `npm run desktop:check` (또는 `tsc --noEmit`) 명령이 에러 없이 통과됨.

## Allowed Paths

- `apps/desktop/src/renderer/main.tsx`
- `apps/desktop/src/renderer/styles.css`

## Verification

- 데스크톱 앱 "로그" 섹션 진입 시 좌측 패널 상단에 "로그" 제목과 카운트가 보이지 않는지 확인.
- 로그 목록이 패널 상단에 밀착되어 표시되는지 확인.
- 로그 클릭 시 우측에 로그 내용이 정상적으로 표시되는지 확인.
