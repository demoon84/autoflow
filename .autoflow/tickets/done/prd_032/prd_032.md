- AI: Codex

## Core Scope

- Goal: The Wiki/Knowledge page should always show the right-side preview area in its normal desktop layout, reusing the existing split-pane structure from the previous Wiki preview flow.
- In Scope:
  - Change the Wiki page initial preview state from closed to open.
  - Keep the preview pane visible when entering the Wiki page, after selecting Wiki/Handoff/query results, and after returning from another settings section.
  - Remove or disable the Wiki-specific close/toggle path that hides the preview pane in normal desktop UI.
  - Preserve the existing left list/search/source pane and right preview pane split layout.
  - Preserve existing item selection behavior: selecting a WikiList, HandoffList, or WikiQueryPanel result still loads that item into the preview.
  - Keep narrow-width responsive behavior safe; if panes stack, the preview must still be visible rather than hidden by the old closed state.
- Out of Scope:
  - Do not redesign the Wiki page beyond the always-expanded preview behavior.
  - Do not change wiki query, wiki update, board file reading, or markdown rendering IPC behavior.
  - Do not change other settings sections, ticket workspace, workflow page, runner controls, or metrics UI.
  - Do not manually edit `.autoflow/wiki/`; wiki synthesis remains Wiki AI responsibility.
