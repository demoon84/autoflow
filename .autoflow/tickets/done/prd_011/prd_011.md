# Project PRD

## Meta

- Project Key: prd_011
- Title: Redesign Ticket Board as tabbed PRD and issued-ticket reading workspace
- Status: populated

## Goal

Redesign the desktop "티켓 보드" screen from a Kanban-style state board into a PRD-centered workspace for browsing PRDs and the tickets issued from them.

The user should be able to switch between PRD/ticket groups with tabs, select a ticket from the left-side list, and read the ticket or PRD markdown content in a right-side detail pane without opening a separate modal for the main reading flow.

## Core Scope

### In Scope

- Keep the existing sidebar "티켓 보드" menu.
- Replace the existing Kanban-column experience with a tabbed, two-pane reading UI.
- Use tabs for board navigation:
  - `전체`
  - `PRD`
  - `발급 티켓`
  - `진행 중`
  - `검증/완료`
  - `반려`
- Main layout:
  - Left pane: ticket/PRD list.
  - Right pane: selected item content rendered with `MarkdownViewer`.
- Left list item fields:
  - Display id (`PRD-NNN`, `tickets_NNN`, `Reject-NNN`)
  - Title
  - Status badge
  - PRD / project key when available
  - Assigned AI when available, displayed as `AI-N`
  - Last modified time
- Right detail pane:
  - Header with display id, title, status badge, project key, and modified time.
  - Markdown body rendered inline with `MarkdownViewer`.
  - Loading, error, and empty states.
- Data sources:
  - PRDs from `board.tickets.backlog` and archived `prd_*.md` / `project_*.md` files under `board.tickets.done`.
  - Tickets from `board.tickets.todo`, `board.tickets.inprogress`, `board.tickets.verifier`, `board.tickets.done`, and `board.tickets.reject`.
- Ticket-to-PRD relationship:
  - Prefer `Project Key` or `PRD Key` fields parsed from markdown.
  - Fall back to path context for done archives when possible.
  - Items with no detected PRD key still appear in `전체` and relevant status tabs.
- Selection behavior:
  - First available item in the active tab is selected automatically.
  - Changing tabs selects the first item in that tab when the previous selection is not visible.
  - Clicking a list item loads its file with `window.autoflow.readBoardFile`.
- Empty states:
  - If there are no PRDs or tickets: "표시할 PRD 또는 티켓이 없습니다."
  - If a tab has no items: "이 탭에 표시할 항목이 없습니다."
  - If no item is selected: "왼쪽 목록에서 항목을 선택하세요."
- Reuse existing shadcn/Radix tab components if available. If tabs are not yet present, add a small Radix-based wrapper under the existing UI component conventions.
- Keep the screen read-only.

### Out of Scope

- Drag-and-drop state transitions.
- Editing PRD or ticket markdown.
- Creating a new PRD from this screen.
- Triggering ticket-owner, verification, retry, or runner actions.
- Backend IPC signature changes.
- New file-watch or live-update behavior.
- Modal-first reading flow for this screen.
