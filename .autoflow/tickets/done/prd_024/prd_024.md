# Project PRD

## Project

- ID: prd_024
- Title: Convert ticket workspace right preview into a click-to-open layer like the workflow PRD pin
- AI:
- Status: draft

## Core Scope

- Goal: 데스크톱 `티켓 정보` 페이지의 우측 고정 미리보기 패널을 제거해 좌측 카드 리스트가 화면을 가득 차지하게 하고, 카드 클릭 시 작업 흐름 페이지의 PRD/반려 핀이 열리는 것과 같은 **dialog/layer 형태로 본문이 열리도록** 동작을 통일한다. 사용자가 한 번에 하나의 티켓 본문에 집중하고, 평소에는 카드 리스트가 넓게 보이도록 한다.
- In Scope:
  - **레이아웃 변경** (`apps/desktop/src/renderer/main.tsx`):
    - 티켓 정보 페이지의 grid layout 에서 우측 preview column 을 제거. 좌측 카드 리스트가 가용 폭 100% 사용.
    - 페이지 헤더(`마지막 업데이트 ...`, 탭 줄 등) 는 유지.
  - **카드 클릭 → 레이어 오픈**:
    - 각 ticket / PRD 카드 (`Ticket-014`, `PRD-022` 등) 가 button 으로 동작하도록 wrapping. 클릭 시 **shadcn Dialog 기반 레이어** 가 열리며 해당 ticket/PRD 의 markdown 본문 + 메타데이터(Stage, Owner, 경로) 를 보여준다.
    - 작업 흐름 페이지의 `WorkflowPinLayer` 컴포넌트와 같은 인터랙션 패턴 (좌측 헤더 + 본문 스크롤 + ESC/배경 클릭으로 닫기). 가능하면 컴포넌트를 재사용하거나 동일 base (`Dialog`, `MarkdownViewer`) 위에서 새 컴포넌트(`TicketDetailLayer`) 작성.
  - **레이어 본문 구성**:
    - 헤더: 카드와 동일한 식별자(예: `Ticket-014`), 단계 badge, 마지막 업데이트, 파일 경로.
    - 메타: ID / PRD Key / Stage / AI / Claimed By / Last Updated 라인을 한 줄짜리 키-값 그리드로 표시.
    - 본문: 티켓 markdown 의 `## Goal`, `## References`, `## Allowed Paths`, `## Done When`, `## Next Action`, `## Notes` 등 표준 섹션을 `MarkdownViewer` 로 렌더.
    - 닫기 버튼 + ESC + 배경 클릭 모두로 닫힘.
  - **상태 관리**:
    - 페이지 단위 상태에 `activeTicketLayerId: string | null` 추가. 카드 클릭 시 set, 닫을 때 null.
    - URL/localStorage 동기화는 본 PRD 범위 밖 (단순 ephemeral 상태).
  - **키보드 / 접근성**:
    - 카드는 `<button>` 또는 role=button 으로, focus ring 표시.
    - 레이어 오픈 시 focus trap 작동(shadcn Dialog 기본 동작).
- Out of Scope:
  - PRD 카드와 ticket 카드의 시각 디자인 변경 (현재 그대로 유지).
  - 레이어 안에서의 ticket 편집 (읽기 전용).
  - 카드 우상단의 stage badge 색/문자 변경.
  - 작업 흐름 페이지 변경.
  - 다른 페이지(`통계`, `Wiki`, `AI 관리`) 의 레이아웃.

## Global Acceptance Criteria

- [ ] 티켓 정보 페이지 진입 시 우측 미리보기 column 이 보이지 않고, 카드 리스트가 화면 폭 전체를 사용한다.
- [ ] 어느 카드 (`Ticket-NNN` 또는 `PRD-NNN`) 든 클릭하면 dialog 레이어가 열려 해당 항목의 본문이 표시된다.
- [ ] 레이어 헤더에 카드와 동일한 식별자, 단계 badge, 마지막 업데이트, 파일 경로가 보인다.
- [ ] 레이어 본문이 ticket / PRD markdown 의 표준 섹션을 정상 렌더한다 (`MarkdownViewer` 와 동일 톤).
- [ ] ESC, 배경 클릭, 닫기 버튼 모두로 레이어가 닫힌다.
- [ ] 레이어 오픈 시 focus 가 닫기 버튼 또는 본문 첫 element 로 이동하고, 닫힌 뒤 클릭한 카드로 focus 복귀.
- [ ] 다른 카드를 클릭하면 기존 레이어 내용이 새 카드 내용으로 교체되거나 (또는 닫힘 후 다시 오픈) 깜빡임 없이 자연스럽게 전환된다.
- [ ] 탭 전환 (`PRD` ↔ `발급 티켓`) 시 열려 있던 레이어가 닫히거나 / 그대로 유지되는 정책이 일관된다 (권장: 탭 전환 시 자동 닫힘).
- [ ] `cd apps/desktop && npx tsc --noEmit` 가 0 errors.
- [ ] `cd apps/desktop && npm run check` 가 통과한다.
- [ ] 시각 회귀: 작업 흐름·통계·Wiki·AI 관리 페이지 영향 없음.

## Notes

- `WorkflowPinLayer` 가 이미 동일 패턴(Dialog + 헤더 + MarkdownViewer + ESC/배경 닫기) 을 구현해 두어 새 컴포넌트는 사실상 그것의 변형. 가능한 부분은 wrap.
- ticket 의 markdown 경로는 `board.tickets[stage]` 에서 가져오는 file path. 이미 readLog 같은 IPC 가 있다면 그대로 사용 (지연 로딩) 또는 board 로드 시 미리 inline content 가 있다면 prop 로 전달.
- 닫힘/오픈 애니메이션은 shadcn Dialog 기본 transition 그대로 — 별도 커스텀 모션 없음.
- prd_019 (티켓 탭 PRD/발급 티켓 둘만) 와 prd_023 (좌측 보더 제거) 와 변경 영역 겹침 가능성: 모두 `apps/desktop/src/renderer/main.tsx` 의 티켓 정보 페이지 부근. 머지 순서 prd_019 → prd_023 → prd_024 가 충돌 최소.
- 레이어 안 본문에 코드 블록이 길 때 가로 스크롤이 자연스럽게 되도록 `MarkdownViewer` 의 기본 overflow 정책 유지.
