# Project PRD

## Project

- ID: prd_135
- Name: runner self-resurrect 복구
- Title: desktop 재시작 후 사용자 명시 stop 이 아닌 runner 자동 부활
- Goal: desktop app 재시작, dev reload, 호스트 재기동 후 enabled runner 가 `status=stopped` / `last_result=loop_stopped` 상태로 방치되지 않도록, 사용자 명시 stop 과 비의도 parent 종료를 구분하고 desktop startup 경로에서 runner 를 자동 복구한다.
- AI: planner
- Status: generated

## Core Scope

- Goal: Autoflow 1원칙에 맞게 desktop process 가 runner lifecycle owner 로 동작하더라도, desktop 재기동 시 사용자 명시 정지가 아닌 stopped enabled runner 를 다시 running 상태로 회복시킨다.
- In Scope:
  - `packages/cli/runners-project.sh` 의 `stop` 경로가 사용자 명시 stop 을 state 에 기록할 수 있도록 `stopped_by=user` 또는 동등한 marker 를 저장한다.
  - `loop-worker` 의 `TERM` / `INT` trap 또는 desktop shutdown 경로가 비의도 parent 종료와 구분 가능한 stop reason 을 state 에 남긴다. 최소한 `last_result=loop_stopped` 만으로 사용자 명시 stop 으로 간주하지 않는다.
  - `apps/desktop/src/main.js` startup / project scope registration 경로가 enabled runner 목록과 state 를 읽고, `status=stopped` 이면서 사용자 명시 stop marker 가 없는 runner 에 대해 `bin/autoflow runners start <id> <projectRoot> <boardDirName>` 를 호출한다.
  - desktop 의 quit/shutdown cleanup 은 runner 를 종료할 수 있지만, 그 종료가 다음 desktop startup 에서 자동 복구 가능한 non-user stop 으로 남아야 한다.
  - runner self-heal 는 중복 `start` 호출을 만들지 않도록 기존 `runnerControlInflight` 또는 동등한 per-runner guard 를 사용한다.
  - 기존 UI stop 버튼 또는 `autoflow runners stop` 의 사용자 정지 의도는 유지되어야 하며, 사용자가 명시적으로 멈춘 runner 는 desktop restart 만으로 자동 시작하지 않는다.
- Out of Scope:
  - product feature 구현 외 launchd / systemd 배포 파일 추가.
  - runner interval, adapter timeout, 모델 설정 변경.
  - ticket-owner / verifier / wiki 구현 로직 변경.
  - runner process kill 정책의 전면 재설계.
  - `git push`.

## Global Acceptance Criteria

- [ ] `packages/cli/runners-project.sh stop` 이 runner state 에 사용자 명시 stop marker (`stopped_by=user` 또는 동등한 key) 를 기록한다.
- [ ] `loop-worker` 가 `TERM` / `INT` 로 정지될 때 사용자 명시 stop marker 를 무조건 남기지 않고, desktop parent 종료와 구분 가능한 state key (`last_stop_reason=parent_terminated` 또는 동등한 key) 를 기록한다.
- [ ] `apps/desktop/src/main.js` 가 startup 또는 project scope 등록 직후 enabled runner 중 `status=stopped` 이고 사용자 명시 stop marker 가 없는 runner 를 자동으로 `runners start` 한다.
- [ ] `apps/desktop/src/main.js` self-heal 경로가 동일 runner 에 대해 병렬 중복 start 를 만들지 않는다.
- [ ] 사용자가 UI stop 또는 CLI stop 으로 명시 정지한 runner 는 desktop restart self-heal 에 의해 자동 시작되지 않는다.
- [ ] desktop quit/reload 로 인해 `last_result=loop_stopped` 가 기록된 runner 는 다음 desktop startup self-heal 대상이 된다.
- [ ] `runner_status_count` 에서 `runner_stopped_count` 가 양수인 상태라도, enabled + non-user stopped runner 는 self-heal 후 5초 안에 `status=running` 으로 전환된다.
- [ ] `node --check apps/desktop/src/main.js` 와 `bash -n packages/cli/runners-project.sh` 가 exit 0 으로 통과한다.
