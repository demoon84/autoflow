# Project

- ID: prd_159
- Title: Differential context — planner 가 보드 전체 대신 변경분만 + 이전 tick 요약 전달
- AI: planner/worker
- Status: draft

## Core Scope

- Goal: planner 가 매 tick 보드 전체 markdown (backlog/todo/inprogress/reject) 을 prompt 에 포함하던 것을 변경분 + 이전 tick 요약으로 전환해 input 토큰 20-40% 절감 (planner avg 105K → 60K 이하 목표).
- In Scope:
  - **Diff 계산**:
    - 직전 tick 의 보드 매니페스트 hash + summary 저장.
    - 현재 tick 에서 매니페스트 비교 → added/modified/removed 파일 목록.
  - **Prompt 구성**:
    - Header: 직전 tick 요약 (이전 tick 의 결정 / 처리 결과 한 줄)
    - Diff section: 변경된 파일 목록 + 본문 (full or excerpt)
    - Optional: 변경 안 된 파일 목록만 (제목 + path, 본문 elide)
  - **Fallback 정책**:
    - 첫 tick / context reset 시 전체 보드 전달.
    - Diff 가 보드 30% 이상이면 전체 전달 (의미 없는 partial).
    - planner 가 "전체 컨텍스트 필요" 시그널 (output marker) 보내면 다음 tick 전체 전달.
  - 환경변수:
    - `AUTOFLOW_PLANNER_DIFFERENTIAL_ENABLED` (기본 0, opt-in)
    - `AUTOFLOW_PLANNER_DIFFERENTIAL_FULL_THRESHOLD_PERCENT` (기본 30)
- Out of Scope:
  - worker / verifier 의 differential context (별도 PRD — 각자 scope 다름).
  - prompt 자동 요약 (LLM 호출 1회 추가) — 단순 diff 만.
