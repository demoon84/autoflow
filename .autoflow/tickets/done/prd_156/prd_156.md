# Project

- ID: prd_156
- Title: Tick interval 동적 조정 — idle 비율 높은 runner 의 tick 간격 자동 확장
- AI: planner/worker
- Status: draft

## Core Scope

- Goal: idle 비율이 높은 시기에 runner tick interval 을 자동 확장 (1분 → 2분 → 5분) 해 호출 빈도 자체를 낮춰 토큰 누계 절감 + 호스트 자원 사용 감소.
- In Scope:
  - **점진적 backoff 정책**:
    - 연속 N회 (기본 5) idle (`*_inputs_unchanged`) → tick interval × 2 (cap: 5분).
    - 입력 변경 감지 → tick interval 즉시 원래 값으로 복귀.
  - 환경변수:
    - `AUTOFLOW_TICK_BACKOFF_ENABLED` (기본 1, 0 으로 끔)
    - `AUTOFLOW_TICK_BACKOFF_THRESHOLD_IDLE_TICKS` (기본 5)
    - `AUTOFLOW_TICK_BACKOFF_MAX_INTERVAL_SECONDS` (기본 300)
  - state 기록: `current_interval_seconds`, `idle_streak_count`.
  - 데스크톱 표시: 현재 동적 interval 시각 (선택).
- Out of Scope:
  - 사용자 명시적 trigger (예: 수동 plan/todo) 가 있을 때 backoff 무시 — 즉시 처리.
  - Burst detection (입력 폭증 시 interval 축소 < 1분) — 후속.
