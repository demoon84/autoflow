## Core Scope

- Goal: `tickets/inbox/memo_045.md`의 요청을 데스크톱 최근 프로젝트 경로 검증 및 fallback 개선 작업으로 승격한다.
- In Scope:
  - `localStorage["autoflow.recentProjects"]`에서 읽은 최근 프로젝트 목록에 존재하지 않는 경로가 섞여 있어도 앱이 빈 화면으로 멈추지 않게 한다.
  - 최근 프로젝트 메뉴와 프로젝트 선택 overlay의 목록 항목을 표시하거나 선택할 때 경로 존재 여부를 확인하고, 존재하지 않는 경로는 자동 제거하거나 disabled/안내 상태로 처리한다.
  - 존재하지 않는 경로를 선택한 경우 `options.projectRoot`를 그대로 고정하지 않고, 사용자에게 오류 toast 또는 안내를 보여주며 프로젝트 폴더 선택 화면으로 돌아갈 수 있게 한다.
  - `persistRecentProjects`에 저장되는 목록에서 검증 결과 사라진 경로가 다시 유지되지 않도록 정리한다.
  - `readBoard` 응답 또는 새 경량 IPC 중 가장 좁은 방식으로 프로젝트 루트 존재 여부를 renderer가 알 수 있게 한다.
  - board directory가 없어서 설치가 필요한 정상 케이스와 project root 자체가 없는 케이스를 UI 상태에서 구분한다.
- Out of Scope:
  - 최근 프로젝트 목록 외 다른 메뉴, navigation 순서, runner 제어, ticket/wiki/log 화면 동작 변경.
  - 프로젝트 설치 flow 자체의 설계 변경이나 board scaffold 생성 로직 변경.
  - 입력/버튼 크기, 레이어 깜박임, 티켓 디테일 메타 그리드 같은 다른 inbox 요청과의 결합.
  - localStorage key 이름 변경 또는 기존 정상 경로 목록의 포맷 migration.
