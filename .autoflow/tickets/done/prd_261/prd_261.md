# PRD

## Metadata

- PRD Key: prd_261
- Title: Ticket ownership lock을 stable runner-id + liveness hybrid로 전환
- Priority: high
- Status: ready
- Source: order_230
- Created At: 2026-05-10T20:56Z

## Problem

`start-ticket-owner.sh`가 ticket의 ownership 라인에 PTY 세션마다 새로 생성된 thread UUID를 써넣는다. 다음 PTY 세션이 같은 ticket을 보면 "다른 owner가 claim 중"으로 거부한다. PTY가 재시작될 때마다 UUID가 바뀌어 재시작 직후 모든 inprogress/부분 claim ticket이 영원히 stuck 된다. 실제로 Todo-259/263/264가 stale UUID 때문에 막힌 사고가 발생했다.

## Goal

ticket ownership lock을 **stable runner-id (worker/planner/wiki/worker-N) + liveness check (kill -0)** 의 hybrid로 전환한다. PTY 재시작 시 같은 runner-id가 자동 takeover하고, 진짜 다른 runner가 alive PID를 가진 경우만 claim을 거부한다.

## Scope

### Ownership 라인 포맷 변경

```
- Result: pending ticket-owner by <runner-id>:<runner-pid>:<spawned-at-iso>
```

레거시 UUID 라인은 1회 호환 파싱 후 무조건 stale로 처리(takeover).

### Claim 검사 흐름 (start-ticket-owner.sh)

- **case A** — 비어있음: 즉시 claim 가능
- **case B** — runner-id가 본인과 동일: takeover 허용, `event=ticket_claim_takeover_same_runner` 로그
- **case C** — runner-id가 다른 runner: `kill -0 <pid>`로 liveness 확인. alive→거부, dead→takeover(`event=ticket_claim_takeover_dead_owner`)
- **case D** — 같은 runner-id인데 PID 불일치(PTY 재시작): 이전 PID 죽었으면 takeover, 살아있으면 거부(동시 두 인스턴스 사고 방지)

### Finish 연동

`finish-ticket-owner.sh pass/fail` 시 ownership 라인을 정리해 다음 worker가 claim 가능한 상태로 둔다.

### Audit trail

takeover 발생 시 ticket `Notes`에 1줄 추가: `stale lock takeover by <new-runner>:<new-pid> at <iso>`

## Execution Candidates

### Candidate 1: start/finish-ticket-owner.sh ownership lock hybrid 전환

start-ticket-owner.sh의 claim 단계를 case A-D 로직으로 교체하고, finish-ticket-owner.sh의 pass/fail에서 ownership 라인을 정리한다. runner-common.sh에 liveness check 헬퍼 추가. runtime/board-scripts/ 미러 동기화. 기존 stale UUID ticket들이 다음 tick에서 자동 takeover되어 stuck 해소.

## Allowed Paths

- `.autoflow/scripts/start-ticket-owner.sh`
- `.autoflow/scripts/finish-ticket-owner.sh`
- `.autoflow/scripts/runner-common.sh`
- `runtime/board-scripts/start-ticket-owner.sh`
- `runtime/board-scripts/finish-ticket-owner.sh`
- `runtime/board-scripts/runner-common.sh`

## Global Acceptance Criteria

- ticket ownership 라인 포맷이 `<runner-id>:<runner-pid>:<spawned-at-iso>` 형태
- 같은 runner-id가 PID 다른 채로 재claim 시 takeover (PTY 재시작 정상 케이스)
- 다른 runner-id가 alive PID를 가지고 있을 때만 claim 거부
- stale UUID ticket들이 다음 tick에서 자동 takeover
