# Planner Commit Burst Mitigation

## Problem

Excessive and frequent Git commits were being generated by the Autoflow planner during cleanup, wiki updates, and runtime processes, particularly as observed in `order_152.md`. This "commit burst" led to operational overhead and obscured meaningful changes in the Git history.

## Decision

To address the commit burst issue identified in PRD-172, a series of mitigations were implemented targeting the Autoflow planner and its related components. The core decision was to reduce unnecessary commit generation and improve the traceability of adapter-related issues.

## Implementation Details

Key aspects of the implementation include:

*   **Cleanup Batching:** Consolidating dirty paths during planner cleanup into single commits to reduce commit volume.
*   **Wiki Debounce Enhancement:** Ensuring that wiki metric-only page updates do not bypass the debounce count, thereby preventing unnecessary commits for minor changes. This involves improvements to `packages/cli/wiki-project.sh`.
*   **Adapter Watchdog & Telemetry:** Enhancements to adapter watchdog functionality to accurately distinguish between graceful exits and actual adapter timeouts/truncations. This includes logging `output_truncated=true` to track instances where adapter output is truncated. This capability, likely involving `packages/cli/run-role.sh`, allows for better diagnosis of issues without accumulating commits from incomplete or failed adapter runs. This builds upon previous work from `prd_154` (output caps) and `prd_168` (check ledger live-lock fix).

## Impact

This decision and its implementation are expected to:

*   Significantly reduce the volume of Git commits generated by the Autoflow system.
*   Improve the clarity and signal-to-noise ratio of the Git history.
*   Provide better diagnostic capabilities for adapter-related issues (e.g., timeouts, output truncation) by preventing commits from being accumulated without successful completion.
*   Enhance overall system stability and operational efficiency.
