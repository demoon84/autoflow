# worker.state.last_result 잔류 자가 리셋

## One-liner

worker.state.last_result=ticket_stage_blocked 잔류 자가 reset

## Summary

PROJECT_ROOT의 dirty path 정리 및 blocked-dirty orchestration cleanup 완료 후에도 `worker.state.last_result`에 `ticket_stage_blocked`가 stale 상태로 남아 모니터/데스크톱 UI에 worker가 계속 "blocked"로 표시되는 문제를 해결합니다. worker tick 시작 시 자가 reset과 planner cleanup 후 명시적 reset을 동시에 적용하여 user-visible "blocked" 표시가 실제 상태와 동기화되도록 합니다.

## Entities

- Planner orchestration runtime (Module): .autoflow/scripts/start-plan.sh, runtime/board-scripts/start-plan.sh
- Shared planner/worker helpers (Module): .autoflow/scripts/common.sh, runtime/board-scripts/common.sh
- Worker tick dispatcher (Module): packages/cli/run-role.sh
- planner (AI)

## Concepts

- planner cleanup 명시적 reset: blocked-dirty orchestration cleanup 종료 시 `worker.state`의 stale `ticket_stage_blocked`를 명시적으로 비우는 후처리
- race-safe 동작: worker/planner 동시 tick 시 `last_result`만 다루고 다른 상태 필드는 건드리지 않음
- worker tick 자가 reset: worker tick 시작 시 `last_result`를 빈 값 또는 `idle`로 리셋하는 로직
- worker.state.last_result: `ticket_stage_blocked` 값이 stale 상태로 남아 모니터/데스크톱 UI에 "blocked"를 표시하는 문제
