# 플래너 커밋 폭증 완화

## One-liner

cleanup, wiki, runtime 커밋 폭증을 완화하고 worker adapter timeout/truncation 증거를 추적 가능하게 만들어 불필요한 commit 생성을 차단합니다.

## Summary

이 프로젝트는 `order_152.md`에서 보고된 시간당 과도한 cleanup, wiki, runtime commit 생성을 줄이는 것을 목표로 합니다. 주요 목표는 플래너 cleanup 시 더티 경로를 단일 커밋으로 묶고, 위키 메트릭 전용 페이지 갱신 시 디바운스 카운트를 무력화하지 않게 하며, 어댑터 타임아웃/트렁케이션 증거를 정상 종료와 구분하여 실제 구현 없이 보드/위키/텔레메트리 커밋만 누적되는 상태를 차단하는 것입니다.

## Concepts

- SIGTERM: 프로세스 종료 시그널
- output_truncated=true: 어댑터 출력의 잘림을 나타내는 로그 필드
- 디바운스 (debounce): 위키 업데이트 시 의미 없는 변경에 대한 커밋을 줄이는 기법
- 어댑터 워치독 (adapter watchdog): 어댑터의 타임아웃 및 출력 잘림을 감지하고 분류하는 기능
- 커밋 폭증: 시간당 과도하게 발생하는 git commit
- 클린업 배치 (cleanup batching): 플래너 클린업 시 여러 dirty path를 하나의 커밋으로 묶는 처리 방식
- 텔레메트리 요약 (telemetry summary): runner-timing, runner-health, prompt-evolution과 같은 메트릭 전용 위키 페이지 갱신 내용
