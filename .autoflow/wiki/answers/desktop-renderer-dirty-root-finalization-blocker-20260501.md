# desktop-renderer-dirty-root-finalization-blocker-20260501

## Answer

`reject_071`이 남긴 핵심 제약은 visible criteria 가 이미 `PROJECT_ROOT`의 넓은 `renderer` rewrite 안에서 만족되더라도, ticket 전용 current-head diff 가 없고 stale worktree 만 남아 있으면 pass finalization 을 하면 안 된다는 점이다. `setupRequired` 관련 선행 범위는 `prd_073`의 상단 `FolderPlus` 제거와 `prd_074`의 `progress` 화면 고정으로 충분하며, dirty root 는 `prd_006` 규칙대로 Allowed Paths 와 충돌하지 않는 unrelated 변경일 때만 허용된다. 따라서 같은 패턴이 재발하면 broad rewrite 를 먼저 land 하거나, 최신 HEAD 기준의 isolated diff 를 새 티켓으로 재생성한 뒤 검증을 다시 해야 한다.
