## Core Scope

- Goal: claude-code 가 이번 세션에서 수동으로 수행한 60초 polling monitoring 패턴을 Autoflow 자체 기능으로 표준화해, 사용자가 별도 감시 스크립트를 돌리지 않아도 자율 흐름의 운영 건강성을 확인하고 문제 신호를 후속 order/check 로 닫힌 루프에 넣는다.
- In Scope:
  - 기본 runner topology 에 별도 `monitor` runner 를 추가한다. 기본 role 은 `monitor`, 권장 agent 는 `gemini`, mode 는 `loop`, interval 은 `AUTOFLOW_MONITOR_INTERVAL_SECONDS` 기본 60초와 정합시킨다.
  - `packages/cli/run-role.sh` 와 `runtime/board-scripts/run-role.sh` 에 `monitor` runtime role 을 추가하고, monitor tick 이 `start-monitor.sh` / `monitor-project.sh` 출력의 `status=`, `source=`, `signal_count=`, `order_created=` 같은 key=value evidence 를 남기게 한다.
  - `packages/cli/monitor-project.sh` 를 신설해 8개 감시 항목을 측정한다: runner stale, repeated `last_result`, process pressure summary, metrics/token-cache stale 또는 token quota sanity, inbox/inprogress/reject 적체, dirty `PROJECT_ROOT` count/baseline, 정확한 `Recovery State: needs_user` 패턴, 이전 baseline 과 다른 anomaly 후보.
  - 신호 검출 시 중복 noise 를 막기 위해 fingerprint/cooldown state 를 `.autoflow/runners/state/monitor.*` 아래에 기록하고, 임계 이상인 새 신호만 `source: autoflow-monitor-agent` frontmatter 를 가진 order 로 발행한다. 자동 개입 결과 기록이 더 맞는 경우에는 기존 `check_NNN.md` 형식과 `record_orchestration_check` 계약을 재사용한다.
  - Cross-verification 정책을 넣어 단일 source 는 `confidence=hypothesis`, 복수 source 가 같은 root cause 를 가리키면 `confidence=confirmed` 로 분류한다. telemetry token 단위 불일치처럼 source 간 차이가 큰 경우 원본 값과 비율을 order 본문에 남긴다.
  - `AUTOFLOW_MONITOR_ENABLED`, `AUTOFLOW_MONITOR_INTERVAL_SECONDS`, `AUTOFLOW_MONITOR_STUCK_THRESHOLD_SECONDS`, `AUTOFLOW_MONITOR_REPEAT_FAIL_THRESHOLD`, `AUTOFLOW_MONITOR_AUTO_ORDER_ENABLED`, `AUTOFLOW_MONITOR_COOLDOWN_SECONDS` 환경변수를 지원한다.
  - `bin/autoflow monitor scan <project-root> <board-dir-name>` 또는 동등한 CLI entrypoint 를 추가해 monitor tick 을 수동/테스트에서 실행할 수 있게 한다.
  - `.autoflow/agents/monitor-agent.md`, `AGENTS.md`, `.autoflow/README.md`, `.autoflow/automations/README.md` 에 monitor 의 역할, 경계, no-push/no-kill/no-runner-stop 정책, auto-order/check 발행 정책을 문서화한다.
  - package/scaffold 경로가 새 monitor agent/script/CLI 자산을 설치 board 에 포함하도록 `packages/cli/package-board-common.sh` 와 runtime mirror 를 동기화한다.
  - Desktop 은 AI 진행 현황 또는 통계 화면에서 monitor runner 상태, 최근 monitor signal count, 자동 발행된 order/check 목록으로 이동할 수 있는 표시를 제공한다. 기존 `checks` 메뉴와 runner health toast 를 재사용하되, monitor 전용 카드/라벨을 추가한다.
- Out of Scope:
  - monitor 가 runner/process 를 직접 kill/stop/restart 하거나 OS process cleanup 을 수행하는 작업.
  - emergency stop, circuit breaker halt, token budget enforcement, prompt injection sanitizer 의 전체 구현. 이번 ticket 은 그런 신호를 검출하고 order/check 로 발행하는 closed loop 를 만든다.
  - Hermes self-learning skill 자동 활용까지 결합하는 작업. `order_146` 계열의 skill lifecycle 은 이미 별도 흐름으로 다룬다.
  - 외부 watchdog, Slack/email/webhook 알림, 분산/multi-host monitoring.
  - 현재 dirty `.autoflow/wiki/` 페이지나 기존 runner state 를 정리하는 작업.
