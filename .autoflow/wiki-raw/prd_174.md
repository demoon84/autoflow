# Project PRD

## Project

- ID: prd_174
- Name: runner config apply feedback
- Priority: high
- Title: Runner config 저장 적용 대기 피드백
- Goal: 데스크톱에서 AI runner 설정 저장 IPC 응답을 실제 적용 완료로 오해하지 않도록, runner state 에 적용 시점 증거를 기록하고 저장 버튼/카드가 새 설정 적용 전까지 명확한 대기 상태를 유지하게 한다.
- AI: planner
- Status: generated

## Core Scope

- Goal: `tickets/inbox/order_158.md` 의 요구를 데스크톱 runner 설정 저장 흐름에 좁게 적용한다. 저장 IPC 성공은 "config.toml 쓰기 완료"로만 취급하고, 실제 runner tick 이 새 설정을 읽은 증거가 state 에 기록될 때까지 UI 는 적용 대기 상태를 유지한다.
- In Scope:
  - runner tick 시작 시 현재 runner config fingerprint 와 주요 적용값(`agent`, `model`, `reasoning`)을 state 에 기록하고, config fingerprint 가 바뀐 tick 에 `config_applied_at=<ISO>` 계열 증거를 남긴다.
  - `configureRunner` / board snapshot 경로가 저장 직후 UI 에 필요한 `config_updated_at` 또는 saved fingerprint 와, runner state 의 `config_applied_at` / applied fingerprint 를 읽을 수 있게 한다.
  - renderer 의 config save action 을 `config_applying` 같은 transition action 으로 분리하고, 새 state 적용 증거가 도착하거나 timeout 될 때까지 저장 버튼과 같은 runner 의 다른 단발 액션을 disabled 로 유지한다.
  - 저장 후 카드에 적용 대기 badge / spinner / 한국어 토스트를 표시하고, 적용 감지 시 badge 제거와 완료 토스트를 표시한다.
  - 명시적 사용자 선택으로만 동작하는 "저장하고 재시작" 경로를 추가하거나, 기존 restart action 과 안전하게 결합할 수 있는 UI affordance 를 제공한다.
- Out of Scope:
  - order_147 의 graceful stop 런타임 의미 변경과 order_148 의 start/stop transition disabled 구현 자체.
  - runner agent/model/reasoning 선택지 정책 변경.
  - adapter 실행, ticket owner claim, planner recovery, wiki commit 정책 변경.
  - 자동 runner restart 를 기본값으로 만드는 동작.

## Global Acceptance Criteria

- [ ] `packages/cli/run-role.sh` 또는 동등한 runner tick 진입부가 현재 runner config fingerprint 와 적용값을 `.autoflow/runners/state/<runner>.state` 에 기록하고, fingerprint 변화가 감지된 tick 에 `config_applied_at=<ISO>` 또는 동등 필드를 갱신한다.
- [ ] `apps/desktop/src/main.js` 의 `autoflow:configureRunner` / `autoflow:readBoard` 경로가 저장된 config fingerprint 또는 updated timestamp 와 runner state 의 applied fingerprint / `config_applied_at` 을 renderer 가 비교할 수 있는 형태로 전달한다.
- [ ] `apps/desktop/src/renderer/main.tsx` 에서 설정 저장 클릭 후 action key 가 `config_applying` 계열로 유지되고, 새 설정 적용 증거가 board polling 또는 state refresh 로 확인되기 전까지 저장 버튼과 같은 runner 의 config/run/dry-run/start/stop/restart 액션이 중복 실행되지 않는다.
- [ ] 적용 대기 중 저장 버튼은 spinner 와 `저장 중...` 또는 `적용 대기...` 라벨을 표시하고, runner 카드에는 적용 대기 badge 가 표시된다.
- [ ] 적용 확인 시 action key 가 해제되고 완료 토스트가 표시되며, timeout(`interval_seconds + 30s` 또는 최소 90초) 시 action key 가 해제되고 경고 토스트가 표시된다.
- [ ] "저장하고 재시작"은 명시 선택 시에만 실행되며, 저장 IPC 성공 후 restart를 이어 실행하고 state 가 running + 새 applied fingerprint 를 보일 때까지 transition UI 를 유지한다.
- [ ] 다중 runner 저장은 runner id 별로 독립 추적되고, 한 runner 의 적용 대기가 다른 runner 의 설정 저장을 막지 않는다.
