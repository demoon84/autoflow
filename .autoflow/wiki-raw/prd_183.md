## Core Scope

- Goal: wiki 자동화의 debounce/admission/autocommit 경로가 파일 종류, diff 내용, line 변화량, 신규/삭제 여부를 기준으로 commit 여부를 판단하게 하여 git history noise 와 불필요한 wiki adapter 호출을 줄인다.
- In Scope:
  - `packages/cli/run-role.sh` 의 wiki loop debounce 판단을 `AUTOFLOW_WIKI_DEBOUNCE_MIN_CHANGES` 단순 개수 기준에서 의미 있는 file-weight 기준으로 대체하거나 보강한다.
  - wiki scoped autocommit 직전 diff 를 검사해 zero-weight 파일만 변경된 경우, whitespace-only/cosmetic-only diff 인 경우, 또는 line 변화량이 임계 미만인 경우 commit 을 건너뛰고 runner log/state 에 skip evidence 를 남긴다.
  - 기본 weight 정책을 추가한다: `wiki/index.md`, `wiki/log.md`, `*.manifest`, `*.history`, `*.fingerprint` 는 commit trigger 0점, `wiki/sources/`, `wiki/operations/`, `wiki/architecture/`, `wiki/agents/` 는 1점, `wiki/features/`, `wiki/decisions/` 는 3점, `wiki/answers/`, `wiki/learnings/`, `wiki/skills/`, `wiki/skills-local/` 는 5점으로 본다.
  - `AUTOFLOW_WIKI_COMMIT_WEIGHT_THRESHOLD` 기본값은 5로 두고, `AUTOFLOW_WIKI_COMMIT_MIN_LINES` 기본값은 30으로 둔다. 신규 파일 추가와 파일 삭제는 line 임계와 무관하게 의미 있는 변화로 인정한다.
  - commit 이 발생할 때 subject 를 `[wiki] update: <primary category summary> / <total pages> total, +<add>/-<del>` 형식으로 만들어 어떤 wiki content 가 바뀌었는지 git log 에서 바로 보이게 한다. PRD key 가 diff 에 포함된 경우에도 subject 는 wiki category summary 를 포함해야 한다.
  - wiki runner agent 문서와 `AGENTS.md` rule 18 을 새 정책에 맞게 갱신한다.
  - smoke test 를 추가하거나 갱신해 temp git repo 에서 zero-weight diff, cosmetic-only diff, low-weight 단일 수정, high-weight answer 추가, 파일 삭제, 의미 있는 commit subject 를 각각 검증한다.
- Out of Scope:
  - wiki content 자체의 새 지식 작성, 기존 wiki 페이지 내용 재작성, 또는 `.autoflow/wiki/` 현재 dirty 파일 정리.
  - ticket finalizer 가 `.autoflow/wiki/` 를 stage 하게 되돌리는 변경.
  - wiki semantic lint, ingest, RAG 검색 알고리즘 재설계.
  - 과거 `[wiki] wiki knowledge update` commit rewrite, rebase, squashing, 또는 git history 정리.
  - runner process stop/start/restart, OS process cleanup, `git push`.
