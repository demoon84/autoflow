## Core Scope

- Goal: `tickets/inbox/order_176.md` 의 runner control UX 요구를 데스크톱 UI와 필요한 main-process state 관찰 경로에 좁게 적용한다. `runnerActionKeys` 를 단순 IPC inflight tracker 가 아니라 runner state transition 의도 tracker 로 다루고, state 가 목표 상태에 도달하거나 timeout 될 때까지 관련 버튼을 비활성화한다.
- In Scope:
  - `apps/desktop/src/renderer/main.tsx` 의 `runnerActionKeys` 또는 동등 helper 를 `"starting"`, `"stopping_pending"`, `"stopping_force"`, `"restarting"`, `"config"`, `"run"`, `"dry-run"` 같은 의미 있는 action state 로 정리한다.
  - start 는 runner state `status=running` 도달 전까지, graceful/force stop 은 `status=stopped` 도달 전까지, restart 는 stop 이후 start 완료 전까지 action state 를 유지한다.
  - transition 중 같은 runner 의 start / stop / restart / config / run / dry-run 버튼을 비활성화한다. 단, graceful stop pending 중에는 확인 다이얼로그를 거쳐 force stop 으로 전환할 수 있는 명시 경로를 유지한다.
  - transition 중 버튼 라벨과 spinner 를 표시한다: `시작 중...`, `중지 예약 중...`, `강제 종료 중...`, `재시작 중...`.
  - IPC 호출 후 state 변화를 빠르게 감지하도록 renderer short-poll 또는 `apps/desktop/src/main.js` state file watch / push 알림을 추가한다. 어떤 방식을 선택하든 60초 timeout fallback 이 action state 를 강제 해제하고 경고 토스트를 표시한다.
  - `apps/desktop/src/renderer/styles.css` 에 transition 상태의 disabled tone, spinner alignment, runner card outline/background 피드백을 추가하거나 기존 class 를 확장한다.
- Out of Scope:
  - `prd_167` / `order_147` 이 다루는 graceful stop CLI/runtime semantics, `stop_pending` state write, `--force` CLI flag, `last_stop_reason` 값 변경.
  - runner timeout watchdog, process cleanup, detached runner reconnect policy, monitor runner 정책.
  - 데스크톱 전체 runner card 레이아웃 대개편, 디자인 시스템 교체, 새 MUI 패턴 추가.
  - `git push`.

## Global Acceptance Criteria

- [ ] Start 클릭 후 IPC 응답이 먼저 도착해도 해당 runner 의 control buttons remain disabled until observed runner state has `status=running`.
- [ ] Graceful stop 클릭 후 `stop_pending=true` 또는 equivalent pending evidence 가 관찰되는 동안 stop UI shows `중지 예약 중...` with a spinner, and normal start/restart/config/run/dry-run actions for that runner remain disabled until `status=stopped`.
- [ ] Graceful stop pending 중 force stop 경로는 확인 다이얼로그를 통해서만 활성화되고, force 선택 후 UI shows `강제 종료 중...` until `status=stopped`.
- [ ] Restart 클릭 후 action state remains active through the stop and subsequent start phases, and clears only after the final target state is observed.
- [ ] Transition state is tracked per runner id: one runner in `"starting"` or `"stopping_pending"` does not disable controls for another runner.
- [ ] If no target state is observed within 60 seconds, action state is cleared and a Korean warning toast tells the user that state 확인이 실패했으며 새로고침 또는 재시도를 권장한다.
- [ ] `apps/desktop/src/renderer/main.tsx` no longer clears start/stop/restart action state solely in the `finally` branch of the IPC call; the clear path is tied to state observation or timeout fallback.
- [ ] `npm run desktop:check` exits 0.
