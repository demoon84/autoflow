# Project PRD

## Meta

- Project Key: prd_010
- Title: Add tickets Kanban menu — board-style view across backlog / todo / inprogress / verifier / done / reject
- Status: populated

## Goal

데스크톱 사이드바에 "티켓 보드" 메뉴를 추가해, `.autoflow/tickets/` 폴더의 각 상태(backlog · todo · inprogress · verifier · done · reject)를 칸반 컬럼으로 시각화한다. 각 카드는 ticket / PRD 식별자와 제목, 담당 AI(있으면), 직전 활동 시각, project key 를 한눈에 보여주고 클릭하면 본문이 인라인 레이어(prd_009/PRD 핀 패턴 재사용)로 열린다. AI 가 어떤 작업을 어디 단계에 두고 있는지 시간순/상태별로 한 화면에서 파악하게 한다.

## Core Scope

### In Scope

- 사이드바 `settingsNavigation` 에 `{ key: "kanban", label: "티켓 보드", icon: KanbanSquare }` 추가 (작업 흐름 다음, Wiki 위 위치).
- 신규 `TicketKanban` 컴포넌트 (`apps/desktop/src/renderer/main.tsx`):
  - 6개 컬럼: `backlog` (PRD), `todo`, `inprogress`, `verifier`, `done`, `reject`. 빈 컬럼도 헤더는 노출 (X건 표기).
  - 각 컬럼 헤더: 한국어 라벨 (PRD 대기 / 실행 대기 / 구현 / 검증 / 완료 / 반려) + 카운트 배지 (shadcn Badge variant 매핑: default/secondary/destructive).
  - 컬럼 본문: 카드 리스트, 수정시각 내림차순 정렬.
  - 카드 구성:
    - 1행: 식별자 (PRD-NNN / tickets_NNN / Reject-NNN — `workflowFileDisplayName` 재사용).
    - 2행: 제목 (한 줄 ellipsis).
    - 3행 메타: 담당 AI (있으면 `AI-N`, 없으면 빈 자리), project key (`prd_NNN`), `formatDate(modifiedAt)`.
    - 변경 단계 색상: 좌측 3px 보더로 컬럼별 톤 구분 (todo=muted, inprogress=primary, verifier/done=chart-2, reject=destructive).
- 카드 클릭 → 기존 `WorkflowPinLayer` 와 동일한 shadcn Dialog 기반 본문 미리보기. 헤더에 "← 이전" 없음 (단일 카드 즉시 열림), `MarkdownViewer` 로 본문 렌더, ESC / outside click 으로 닫힘.
- 로딩: 클릭 시 `window.autoflow.readBoardFile({...options, filePath})` 호출. 로딩/에러 상태도 layer 안에서 처리.
- 데이터 매핑:
  - backlog: `board.tickets.backlog` 의 `prd_*.md` (PRD)
  - todo / inprogress / verifier / reject: 동일 폴더의 `tickets_*.md` 또는 `reject_*.md`
  - done: `board.tickets.done` recursive — `tickets_*.md` 만 (PRD 보관본 `prd_*.md` 는 backlog 컬럼에 안 보이지만 done 컬럼에는 노출)
  - 각 카드의 담당 AI 는 ticket 파일의 `- AI:` / `- Claimed By:` 필드 값을 추출 — board snapshot 에 이미 있으면 사용, 없으면 카드에서는 생략 (renderer 부담 X).
- 가로 스크롤: 6컬럼이 화면 폭을 초과하면 자연스럽게 가로 스크롤 가능. 각 컬럼은 `min-width: 240px` / `flex: 0 0 280px`.
- 빈 보드(어떤 컬럼도 데이터 없음) 처리: "처리할 티켓이 없습니다" 안내 + AI 관리 메뉴로 가는 링크 (또는 안내문).
- 도움말(prd_004 가 done 되면) 의 메뉴 가이드 카드에도 "티켓 보드" 항목 추가.

### Out of Scope

- 드래그 앤 드롭으로 카드 이동 (상태 전이) — 본 PRD 는 read-only 시각화.
- 사용자가 카드를 직접 편집 / 메모 추가 — 향후 PRD.
- 칼럼별 정렬 옵션 변경 UI (수정시각 내림차순 고정).
- 가상 스크롤 / 1000건 이상 대량 데이터 최적화 — 일반 운영 규모 가정.
- 모바일 / 좁은 폭 전용 레이아웃.
- 다른 AI 의 작업 진행 상태 실시간 watch — 기본 board snapshot refresh 주기에 의존.
