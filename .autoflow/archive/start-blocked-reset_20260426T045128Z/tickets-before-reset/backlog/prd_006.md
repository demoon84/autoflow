# Project Spec

## Meta

- PRD Key: prd_006
- Title: LLM-driven wiki — auto ingest, synth query, semantic lint, automated maintainer trigger
- Status: populated

## Goal

Karpathy LLM Wiki 패턴의 핵심 — LLM 이 wiki bookkeeping 을 자동 수행 — 까지 Autoflow 를 끌어올린다. 현재 deterministic / heuristic 으로만 동작하는 wiki update / query / lint 를, 기존 adapter+runner 인프라를 재사용해 LLM 호출 경로를 추가한다. finish-ticket-owner 가 pass 한 직후 wiki-maintainer 어댑터가 자동으로 실행되어 entity/concept 페이지(decisions / features / architecture / learnings) 를 새 done 티켓·handoff 로부터 생성·병합하고, 사용자는 `autoflow wiki query --synth` 로 LLM 합성 답변을, `autoflow wiki lint --semantic` 으로 페이지 간 모순을 받을 수 있게 한다.

## Core Scope

### In Scope

- Auto-trigger after pass:
  - `runtime/board-scripts/finish-ticket-owner.sh` 와 `.autoflow/scripts/finish-ticket-owner.sh` 의 pass 경로에서 기존 `auto_update_wiki` (deterministic) 직후, 새 단계 `auto_run_wiki_maintainer` 를 호출. 출력은 `wiki_maintainer.<key>=<value>` 형태로 prefix.
  - 동작:
    1. board 에 `wiki-maintainer` role 의 enabled runner 가 있는지 확인.
    2. 있으면 1-shot dry-run-aware 형태로 `autoflow run wiki <project_root> --runner <id>` 호출 (loop 모드에 영향 없음).
    3. 없거나 비활성화면 `wiki_maintainer.status=skipped_no_runner` 로 graceful skip.
    4. 어댑터 비활성/비인증/실패 시 `wiki_maintainer.status=failed` 로 비차단 기록 (전체 finish 는 그대로 pass).
  - 환경 변수 `AUTOFLOW_WIKI_MAINTAINER_AUTO=off` 로 비활성화 가능 (기본 on).
- Wiki maintainer agent procedure 명시화:
  - `.autoflow/agents/wiki-maintainer-agent.md` + scaffold 미러: 자동 트리거에서 어댑터가 실행될 때 따라야 할 절차를 단계별로 적는다. 입력(최신 done ticket / handoff / 기존 wiki 페이지), 출력(`wiki/decisions/<topic>.md` 신규 또는 병합, `wiki/learnings/<topic>.md`, `wiki/features/<topic>.md`, `wiki/architecture/<topic>.md`), 페이지 형식(template 준수), 인용 규칙(ticket/log 경로 명시), idempotent 원칙(같은 입력 두 번 = 같은 결과), 사람-편집 영역 보존(`AUTOFLOW:BEGIN/END` 외부는 손대지 않음).
- `autoflow wiki query --synth`:
  - 기존 grep 결과를 가지고 wiki-maintainer adapter 로 한 번 호출 → 사용자 질문에 대한 합성 답변과 함께 사용한 `result.N.path` 인용을 출력.
  - 출력 키 추가: `synth_status=ok|skipped|failed`, `synth_answer=<text>`, `synth_citation.N=<wiki path>`.
  - 어댑터가 없을 때는 `synth_status=skipped_no_adapter` 로 graceful, 기존 grep 결과는 그대로 유지.
- `autoflow wiki lint --semantic`:
  - 기본 lint(orphan/citation/stale) 후, 페이지 본문을 어댑터에 전달해 모순·오래된 주장 탐지.
  - 출력 키 추가: `semantic_status=ok|skipped|failed`, `semantic_finding.N.page=<path>`, `semantic_finding.N.kind=contradiction|stale_claim|missing_link`, `semantic_finding.N.summary=<text>`.
  - 어댑터 없을 때 graceful skip.
- CLI 도움말 / dispatcher:
  - `bin/autoflow` 와 `bin/autoflow.ps1` 사용법에 `--synth` / `--semantic` 플래그 표기.
  - `packages/cli/wiki-project.sh` 의 usage 와 `wiki-project.ps1` ValidateSet 갱신.
- Adapter 호출 헬퍼:
  - 새 함수 `wiki_invoke_adapter` (또는 기존 `runners-project.sh::run_adapter` 재사용)를 `cli-common.sh` 또는 `wiki-project.sh` 내부에 두어 query/lint synth/semantic 양쪽이 동일한 경로로 LLM 을 호출. 어댑터 결정 우선순위: 명시 플래그 > board 의 wiki-maintainer runner 의 agent/model 설정 > 기본값.
- 기본 wiki-maintainer runner 등록:
  - `autoflow init` 또는 `upgrade` 시 기본 runner 목록에 `wiki-maintainer-1` 을 disabled 상태로 추가 (사용자가 활성화하기 전엔 finish-pass 트리거 시 graceful skip). 기존 runners.toml 에 이미 등록돼 있으면 건너뜀.
- Documentation:
  - `AGENTS.md`, `CLAUDE.md`, `README.md` 의 Wiki 섹션에 "LLM 자동 ingest 흐름" 단락 추가.
  - `.autoflow/rules/wiki/README.md` 에 새 규칙: "automatic ingest preserves human-authored sections; runs are idempotent".
- 테스트:
  - 기존 `tests/smoke/ticket-owner-smoke.sh` 가 깨지지 않게 — wiki-maintainer runner 가 없는 상태에서 finish-pass 가 그대로 status=done 을 유지하는지 검증.
  - 새 smoke or 단위 검증: `autoflow wiki query --synth --no-tickets --no-handoffs --term wiki` 가 어댑터 부재 시 `synth_status=skipped_no_adapter` 를 출력하는지.

### Out of Scope

- 새로 생성되는 wiki 페이지의 자동 git add/commit — 사용자가 검토 후 직접 commit (push 는 어떤 자동화에서도 금지 유지).
- 임베딩 기반 검색·랭킹 — grep 기반 유지.
- 특정 LLM provider lock-in — 모든 호출은 기존 adapter 추상(`agents/adapters/*.md`) 통해 수행.
- 트리거 단어 변경 (`/af`, `#af` 그대로).
- 대화형 wiki 편집 UI / 데스크톱 데스크탑 패널 변경 — 본 PRD 는 backend / CLI 한정. 데스크톱 표시는 자연스럽게 wiki 파일 갱신을 통해 반영.
- `AUTOFLOW_WIKI_MAINTAINER_AUTO` 외 추가 옵션 (per-project 토글, 시간대 제한 등).
- finish-fail 경로에서의 LLM 호출 — pass 만 트리거.

## Global Rules

- LLM 호출은 기존 adapter 추상을 통해서만. 직접 provider SDK 호출 금지.
- wiki-maintainer 는 `wiki/` 만 수정. 다른 디렉터리 (`tickets/`, `logs/`, `conversations/`) 는 read-only.
- 사람-편집 영역(`AUTOFLOW:BEGIN/END` 바깥) 은 절대 덮어쓰지 않는다.
- 같은 입력으로 여러 번 실행해도 결과가 수렴해야 한다 (idempotent).
- finish-ticket-owner 의 done/commit 는 wiki-maintainer 결과와 무관하게 진행. 어댑터 실패는 비차단 기록.
- `git push` 는 어떤 자동화에서도 금지.
- 한국어 UI / 영어 AI 계약 (AGENTS.md rule 15).
