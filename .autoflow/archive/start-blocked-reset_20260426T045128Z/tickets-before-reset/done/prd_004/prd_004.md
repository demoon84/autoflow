# Project Spec

## Meta

- PRD Key: prd_004
- Title: Add in-app Help section explaining each sidebar menu and core terms
- Status: populated

## Goal

데스크톱 사이드바에 "도움말" 메뉴를 추가해, 각 사이드바 항목이 무엇을 보여주고 사용자가 어떤 행동을 할 수 있는지, 그리고 Autoflow 의 핵심 용어(spec / ticket / runner / verifier / handoff / wiki / hook)와 트리거(`/af`, `#af`, `/autoflow`, `#autoflow`)를 한 화면에서 안내한다. 첫 사용자가 외부 문서 없이도 메뉴와 데이터 흐름을 이해할 수 있게 한다.

## Core Scope

### In Scope

- 사이드바 `settingsNavigation` 에 `{ key: "help", label: "도움말", icon: HelpCircle }` 추가 (마지막 위치). lucide-react 의 `HelpCircle` (또는 `LifeBuoy`) 사용.
- `activeSettingsSection === "help"` 일 때 렌더되는 신규 `HelpSection` 컴포넌트.
- HelpSection 구성:
  1. 개요 카드: Autoflow 가 무엇이고 데스크톱이 어떤 보드(`.autoflow/`)를 보여주는지 1~2문단.
  2. 메뉴 가이드 (현재 사이드바의 모든 항목별 카드):
     - 작업 흐름 — AI runner 별 진행 단계와 현재 처리 중인 ticket. 데이터 출처: `runners/state/*.state`, `tickets/inprogress/`.
     - AI 관리 — runner(에이전트/모델/모드/주기) 설정, 시작/중지/추가/삭제. 데이터 출처: `runners/runners.toml`, `runners/state/*.state`, `runners/logs/`.
     - Wiki — 완료된 작업에서 파생된 지식과 raw source(handoff/ticket/log). 검색은 `autoflow wiki query` 호출. 데이터 출처: `wiki/`, `conversations/`, `tickets/done/`, `logs/`.
     - 처리 지표 — 메트릭 스냅샷 / 히스토리 차트. 데이터 출처: `metrics/*.jsonl`.
     - 자동화 상태 — 중단 방지 훅, 파일 감시기, 진단 등 자동화 health 한눈 보기.
   - 각 카드 구조: 메뉴 라벨 + 1줄 요약 + "여기서 할 수 있는 일" 불릿 2~4개 + "데이터 위치" 코드 라벨.
  3. 용어 사전 카드: spec / ticket / runner(ticket-owner) / verifier / handoff(spec-handoff) / wiki / stop-hook / file watcher 각각 한 줄 정의.
  4. 트리거 안내 카드: `/af` `#af` `/autoflow` `#autoflow` 가 무엇을 시작하는지, "저장 / 바꿔 / 취소" 흐름 요약, ticket 자동 처리로 이어진다는 점.
  5. 데이터 흐름 한눈에 카드: 한 다이어그램 또는 `대화 → backlog spec → ticket-owner runner → verify → done + commit (push 안 함)` 순서 박스/화살표 텍스트로 표시.
- Help 섹션 진입 시 `selectedLogPath` / 다른 섹션 state 에 영향 주지 않음.
- 신규 CSS 클래스 `.help-section`, `.help-card`, `.help-card-grid`, `.help-glossary`, `.help-flow` 추가. 기존 `.settings-section` grid 호환.

### Out of Scope

- 기존 메뉴 위에 추가되는 tooltip / popover 형태의 인라인 가이드.
- 검색 기능, 외부 사이트 링크 (offline 환경 가정), 영상/이미지 자산.
- 다국어 — 한국어 단일.
- README / wiki 페이지 변경.
- 아이콘 라이브러리 변경.
- 도움말 내용 자동 생성(메뉴 변경 시 자동 동기화) — 본 spec 은 정적 텍스트.
- "처음 실행 시 자동 안내(첫 실행 모달)" — 명시적으로 도움말 메뉴를 클릭해야 보임.

## Global Rules

- 사용자용 UI 문구는 한국어 기본 (AGENTS.md rule 15).
- `.settings-section` 의 기본 grid 동작을 깨면 안 됨 — Help 전용 클래스로 분리.
- 다른 섹션의 로직/렌더링/스타일 회귀 없어야 함.
- 텍스트는 현재 보드/메뉴 상태와 사실 일관 — "Wiki" 라벨 등 현재 코드 기준.
- 기존 owner smoke (`tests/smoke/ticket-owner-smoke.sh`) 회귀 없음.
