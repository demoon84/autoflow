# Project Spec

## Meta

- PRD Key: prd_002
- Title: Surface active spec/ticket per runner in workflow view
- Status: populated

## Goal

데스크톱 "작업 흐름" 패널에서 각 AI runner 가 현재 어떤 spec / ticket 을 처리 중인지 한눈에 파악할 수 있게 한다. 단계 점등(plan → todo → inprogress → verifier → done/reject)뿐 아니라 "지금 처리 중: tickets_NNN — <Title> (project_NNN)" 텍스트와 진행 단계가 runner 행에 명확하게 표시되게 만든다.

## Core Scope

### In Scope

- Runner 가 ticket 을 점유할 때 runner state 의 `active_item` 에 ticket id + title + stage + spec ref 가 들어가도록 보강 (`runtime/board-scripts/start-ticket-owner.sh`, `runtime/board-scripts/finish-ticket-owner.sh`).
- Runner state writer (`packages/cli/runners-project.sh` 의 `runner_write_state` 호출 지점) 가 빈 `active_item=` 으로 덮어쓰지 않고 기존 값을 보존하도록 정리. 명시적 클리어가 필요한 곳(완료/실패/stop) 에서만 비움.
- 새 키 추가: `active_ticket_id`, `active_ticket_title`, `active_stage`, `active_spec_ref` — `runner.<n>.active_*` 형태로 `runners list` 출력에 노출 (`packages/cli/runners-project.sh::list_runners`).
- 데스크톱:
  - `AutoflowRunner` 타입에 `activeTicketId`, `activeTicketTitle`, `activeStage`, `activeSpecRef` 추가 (`apps/desktop/src/renderer/vite-env.d.ts`).
  - main process IPC 매핑 (`apps/desktop/src/main.js`)에서 위 4개 필드 파싱.
  - `AiProgressRow` (`apps/desktop/src/renderer/main.tsx`)에 "지금 처리 중" 라인 추가: `<ticket_id> — <title>` + 작은 칩으로 stage. 클릭하면 해당 ticket 파일이 LogPreview 에 열린다.
  - `runnerProgressDetail` 이 `activeTicketTitle` 우선 사용, 머신 raw 로그 fallback 마지막.
- 비어 있을 때 (`runner.stateStatus === "idle"`) "대기 중 — 처리할 백로그/티켓 없음" 같은 인간 친화 메시지로 통일.
- scaffold 동기화: `runtime/board-scripts/start-ticket-owner.sh` / `finish-ticket-owner.sh` / `runners-project.sh` (CLI 는 packages 가 source) 를 살아 있는 `.autoflow/scripts/` 와 동일하게 유지.

### Out of Scope

- Runner 간 동시 처리(multi-ticket per runner) 지원 — 현재 1-runner = 1-ticket 모델 유지.
- thread context 파일(`current.context`) 의 모델 변경 — 그대로 둔다.
- 메트릭 키 이름 변경.
- Wiki 검색 / Sources 패널 변경 (prd_001 의 영역).

## Global Rules

- 사용자용 UI 문구는 한국어 기본, AI 계약 / 스크립트 로그는 영어 기본 (AGENTS.md rule 15).
- runner state 파일 포맷(`status= / active_item= / pid=` 등 key=value lines) 호환성 유지 — 기존 파서가 깨지면 안 됨.
- 신규 필드는 모두 추가 전용. 빈 값일 때도 키는 출력해 desktop 에서 안전히 파싱.
- 기존 owner smoke test (`tests/smoke/ticket-owner-smoke.sh`) 가 통과해야 한다.

- [ ] `AutoflowRunner` 타입에 `activeTicketId`, `activeTicketTitle`, `activeStage`, `activeSpecRef` 가 정의되고, IPC 가 이를 채운다.
