# Project Spec

## Meta

- PRD Key: prd_001
- Title: Restructure Wiki & Handoff panel — handoff as wiki source, not a peer
- Status: populated

## Goal

Karpathy LLM Wiki 의 3-레이어 모델(*raw sources → wiki → schema*)에 맞춰 데스크톱 UI 와 agent 계약을 정리한다. Handoff(`conversations/spec-handoff.md`)를 Wiki 와 동격 카테고리가 아니라, ticket·log 와 함께 wiki 의 입력 raw source 로 노출되도록 한 단계 내려서 사이드바와 패널 구조를 단순화한다.

## Core Scope

### In Scope

- 사이드바 네비게이션 라벨 `Wiki & Handoff` → `Wiki` 단일화 (`apps/desktop/src/renderer/main.tsx` 의 `settingsNavigation` 항목).
- Knowledge 섹션 패널 레이아웃을 (위→아래) `WikiQueryPanel` → `WikiList` → `Sources` collapsible 섹션 순서로 재구성.
- `Sources` collapsible 섹션 내부에 `HandoffList` 를 위치시킴. 기본 펼침 상태로 시작.
- Knowledge 섹션의 `aria-label` 도 `Wiki` 로 통일.
- `boardFileKind` 함수가 `conversations/` 경로에 대해 `Source · Handoff` 같은 raw-source 의미가 드러나는 라벨을 반환하도록 조정.
- HandoffList 의 헤더·빈 상태·메타 텍스트가 raw source 의미축과 일관되도록 수정 (예: "인수인계가 없습니다" 같이 동격을 시사하는 표현 제거).
- `wiki-maintainer-agent.md` 의 Inputs / Procedure 가 "conversation handoff 는 wiki 와 동격 산출물이 아니라 wiki 의 ingest 입력 소스" 라는 의미를 명시하도록 수정.
- scaffold 미러 동기화: `.autoflow/agents/wiki-maintainer-agent.md` 와 `scaffold/board/agents/wiki-maintainer-agent.md` 가 동일.
- CSS 보강: Sources collapsible 섹션 토글 스타일 (`apps/desktop/src/renderer/styles.css`).

### Out of Scope

- CLI 동작 변경. `autoflow wiki query` 는 이미 wiki/ticket/conversation 을 동격 raw source 로 검색하므로 손대지 않는다.
- handoff 를 LLM 으로 요약해 entity/concept 페이지로 자동 변환하는 흐름.
- `conversations/` 디렉터리·파일 경로·CLI 출력 키 이름 변경.
- IPC 시그니처 변경 (`controlWiki`, `readBoardFile` 등 그대로).
- 메트릭 키 (`handoff_count`) 이름 변경.

## Global Rules

- 사용자용 UI 문구는 한국어 기본, AI / 계약 문서는 영어 기본 (AGENTS.md rule 15).
- `wiki/` 는 source of truth 가 아니라는 원칙 유지 — 본 작업은 표시 계층 재배치이지 권위 변경이 아님.
- 기존 IPC (`controlWiki`, `readBoardFile`) 와 데이터 모델 (`wikiFiles`, `conversationFiles`) 은 그대로 사용.
- Verification 명령은 host project root 에서 실행한다.
