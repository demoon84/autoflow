# Project Spec

## Meta

- PRD Key: prd_003
- Title: Wiki panel — horizontal split + collapsible preview with toggle
- Status: populated

## Goal

Wiki 메뉴(설정 → Wiki)의 레이아웃을 현재의 상하 분리(목록 위 / 미리보기 아래)에서 좌우 분리(목록 왼쪽 / 미리보기 오른쪽)로 바꾼다. 미리보기는 기본 닫힌 상태로 두고, 사용자가 목록의 md 파일을 클릭하면 자동으로 펼쳐지면서 목록 영역이 줄어들고, 명시적인 토글 버튼으로 다시 접을 수 있게 한다. 현재처럼 항상 비어 있는 LogPreview 가 화면을 차지하는 어색함을 제거한다.

## Core Scope

### In Scope

- Wiki 섹션(`activeSettingsSection === "knowledge"`)의 레이아웃을 horizontal flex 로 전환:
  - 좌측: WikiQueryPanel + WikiList + Sources collapsible (현재 `tool-panel`).
  - 우측: LogPreview 영역.
- LogPreview 영역의 표시 여부를 제어하는 React state (`isWikiPreviewOpen`) 추가.
- 동작 규칙:
  1. 기본 상태 = `closed` (페이지 진입 시 미리보기 영역이 화면에서 사라지고 좌측 목록이 패널 전체 폭을 차지).
  2. 사용자가 WikiList / HandoffList / WikiQueryPanel 결과 중 하나라도 클릭(`onSelect` → `readLog` 호출 성공)하면 자동으로 `open`.
  3. `open` 상태에서 우측 LogPreview 헤더에 닫기(×) 버튼 노출 — 클릭 시 `closed` 로 토글.
  4. `closed` 로 닫혀도 좌측에서 같은 항목 다시 클릭하면 즉시 다시 `open`.
  5. `open` 상태에서 좌측 목록 폭은 줄고(예: `flex: 0 0 38%` 또는 `min-width: 280px; flex: 1`), LogPreview 가 나머지 폭(`flex: 1`)을 차지.
- 토글 가시성:
  - LogPreview 헤더(`.log-preview-header`)의 우측에 닫기 버튼 (lucide `X` icon, 기존 `Button` UI 컴포넌트 재사용).
  - 미리보기가 닫혀 있고 `selectedLogPath` 가 비어 있지 않은 경우에는 좌측 패널 헤더 영역에 작은 "미리보기 열기" 버튼이 나타나 다시 펼칠 수 있도록 함 (선택 항목 정보를 잃지 않은 상태에서 toggle).
- 패널 라이프사이클:
  - `activeSettingsSection` 이 `knowledge` 가 아니게 되면 `selectedLogPath` 와 `isWikiPreviewOpen` 을 초기화 (다른 섹션 영향 없게).
- CSS 보강:
  - 신규 클래스 `.knowledge-split` (horizontal flex 컨테이너), `.knowledge-list-pane` (좌측), `.knowledge-preview-pane` (우측), `.knowledge-preview-pane--hidden` (닫힌 상태 시 `display: none`).
  - 좁은 창 폭(예: `< 900px`)에서는 자동 stack 으로 fallback 하지만 그 경우에도 닫힘 기본은 유지.

### Out of Scope

- Snapshot / Automation / 기타 settings 섹션의 layout 변경 — 모두 그대로 유지.
- Resizable splitter (드래그로 폭 조절) — 고정 비율만.
- 미리보기 위치 사용자 저장(localStorage) — 세션 한정 토글 상태.
- LogPreview 컴포넌트 자체의 내부 마크다운 렌더링 변경.
- 다른 메뉴(작업 흐름, AI 관리, 진단)의 LogPreview 동작.
- 모바일/터치 전용 레이아웃 최적화.

## Global Rules

- 사용자용 UI 문구는 한국어 기본, AI 계약 문서는 영어 기본 (AGENTS.md rule 15).
- 다른 settings 섹션들이 공유하는 `.settings-section` 의 grid 동작을 깨면 안 된다 — 새 클래스로 분리하여 wiki 섹션만 horizontal layout 적용.
- LogPreview / WikiList / HandoffList / WikiQueryPanel 컴포넌트의 시그니처는 그대로 유지 — 외부에서 props 추가만 허용.
- 기존 owner smoke (`tests/smoke/ticket-owner-smoke.sh`) 회귀 없음.
