# Project PRD

## Meta

- Project Key: prd_009
- Title: Display worker IDs as AI-N inside ticket / log markdown content
- Status: populated

## Goal

ticket 본문(`AI:` / `Claimed By:` / `Execution AI:` / `Verifier AI:` 필드와 `## Notes` / `## Resume Context` / `## Result` / `## Reject History` / `## Verification` 안의 인라인 worker 지칭 텍스트, verifier 로그)에서 `AI-N` 으로 박혀 있는 worker 표기를 사용자에게 보일 때는 `AI-N` 으로 보이도록 통일한다. storage 식별자(`runner.id="owner-1"`, runners.toml, state file 이름) 와 ticket-owner role 키는 backward-compat 위해 그대로 두고, ticket / log 마크다운에 새로 기록되는 값과 기존 데이터의 사용자 노출 표기만 AI-N 형태로 정렬한다.

## Core Scope

### In Scope

- 보드 스크립트가 ticket / 로그 마크다운에 worker 식별자를 기록하는 모든 경로에서 사용자 노출 값을 `AI-N` 형태로 정규화 (`display_worker_id` 헬퍼).
- `runtime/board-scripts/common.sh` 에 `display_worker_id` 추가 + `start-ticket-owner.sh` / `finish-ticket-owner.sh` / `start-todo.sh` / `start-verifier.sh` / `verify-ticket-owner.sh` / `handoff-todo.sh` / `write-verifier-log.sh` 가 worker_id 노출 값을 헬퍼로 통과시키도록 수정.
- 데스크톱 markdown viewer 보조 처리 — 마이그레이션 누락분이 발견되면 텍스트 노드에서만 `AI-N` → `AI-N` 으로 보조 표기 (code block 보존).
- AGENTS.md / CLAUDE.md / scaffold 미러: "사용자 노출 worker 표기는 AI-N" 규칙 명시.
- 기존 라이브 데이터(`.autoflow/tickets/`, `.autoflow/logs/`) 마이그레이션은 사용자가 별도로 즉시 처리(이번 세션) 했으므로 본 PRD 의 자동화 대상은 신규 기록 경로 + viewer 보조에 집중.

### Out of Scope

- runner ID storage 변경 (config.toml `id = "owner-1"` → `ai-1`).
- runner role 키 (`role = "ticket-owner"`) 변경.
- 함수 / 변수명 (`runnerIdPrefix`, `owner_id`, `find_owned_inprogress_ticket`) 변경.
- runner state file 이름 (`runners/state/owner-N.state`) 변경.
- 사용자 입력 자유 텍스트 영역(`## Reject Reason`, `## Result.Summary`) 안의 raw 인용은 보존.
- code block / 인라인 ` ` 안의 토큰은 보존.
