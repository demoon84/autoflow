# Verifier Checklist Template

Use this checklist before pass or fail.

- [ ] Ticket path is correct.
- [ ] Referenced spec was read.
- [ ] `Allowed Paths` were respected.
- [ ] `Done When` items were checked.
- [ ] Acceptance criteria were checked.
- [ ] Verification command was run from the correct working root.
- [ ] Output and exit code were recorded.
- [ ] Browser evidence was collected only if needed.
- [ ] `Verification` and `Result` were updated.
- [ ] Completion log was written.
- [ ] No `git push` was run.
